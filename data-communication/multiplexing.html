다중화(Multiplexing)

다중화는 여러 신호를 하나의 전송 매체에 함께 실어 보내는 기술입니다. 자원을 나누는 기준에 따라 시간(TDM), 주파수(FDM), 파장(WDM), 코드(CDM)로 나뉘며, 수신 측은 반대로 분리하는 역다중화(Demultiplexing)를 수행합니다.

구체적인 개념 설명

케이블 하나, 무선 채널 하나, 광섬유 한 가닥은 비싼 자원입니다. 사용자마다 전용 매체를 깔 수 없으므로 여러 통신을 한 매체에 섞어 보내야 합니다. 이때 신호들이 서로 섞여도 수신자가 다시 분리할 수 있도록 시간, 주파수, 파장, 코드 같은 축을 나눠 쓰는 것이 다중화입니다.

다중화는 네트워크의 "여러 연결을 한 링크로 모으는" 기본 기술입니다. 전화망, 케이블 TV, 광통신, LTE/5G, Wi-Fi까지 거의 모든 통신 시스템의 핵심입니다.

비유 — 고속도로 차선

한 도로를 한 차만 쓰면 낭비입니다. 차선을 나누거나, 시간대를 나누거나, 차량 종류별 전용 차선을 만들면 같은 도로를 여러 사용자가 함께 쓸 수 있습니다. 다중화도 하나의 매체를 여러 신호가 나눠 쓰도록 차선을 만드는 일입니다.

왜 필요한가?

  • 매체 효율 — 비싼 링크 하나를 여러 사용자가 공유해 비용을 줄인다.
  • 동시성 — 여러 통화·스트림·센서 데이터를 동시에 전송.
  • 대역폭 활용 — 남는 시간/주파수/파장을 다른 신호가 사용.
  • 확장성 — 사용자가 늘어도 물리 매체를 매번 새로 깔지 않는다.

어떻게 동작하는가?

1) 다중화와 역다중화 흐름

다중화는 MUX에서 합치고 DEMUX에서 같은 기준으로 되돌리는 구조다.

2) TDM(Time Division Multiplexing)

시간을 작은 슬롯으로 쪼개 각 사용자에게 번갈아 할당합니다. 한 순간에는 한 사용자만 링크를 쓰지만, 매우 빠르게 번갈아 쓰기 때문에 동시에 쓰는 것처럼 보입니다.

TDM은 각 사용자가 자기 시간 슬롯에만 전송한다.
  • 동기식 TDM — 보낼 데이터가 없어도 슬롯이 고정으로 배정됩니다. 단순하지만 낭비 가능.
  • 통계적 TDM — 실제 보낼 데이터가 있는 사용자에게만 동적으로 슬롯을 배정합니다. 효율은 높지만 제어 정보가 필요합니다.

3) FDM(Frequency Division Multiplexing)

전체 주파수 대역을 여러 채널로 나눠 각 사용자에게 다른 주파수 범위를 할당합니다. 모든 사용자가 동시에 전송하지만 서로 다른 주파수 대역을 쓰므로 필터로 분리할 수 있습니다.

FDM은 라디오·케이블 TV처럼 주파수 대역을 나눠 여러 신호를 동시에 보낸다.

4) WDM(Wavelength Division Multiplexing)

광섬유에서 빛의 파장을 나눠 여러 채널을 동시에 보내는 방식입니다. 원리는 FDM과 비슷하지만 전기 신호의 주파수 대신 빛의 색(파장)을 나눕니다. DWDM(Dense WDM)은 아주 촘촘한 파장 간격으로 수십~수백 채널을 한 광섬유에 실을 수 있습니다.

5) CDM(Code Division Multiplexing)

같은 시간, 같은 주파수를 쓰지만 사용자마다 서로 다른 코드를 곱해 보냅니다. 수신자는 자신이 아는 코드로 상관(correlation)을 계산해 원하는 신호만 꺼냅니다. 코드들이 서로 직교(orthogonal)에 가까울수록 서로 간섭하지 않습니다. 

사용자 A 비트: +1
A 코드:       [+1, -1, +1, -1]
전송 칩열:    [+1, -1, +1, -1]

수신자는 A 코드와 내적 → 큰 값이면 A의 신호, 0에 가까우면 다른 사용자 신호

6) 다중화와 다중 접속의 차이

다중화(Multiplexing)는 여러 신호를 하나의 링크에 합치는 기술이고, 다중 접속(Multiple Access)은 여러 사용자가 공유 매체에 접근하는 규칙입니다. 이름은 비슷하지만 전자는 신호 결합/분리, 후자는 사용자 경쟁 제어에 가깝습니다. TDMA/FDMA/CDMA는 각각 TDM/FDM/CDM의 아이디어를 사용자 접속 방식으로 적용한 것입니다.

시간/공간 복잡도 또는 trade-off

방식나누는 축장점단점대표 사용
TDM시간주파수 대역 하나로 여러 사용자동기화 필요, 빈 슬롯 낭비디지털 전화망, 버스 통신
FDM주파수동시 전송 가능가드 밴드 필요, 필터 복잡라디오, 케이블 TV
WDM빛의 파장광섬유 용량 극대화광 장비 비용백본 광통신
CDM코드같은 시간/주파수 공유전력 제어와 코드 설계 어려움3G CDMA, GPS
통계적 TDM시간(동적)낭비 감소버퍼링/주소 정보 필요패킷 네트워크

실무에서 어디에 쓰이는가?

  • 광통신 백본 — WDM/DWDM으로 한 가닥 광섬유에 다수의 100G/400G 채널을 실음.
  • 케이블 TV / 라디오 — FDM으로 채널마다 다른 주파수 할당.
  • 이동통신 — TDMA, FDMA, CDMA, OFDMA 등 여러 다중화/다중접속 기술 조합.
  • 패킷 네트워크 — 통계적 다중화. 실제 트래픽이 있는 흐름이 링크를 동적으로 공유.

면접에 나올 질문과 답변

다중화가 필요한 이유는?
전송 매체는 비싸고 제한된 자원이므로 사용자마다 전용 매체를 줄 수 없습니다. 시간·주파수·파장·코드 같은 축을 나눠 하나의 매체를 여러 신호가 공유하게 만들어 비용과 대역폭 사용 효율을 높입니다.
TDM과 FDM의 차이는?
TDM은 같은 주파수 대역을 시간 슬롯으로 나눠 번갈아 쓰고, FDM은 시간을 동시에 쓰되 주파수 대역을 나눠 씁니다. TDM은 동기화가 중요하고, FDM은 채널 간 간섭을 막기 위한 가드 밴드와 필터가 중요합니다.
WDM은 FDM과 무엇이 다른가요?
원리는 거의 같지만 적용 대상이 광섬유입니다. FDM이 전기/무선 신호의 주파수 대역을 나눈다면, WDM은 빛의 파장(색)을 나눠 여러 광 채널을 동시에 보냅니다.
통계적 TDM은 왜 효율적인가요?
동기식 TDM은 사용자가 보낼 데이터가 없어도 슬롯이 예약되어 낭비됩니다. 통계적 TDM은 실제 데이터가 있는 사용자에게만 동적으로 슬롯을 배정하므로 평균 이용률이 높습니다. 대신 주소/제어 정보와 버퍼링이 필요합니다.
CDM은 어떻게 같은 주파수를 같이 쓰나요?
사용자마다 서로 다른 코드를 곱해 전송하고, 수신자는 원하는 사용자의 코드와 상관을 계산해 해당 신호만 꺼냅니다. 코드가 서로 직교에 가까우면 다른 사용자의 신호는 평균적으로 상쇄됩니다.
다중화와 다중 접속은 같은 말인가요?
다릅니다. 다중화는 여러 신호를 하나의 링크에 합쳐 보내는 기술이고, 다중 접속은 여러 사용자가 공유 매체에 접근하는 규칙입니다. TDMA/FDMA/CDMA는 다중화 아이디어를 사용자 접속 제어에 적용한 형태입니다.
