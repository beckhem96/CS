신호 인코딩(Line Coding)

신호 인코딩은 디지털 비트 0과 1을 실제 전송 매체의 전압·전류·빛의 세기·전이 패턴으로 표현하는 규칙입니다. 대표 방식은 NRZ, Manchester, 4B/5B, 8B/10B이며, 핵심 기준은 DC 성분, 클록 복구, 대역폭, 오류 검출 가능성입니다.

구체적인 개념 설명

컴퓨터의 비트는 추상적인 값이지만 케이블과 회로는 실제 전압 변화를 봅니다. 따라서 101100 같은 비트열을 어떤 전압 패턴으로 보낼지 정해야 합니다. 이 규칙이 라인 코딩(line coding), 즉 신호 인코딩입니다.

좋은 인코딩은 단순히 0과 1을 구분하는 것에서 끝나지 않습니다. 긴 0이나 1이 반복되어도 수신기가 타이밍을 잃지 않아야 하고, 평균 전압이 한쪽으로 치우치지 않아야 하며, 가능한 한 적은 대역폭을 써야 합니다.

비유 — 북소리로 0과 1 보내기

멀리 있는 사람에게 북으로 0과 1을 보내려면 "큰 소리는 1, 작은 소리는 0"처럼 약속이 필요합니다. 그런데 오랫동안 같은 소리만 반복되면 박자를 잃기 쉽죠. 그래서 중간에 일부러 박자 변화를 넣는 규칙이 필요합니다. Manchester나 8B/10B가 그런 역할을 합니다.

왜 필요한가?

  • 비트를 물리 신호로 변환 — 추상적 0/1을 전압 레벨이나 전이로 표현.
  • 클록 복구 — 수신기가 언제 비트를 읽어야 하는지 동기화.
  • DC 성분 제거 — 평균 전압이 한쪽으로 치우치면 변압기/커패시터 결합 링크에서 문제 발생.
  • 오류 검출 보조 — 허용되지 않는 코드 패턴이 나오면 오류로 판단 가능.

어떻게 동작하는가?

1) NRZ — 가장 단순한 전압 레벨

NRZ(Non-Return-to-Zero)는 1은 높은 전압, 0은 낮은 전압처럼 비트 동안 전압을 유지합니다. 단순하고 대역폭 효율이 좋지만, 000000이나 111111처럼 같은 비트가 길게 이어지면 전이가 없어 수신기가 클록을 잃기 쉽습니다.

2) Manchester — 매 비트 중간에 전이

Manchester 인코딩은 비트 중간에 반드시 전이를 넣습니다. 예를 들어 0은 high→low, 1은 low→high처럼 정의합니다(표준마다 반대일 수 있음). 전이가 항상 있으므로 클록 복구가 쉽지만, 전이가 많아 필요한 대역폭이 커집니다.

NRZ는 전이가 적어 효율적이지만 긴 동일 비트열에 약하고, Manchester는 전이가 많아 클록 복구가 쉽다.

3) 4B/5B — 긴 0을 막기 위한 블록 코딩

4비트 데이터를 5비트 코드로 바꿔 전송합니다. 5비트 코드 중 전이가 충분한 패턴만 골라 쓰기 때문에 긴 0이 이어지는 문제를 줄입니다. 4B/5B 자체는 보통 NRZI 같은 라인 코딩과 결합됩니다. 100Mbps 이더넷(100BASE-TX)에서 쓰였습니다. 

데이터 4비트 → 전송 5비트 예시
0000 → 11110
0001 → 01001
0010 → 10100
...
효율 = 4/5 = 80%

4) 8B/10B — DC 균형과 전이 보장

8비트 데이터를 10비트 코드로 바꿉니다. 0과 1의 개수 균형을 맞춰 DC 성분을 줄이고, 전이가 충분히 생기도록 설계되어 클록 복구가 쉽습니다. PCIe 1.x/2.x, SATA, Gigabit Ethernet 일부 물리 계층 등에서 사용됐습니다.

블록 코딩은 비트를 더 많이 쓰는 대신 클록 복구와 신호 품질을 얻는다.

5) DC 성분과 자기 동기화

  • DC 성분 — 신호 평균값이 0에서 멀어지는 성분. 길게 1만 이어지면 평균 전압이 올라가 회로가 포화되거나 기준 전압이 흔들릴 수 있습니다.
  • 자기 동기화(Self-clocking) — 신호 안의 전이만 보고 수신기가 클록을 복구할 수 있는 성질. 전이가 충분히 있어야 합니다.
  • Run length — 같은 레벨이 연속되는 최대 길이. 짧을수록 클록 복구가 쉽지만 대역폭은 더 듭니다.

시간/공간 복잡도 또는 trade-off

방식효율클록 복구DC 균형대표 사용
NRZ높음약함약함짧은/단순 링크
Manchester낮음강함좋음초기 이더넷

실무에서 어디에 쓰이는가?

  • 이더넷 물리 계층 — 10Mbps Manchester, 100Mbps 4B/5B, 10Gbps 이상 64B/66B 계열.
  • PCI Express / SATA — 과거 세대에서 8B/10B 사용, 이후 더 효율적인 128B/130B로 이동.
  • 광통신 — DC 균형과 클록 복구를 위해 블록 코딩과 스크램블링 사용.
  • 임베디드 버스 — 링크 특성에 맞춰 NRZ, Manchester, NRZI 등이 사용.

면접에 나올 질문과 답변

신호 인코딩과 변조는 뭐가 다른가요?
신호 인코딩은 디지털 비트를 전압/전이 패턴으로 표현하는 규칙이고, 변조는 그 신호를 반송파의 진폭·주파수·위상 등에 실어 보내는 과정입니다. 유선 baseband 링크에서는 라인 코딩이 핵심이고, 무선/광대역 전송에서는 변조가 함께 등장합니다.
NRZ의 문제점은?
같은 비트가 길게 이어지면 신호 전이가 없어 수신기가 클록을 복구하기 어렵고, 평균 전압이 한쪽으로 치우쳐 DC 성분 문제가 생길 수 있습니다. 대신 단순하고 대역폭 효율은 좋습니다.
Manchester 인코딩의 장단점은?
매 비트 중간에 전이가 있어 클록 복구가 쉽고 DC 균형도 좋습니다. 하지만 전이가 많아 같은 데이터율을 보내려면 더 넓은 대역폭이 필요합니다.
4B/5B와 8B/10B는 왜 비트를 늘리나요?
허용되는 코드워드를 골라 긴 0/1 반복을 막고 전이를 충분히 만들며, 특히 8B/10B는 0과 1의 균형까지 관리합니다. 비트 효율은 80%로 줄지만 클록 복구와 신호 안정성을 얻습니다.
DC 균형이 왜 중요한가요?
평균 전압이 한쪽으로 치우치면 AC 결합 회로나 변압기 결합 링크에서 기준점이 흔들리거나 신호가 왜곡될 수 있습니다. DC 균형 코드는 장시간 평균을 0에 가깝게 유지해 이런 문제를 줄입니다.
64B/66B가 8B/10B보다 좋은 점은?
8B/10B는 20% 오버헤드가 있지만 64B/66B는 64비트 데이터에 2비트 헤더만 붙여 오버헤드가 약 3%입니다. 고속 이더넷처럼 대역폭 효율이 중요한 곳에서 유리하며, 스크램블링으로 전이와 DC 특성을 보완합니다.
