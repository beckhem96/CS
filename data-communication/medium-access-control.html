매체 접근 제어(MAC, Medium Access Control)

매체 접근 제어(MAC)는 여러 장치가 하나의 공유 전송 매체를 사용할 때 누가 언제 말할지를 정하는 규칙입니다. 대표적으로 유선 이더넷의 CSMA/CD는 충돌을 감지하고, 무선 Wi‑Fi의 CSMA/CA는 충돌을 피하려고 시도합니다.

구체적인 개념 설명

하나의 회의실에서 여러 사람이 동시에 말하면 아무도 알아듣지 못합니다. 공유 네트워크 매체도 마찬가지입니다. 허브 기반 이더넷이나 무선 채널처럼 여러 장치가 같은 매체를 쓰면 동시에 전송할 때 충돌이 생깁니다. MAC은 이런 충돌을 줄이고, 생겼을 때 어떻게 복구할지 정합니다.

현대 스위치 기반 유선 이더넷은 대부분 full-duplex라 충돌이 사실상 사라졌지만, Wi‑Fi는 여전히 공기를 공유하기 때문에 MAC이 매우 중요합니다.

비유 — 무전기 대화

무전기는 한 사람이 말할 때 다른 사람이 말하면 소리가 겹칩니다. 그래서 말하기 전에 채널이 조용한지 듣고, 겹치면 잠시 기다렸다 다시 말합니다. CSMA 계열도 "먼저 듣고(carrier sense), 비었으면 말하고, 문제 생기면 기다리는" 규칙입니다.

왜 필요한가?

  • 충돌 감소 — 여러 장치가 동시에 전송해 프레임이 깨지는 일을 줄인다.
  • 공정성 — 한 장치가 매체를 독점하지 않도록 한다.
  • 처리량 유지 — 충돌과 재전송이 많아지면 실효 대역폭이 급감한다.
  • 무선 환경 대응 — 숨은 노드, 노출 노드 같은 유선에는 없는 문제가 있다.

어떻게 동작하는가?

1) CSMA — 말하기 전에 듣기

CSMA(Carrier Sense Multiple Access)는 전송 전에 채널이 사용 중인지 감지합니다. 비어 있으면 전송하고, 사용 중이면 기다립니다. 하지만 두 장치가 동시에 "비었다"고 판단해 전송하면 충돌이 생길 수 있습니다.

2) CSMA/CD — 충돌 감지

CSMA/CD(Collision Detection)는 전송 중에도 매체를 계속 감시해 충돌을 감지합니다. 충돌이 감지되면 즉시 전송을 중단하고 jam signal을 보낸 뒤, binary exponential backoff로 랜덤 시간 대기 후 재시도합니다.

CSMA/CD는 유선 공유 매체에서 가능하다. 송신 중 자기 신호와 매체 신호를 비교해 충돌을 알 수 있기 때문이다.

3) CSMA/CA — 충돌 회피

무선에서는 송신 중 자기 신호가 너무 강해 동시에 수신을 제대로 할 수 없고, 숨은 노드 문제도 있어 충돌 감지가 어렵습니다. 그래서 Wi‑Fi는 충돌을 감지하는 대신 충돌을 피하려고 합니다. 이것이 CSMA/CA(Collision Avoidance)입니다.

CSMA/CA는 DIFS, 랜덤 backoff, ACK로 충돌 가능성을 낮춘다.

4) 숨은 노드와 RTS/CTS

숨은 노드(hidden node)는 A와 C가 서로의 신호를 못 듣지만 둘 다 AP B와 통신 가능한 상황입니다. A와 C는 채널이 비었다고 착각해 동시에 B로 보내고, B에서 충돌이 생깁니다. RTS/CTS는 짧은 예약 메시지로 이를 줄입니다.

RTS/CTS는 데이터보다 짧은 제어 프레임으로 채널을 예약해 큰 데이터 충돌을 줄인다.

5) 오늘날 이더넷에서는 왜 CSMA/CD를 거의 안 쓰나?

초기 이더넷은 허브와 동축 케이블을 공유했기 때문에 충돌 도메인이 컸습니다. 하지만 현대 이더넷은 스위치 기반이고 각 링크가 full-duplex입니다. 송신과 수신 경로가 분리되어 있어 충돌이 발생하지 않습니다. 그래서 CSMA/CD는 역사적으로 중요하지만 현재 유선 LAN 운영에서는 거의 등장하지 않습니다.

실무에서 어디에 쓰이는가?

  • Wi‑Fi 튜닝 — 채널 혼잡, hidden node, RTS threshold, airtime fairness 문제 분석.
  • 스위치 네트워크 — full-duplex 환경에서는 충돌보다 duplex mismatch, loop, broadcast storm을 더 자주 본다.
  • IoT 무선망 — 저전력 공유 채널에서 backoff와 재전송이 배터리 수명에 영향.
  • 산업용 버스 — CAN처럼 우선순위 기반 arbitration을 쓰는 MAC도 있음.

면접에 나올 질문과 답변

CSMA/CD와 CSMA/CA의 차이는?
CSMA/CD는 전송 중 충돌을 감지하면 즉시 중단하고 backoff 후 재전송합니다. CSMA/CA는 무선에서 충돌 감지가 어렵기 때문에 전송 전에 랜덤 backoff, ACK, 필요시 RTS/CTS로 충돌을 피하려고 합니다.
무선에서 충돌 감지가 어려운 이유는?
송신 중 자기 신호가 너무 강해 동시에 다른 신호를 정확히 듣기 어렵고, 모든 노드가 서로의 신호 범위 안에 있지 않아 숨은 노드 문제가 생깁니다. 그래서 Wi‑Fi는 감지보다 회피 전략을 씁니다.
Binary exponential backoff란?
충돌 횟수가 늘어날수록 랜덤 대기 범위를 2배씩 키우는 방식입니다. 1회 실패 후 0~1 슬롯, 2회 후 0~3 슬롯, 3회 후 0~7 슬롯처럼 늘려 다시 동시에 충돌할 확률을 낮춥니다.
숨은 노드 문제는 무엇이고 어떻게 줄이나요?
A와 C가 서로의 신호를 못 듣지만 둘 다 AP B와 통신할 수 있어, 둘이 동시에 보내면 B에서 충돌하는 문제입니다. RTS/CTS로 AP가 채널 예약을 알리고 주변 노드가 NAV 동안 대기하게 해 완화합니다.
현대 이더넷에서 CSMA/CD가 사라진 이유는?
스위치 기반 full-duplex 링크에서는 송신과 수신 경로가 분리되어 충돌 도메인이 사라졌기 때문입니다. CSMA/CD는 허브/동축 케이블 기반 반이중 공유 이더넷에서 필요했던 기술입니다.
RTS/CTS를 항상 켜면 좋나요?
아닙니다. RTS/CTS 자체가 제어 프레임 오버헤드를 만들기 때문에 작은 프레임이나 숨은 노드가 적은 환경에서는 오히려 성능이 떨어질 수 있습니다. 큰 프레임 충돌 비용이 큰 환경에서 threshold를 조정해 사용합니다.
