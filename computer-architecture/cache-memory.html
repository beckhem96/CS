캐시 메모리(Cache Memory)

캐시 메모리란 CPU와 메인 메모리 사이에 있는 작고 빠른 저장 공간입니다. 자주 쓰거나 곧 다시 쓸 가능성이 높은 데이터를 미리 가까운 곳에 보관해, 느린 메모리 접근을 줄입니다. 핵심 원리는 지역성(Locality)입니다.

구체적인 개념 설명

캐시는 책상 위에 올려둔 자주 보는 책과 같습니다. 도서관 서가(메인 메모리)까지 매번 가면 느리니까, 지금 자주 보는 책과 관련 책 몇 권을 책상 위(L1/L2/L3 캐시)에 올려둡니다. 다음에 같은 책이나 근처 내용을 볼 때는 훨씬 빠르게 읽을 수 있죠.

CPU는 메모리보다 훨씬 빠릅니다. 메모리에서 데이터를 기다리는 동안 CPU가 놀게 되면 성능이 크게 떨어집니다. 캐시는 이 속도 차이를 숨기기 위해 만들어졌습니다. 보통 CPU 코어 가까운 순서로 L1 → L2 → L3 → RAM 계층이 있습니다.

지역성(Locality)

  • 시간 지역성(Temporal Locality) — 방금 쓴 데이터는 곧 다시 쓸 가능성이 높다. 예: 반복문 변수, 루프 안 객체.
  • 공간 지역성(Spatial Locality) — 어떤 주소를 읽으면 그 근처 주소도 곧 읽을 가능성이 높다. 예: 배열 순차 접근.

캐시는 보통 한 바이트만 가져오지 않고 캐시 라인(cache line) 단위(예: 64바이트)로 주변 데이터까지 함께 가져옵니다.

왜 필요한가?

현대 CPU는 한 사이클에 여러 명령어를 처리할 수 있을 만큼 빠르지만, 메인 메모리 접근은 수십~수백 사이클이 걸릴 수 있습니다. 이 차이를 줄이지 않으면 대부분의 시간이 계산이 아니라 메모리 대기에서 사라집니다.

그래서 좋은 성능의 코드는 알고리즘 복잡도뿐 아니라 캐시 친화성(cache friendliness)도 중요합니다. 같은 O(n)이라도 배열처럼 연속 메모리를 순회하는 코드는 빠르고, 포인터를 따라 여기저기 뛰는 코드는 캐시 미스 때문에 느릴 수 있습니다.

어떻게 동작하는가?

1) 메모리 계층 구조

캐시는 CPU와 RAM 사이의 속도 차이를 단계적으로 완충한다.

2) 캐시 히트와 캐시 미스

CPU가 어떤 주소의 데이터를 요청했을 때 캐시에 있으면 캐시 히트, 없으면 캐시 미스입니다. 미스가 나면 더 아래 계층(L2/L3/RAM)에서 데이터를 가져와 캐시에 올리고, CPU는 그만큼 기다립니다.

캐시 미스는 단순히 한 번 느린 게 아니라, 파이프라인 전체를 기다리게 만들 수 있다.

3) 배열 순회와 캐시 친화성

배열은 메모리에 연속으로 저장되므로 공간 지역성이 좋습니다. 한 원소를 읽으면 같은 캐시 라인에 있는 주변 원소들도 같이 올라옵니다. 

// 캐시 친화적: 행 우선 접근 (Java 2차원 배열에서도 내부 배열 단위 접근이 비교적 자연스러움)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        sum += arr[i][j];
    }
}

// 상대적으로 불리할 수 있음: 열 우선 접근
for (int j = 0; j < m; j++) {
    for (int i = 0; i < n; i++) {
        sum += arr[i][j];
    }
}

// 연결 리스트는 노드가 메모리 여기저기에 흩어질 수 있어 캐시 미스가 많다.

시간/공간 복잡도 또는 trade-off

캐시는 Big-O를 바꾸지는 않는 경우가 많지만, 상수 시간을 크게 바꿉니다. 같은 O(n)이라도 캐시 친화적인 O(n)과 캐시 미스가 많은 O(n)은 실제 실행 시간이 몇 배 이상 차이날 수 있습니다.

개념의미영향
Hit Rate요청 중 캐시에 있던 비율높을수록 빠름
Miss Penalty미스가 났을 때 추가 대기 시간RAM 접근은 매우 비쌈
Cache Line캐시에 올라오는 최소 단위공간 지역성을 활용
Associativity한 메모리 블록이 들어갈 수 있는 캐시 위치 수충돌 미스와 복잡도 trade-off

실무에서 어디에 쓰이는가?

  • 배열 vs 연결 리스트 선택 — 배열이 캐시 친화적이라 실제로 더 빠른 경우가 많습니다.
  • DB/검색 엔진 — B+Tree 노드 크기, 페이지 캐시, 버퍼풀 설계가 캐시/페이지 지역성을 활용합니다.
  • 고성능 서버 — false sharing, cache line padding, CPU affinity 같은 최적화와 연결됩니다.
  • 게임/그래픽스 — SoA(Structure of Arrays) 배치가 AoS보다 캐시 효율이 좋은 경우가 많습니다.
  • JVM 성능 — 객체가 흩어져 있으면 포인터 추적으로 캐시 미스가 증가하고, 원시 배열은 더 효율적입니다.

면접에 나올 질문과 답변

캐시 메모리는 왜 필요한가요?
CPU와 RAM의 속도 차이가 매우 크기 때문입니다. CPU가 매번 RAM을 기다리면 대부분의 시간이 대기 상태가 됩니다. 캐시는 자주 쓰거나 근처에 있는 데이터를 CPU 가까이에 저장해 메모리 접근 지연을 줄입니다.
시간 지역성과 공간 지역성을 설명해 주세요.
시간 지역성은 최근 사용한 데이터를 곧 다시 사용할 가능성이 높다는 성질입니다. 공간 지역성은 어떤 주소를 사용하면 그 주변 주소도 곧 사용할 가능성이 높다는 성질입니다. 캐시는 이 두 성질을 활용해 같은 데이터나 근처 데이터를 미리 빠른 저장소에 올려둡니다.
배열이 연결 리스트보다 실제로 빠른 경우가 많은 이유는요?
배열은 메모리에 연속으로 배치되어 공간 지역성이 좋고, 캐시 라인에 주변 원소들이 함께 올라옵니다. 연결 리스트는 노드가 메모리 곳곳에 흩어질 수 있어 다음 노드로 이동할 때마다 캐시 미스가 발생할 가능성이 큽니다. 그래서 Big-O만 보면 둘 다 O(n) 순회라도 실제 성능은 배열이 훨씬 빠른 경우가 많습니다.
캐시 미스의 종류에는 무엇이 있나요?
크게 compulsory miss, capacity miss, conflict miss가 있습니다. compulsory miss는 처음 접근이라 어쩔 수 없는 미스, capacity miss는 캐시 용량이 부족해서 나는 미스, conflict miss는 캐시의 특정 위치에 여러 메모리 블록이 몰려 서로 밀어내며 나는 미스입니다.
