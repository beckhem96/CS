DMA / I/O 시스템 / 버스 — CPU 없이 흐르는 데이터

DMA(Direct Memory Access)는 CPU를 거치지 않고 장치와 메모리가 직접 데이터를 주고받게 해 주는 기법입니다. 넓게는 I/O 시스템이 "CPU·메모리·장치 사이를 어떤 방식·어떤 길로 데이터가 오가는가"를 다루는 영역이고, 그 길이 바로 버스(Bus)입니다. 이 셋을 함께 보면 CPU가 어떻게 디스크·NIC·GPU와 협업하는지가 한눈에 들어옵니다.

구체적인 개념 설명

디스크에서 1MB를 읽어와 메모리에 올리는 작업을 생각해 봅시다. CPU가 한 워드씩 직접 옮긴다면, 그동안 다른 일을 못 합니다. 디스크는 ms 단위로 느리니 CPU가 수십만 ~ 수백만 사이클을 허비하게 됩니다. 이 낭비를 없애는 가장 강력한 방법이 DMA입니다. CPU가 "이 주소부터 1MB를 메모리의 저쪽 주소로 옮겨라"고 DMA 컨트롤러에 시킨 뒤 다른 일을 보면, 전송이 끝났을 때 인터럽트로 알려 줍니다.

하지만 DMA는 혼자 동작하지 않습니다. 버스라는 공유 통로를 두고 CPU·메모리·장치가 함께 사용하므로, 누가 언제 사용할지 중재가 필요합니다. 그리고 장치와 통신하는 방식도 두 가지(MMIO vs 포트 I/O)가 있고, 시스템에 따라 버스 구조도 다양합니다. 이 큰 그림을 알면 NVMe·GPU·RDMA 같은 고성능 I/O가 왜 그 모양인지 이해할 수 있습니다.

I/O 처리는 항상 셋 중 하나다

  • Programmed I/O (PIO) — CPU가 직접 한 워드씩 옮긴다. 단순하지만 느림.
  • Interrupt-driven I/O — 장치가 준비될 때마다 CPU에게 인터럽트로 알려줘 한 워드씩 옮긴다.
  • DMA — DMA 컨트롤러에게 위임하고 CPU는 다른 일. 전송 종료 시 한 번만 인터럽트.

왜 필요한가?

  • CPU 시간 절약 — 대용량 전송에서 CPU를 자유롭게 두면 다른 프로세스가 그 시간을 쓸 수 있어 처리량이 크게 늘어납니다.
  • 응답성 — 인터럽트 기반 I/O는 폴링보다 응답성이 좋고, DMA는 인터럽트 횟수까지 줄여 줍니다.
  • 대역폭 — 메모리 ↔ 장치 직통 경로는 CPU를 거칠 때보다 전송 대역폭이 훨씬 큽니다.
  • 모듈화 — 버스 표준 덕분에 NIC·SSD·GPU를 끼우기만 하면 동작합니다 — CPU가 모든 장치를 직접 알 필요가 없습니다.

어떻게 동작하는가?

1) I/O 처리 방식 3가지 비교

3가지 I/O 방식 — CPU의 개입 정도와 인터럽트 발생 횟수가 다르다.

2) DMA 동작 단계

DMA의 핵심 — 디스크립터(어디서·어디로·얼마) 한 번 셋업하면 그 뒤는 CPU 무관.

3) MMIO vs 포트 I/O — 장치를 어떻게 부를 것인가

CPU가 장치 레지스터에 접근하는 방식은 두 가지입니다. MMIO(Memory-Mapped I/O)는 장치 레지스터를 메모리 주소 공간의 일부로 매핑해 일반 load/store 명령으로 접근하고, 포트 I/O는 별도 I/O 주소 공간을 두고 in/out 같은 전용 명령으로 접근합니다. 현대 시스템은 거의 MMIO 중심입니다 — 같은 명령 세트, 같은 권한 모델, 같은 캐시 정책을 그대로 쓸 수 있기 때문입니다.

4) 시스템 안의 버스 — 누가 무엇을 잇는가

옛날의 단일 시스템 버스 → 메모리 버스와 I/O 버스 분리 → PCIe로 점대점 진화.

5) PCIe — 현대 I/O의 기본

PCIe(PCI Express)는 점대점 시리얼 링크 기반의 I/O 버스 표준입니다. 옛 PCI의 공유 병렬 버스와 달리, 각 장치가 CPU와 독립적인 레인을 갖고 양방향 전이중으로 통신합니다. 레인 수(x1·x4·x8·x16)와 세대(Gen3·4·5)로 대역폭이 정해지고, NVMe SSD·GPU·고속 NIC가 모두 PCIe 위에서 동작합니다. PCIe 장치는 자기 메모리 영역을 호스트의 주소 공간에 MMIO로 매핑받아, CPU가 해당 주소에 쓰면 곧 장치 레지스터에 쓰기가 됩니다.

6) DMA와 캐시 일관성

DMA가 CPU를 거치지 않고 메모리에 쓰면, CPU의 캐시는 옛 값을 가질 수 있습니다. 반대로 CPU가 캐시에만 쓴 값이 메모리에 반영되지 않은 상태에서 DMA가 메모리를 읽으면 옛 데이터를 가져갑니다. 이 문제를 해결하는 두 가지 길이 있습니다.

  • 캐시 일관성 하드웨어 — 현대 x86·ARM은 PCIe DMA가 캐시 일관성 프로토콜에 참여해 자동으로 해결합니다(coherent DMA).
  • 비일관성 DMA — 임베디드/일부 ARM은 DMA가 캐시와 협력하지 않으므로, 드라이버가 캐시 flush/invalidate를 명시적으로 호출해야 합니다.

RDMA·io_uring·NVMe 같은 고성능 I/O 스택은 이 일관성 비용까지 제거하기 위해 장치가 사용자 메모리에 직접 접근하도록 설계됩니다.

실무에서 어디에 쓰이는가?

  • NVMe SSD — PCIe 위에서 호스트 메모리에 직접 DMA로 데이터를 옮겨, 디스크 I/O를 GB/s 단위로 가능하게 만듭니다.
  • 고속 NIC와 RDMA — 100Gb 이더넷, InfiniBand 같은 환경에서 NIC가 사용자 공간 메모리에 직접 read/write합니다.
  • GPU 컴퓨팅 — CUDA의 cudaMemcpyAsync는 PCIe DMA로 호스트↔GPU 메모리를 전송합니다. 최신 시스템은 GPUDirect로 NIC↔GPU 직통도 가능.
  • 오디오·영상 — 사운드 카드·캡처 카드가 DMA로 버퍼를 채우고, CPU는 재생 시점만 동기화.
  • io_uring — 사용자 공간 큐 + 커널 처리 + DMA 조합으로 시스템 콜 비용까지 줄여 NVMe·네트워크 I/O를 극한까지 끌어올립니다.
  • 임베디드 — STM32 같은 MCU도 DMA 컨트롤러를 내장해 SPI·UART·ADC 데이터를 CPU 개입 없이 메모리로 옮깁니다.

면접에 나올 질문과 답변

DMA가 무엇이고 왜 필요한가요?
DMA는 CPU가 데이터 복사에 직접 참여하지 않고도 장치와 메모리가 직접 데이터를 주고받게 해 주는 기법입니다. 대용량 전송에서 CPU가 한 워드씩 옮기면 그 시간 동안 다른 일을 못 하므로, DMA 컨트롤러에게 위임하면 CPU는 다른 프로세스를 실행하고 전송 종료 시 한 번의 인터럽트만 받습니다. 결과적으로 시스템 처리량이 크게 늘고 응답성도 좋아집니다.
Programmed I/O, 인터럽트 기반 I/O, DMA는 어떻게 다른가요?
Programmed I/O는 CPU가 한 워드씩 폴링하며 직접 옮기는 방식으로 단순하지만 CPU를 100% 점유합니다. 인터럽트 기반 I/O는 장치가 준비될 때마다 CPU에 인터럽트를 걸어 한 워드씩 복사하는 방식으로, CPU 시간을 일부 회수할 수 있지만 워드마다 인터럽트가 발생합니다. DMA는 전송 전체를 컨트롤러에 위임하고 종료 시 한 번만 인터럽트를 받으므로, 대용량 전송에 압도적으로 유리합니다.
MMIO와 포트 I/O의 차이는 무엇인가요?
MMIO는 장치 레지스터를 메모리 주소 공간의 일부로 매핑해 일반 load/store 명령으로 접근하는 방식이고, 포트 I/O는 별도 I/O 주소 공간을 두고 in/out 같은 전용 명령으로 접근하는 방식입니다. MMIO는 같은 명령·같은 권한·같은 주소 폭을 그대로 쓸 수 있어 현대 시스템의 표준이 되었고, 포트 I/O는 x86 레거시 장치(PIC, PIT 등)에 일부 남아 있습니다.
DMA와 캐시 일관성은 어떤 관계인가요?
DMA가 메모리에 직접 쓰면 CPU 캐시의 옛 값과 어긋날 수 있고, 반대로 CPU가 캐시에만 쓴 값이 메모리에 반영되지 않은 상태에서 DMA가 읽으면 옛 데이터를 가져갈 수 있습니다. x86·ARM 같은 현대 데스크톱·서버 CPU는 PCIe DMA가 캐시 일관성 프로토콜에 참여해 자동으로 해결하지만(coherent DMA), 일부 임베디드는 드라이버가 직접 캐시 flush/invalidate를 호출해 일관성을 보장해야 합니다.
PCIe는 옛 PCI와 무엇이 다른가요?
옛 PCI는 여러 장치가 한 병렬 버스를 공유하는 구조였습니다 — 한 장치만 동시에 쓸 수 있었고, 속도도 모두가 같이 영향을 받았습니다. PCIe는 점대점(point-to-point) 시리얼 링크로, 각 장치가 CPU와 자기만의 레인을 갖고 양방향 전이중으로 통신합니다. 그래서 한 장치가 바빠도 다른 장치의 대역폭에 영향을 주지 않고, 레인 수(x1~x16)와 세대(Gen3~Gen5)로 필요한 대역폭을 자유롭게 할당할 수 있습니다.
DMA의 단점은 무엇인가요?
DMA 컨트롤러도 CPU와 같은 메모리 버스를 사용하므로, DMA가 활발하면 CPU가 메모리 접근에서 잠시 멈추는 cycle stealing이 발생합니다. 또한 DMA 시작 전 디스크립터(주소·길이) 셋업이 필요하고, 캐시 일관성을 유지하기 위한 비용도 있습니다. 그래서 짧은 전송에서는 DMA의 셋업 비용이 PIO보다 클 수 있고, 보통 일정 크기 이상의 전송에서만 DMA가 이득입니다.
