🖥️ 컴퓨터구조

컴퓨터구조(Computer Architecture)는 프로그램이 실제 하드웨어 위에서 어떻게 실행되는지 이해하는 분야입니다. CPU, 메모리, 캐시, 명령어, 인터럽트 같은 개념을 알면 "왜 이 코드가 빠른가/느린가"를 더 깊게 설명할 수 있습니다. 여기서는 추상적인 하드웨어 용어를 실행 흐름과 그림 중심으로 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 예: 가상 메모리 → TLB / 캐시 → MESI / 파이프라이닝 → 분기 예측.

🧩 핵심 개념

컴퓨터구조를 처음 배우거나 면접 직전에 한 번 훑기 좋은 우선순위 개념들.

CPU와 명령어 실행

프로그램 명령어가 CPU 안에서 어떤 부품을 거쳐 실행되는지 다룹니다.

  • CPU 구성 요소 ALU, 제어 장치, 레지스터, 버스가 함께 명령어를 실행하는 구조.
  • 명령어 사이클 Fetch → Decode → Execute → Writeback 흐름으로 한 명령어가 처리되는 과정.
  • 파이프라이닝(Pipelining) 명령어 단계를 겹쳐 처리해 처리량을 높이는 구조. 해저드가 핵심.

메모리 계층

CPU와 메모리 사이의 속도 차이를 줄이기 위한 계층 구조와 주소 변환을 다룹니다.

  • 캐시 메모리 L1/L2/L3 캐시, 지역성 원리, 캐시 히트/미스와 성능 영향.
  • 가상 메모리 / MMU 가상 주소를 물리 주소로 바꾸고 프로세스별 독립 주소 공간을 제공.

시스템 이벤트

CPU가 외부 장치나 예외 상황에 반응하는 방식입니다.

  • 인터럽트(Interrupt) 외부 장치나 예외가 CPU의 현재 흐름을 잠시 멈추고 처리 루틴으로 이동시키는 메커니즘.

I/O와 버스

CPU와 메모리·주변 장치가 실제로 데이터를 주고받는 통로 — 인터럽트 페이지에서 한 줄로만 다뤘던 영역.

  • DMA / I/O 시스템 / 버스 작성 예정 Programmed I/O vs Interrupt-driven vs DMA, 메모리 버스/시스템 버스/PCIe, MMIO와 포트 I/O.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 주제를 별도 페이지로 정리합니다.

주소 변환·캐시 심화

  • TLB(Translation Lookaside Buffer) 페이지 테이블 조회 결과를 캐싱해 주소 변환 비용을 줄이는 작은 캐시.
  • 캐시 일관성(MESI) 멀티코어에서 여러 캐시의 같은 데이터가 서로 어긋나지 않게 맞추는 규칙.

성능 최적화 심화

  • CPU 분기 예측 분기 결과를 미리 추측해 파이프라인을 계속 채우고, 실패 시 flush 비용을 치르는 구조.
  • 메모리 일관성 모델 작성 예정 happens-before, acquire/release, sequential consistency, JMM.
  • NUMA 구조 작성 예정 소켓 간 메모리 접근 비용, 노드 어피니티.
  • 명령어 수준 병렬성(ILP) 작성 예정 Out-of-Order 실행, 슈퍼스칼라, 명령어 재배치.
  • SIMD / 벡터 명령어 작성 예정 SSE/AVX, 자동 벡터화, 데이터 병렬.
