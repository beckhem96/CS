🌐 Web (기술 트랙)

Web은 단일 기술이 아니라 네트워크·문서 구조·스타일·언어·런타임이 협업하는 거대한 플랫폼입니다. 여기서는 면접·실무에서 매번 나오는 핵심 — 동작 큰 그림, HTML/CSS/JS의 기본 모델, DOM·이벤트·렌더링 파이프라인, HTTP API와 보안 — 을 "왜 그렇게 동작하는가"와 "어디서 실수하는가" 중심으로 정리합니다.

📚 학습 가이드 — 큰 그림 → 도구 → 런타임 → 보안

먼저 Web 동작 큰 그림으로 전체 흐름(주소 입력 → 렌더링)을 잡고, HTML / CSS / JavaScript의 모델을 차례로 본 뒤, DOM·렌더링 파이프라인 같은 런타임 동작과 HTTP API·보안 주제로 넘어가세요.

🧩 핵심 개념

웹 개발자가 "이건 한 번은 끝까지 읽고 가야 한다"고 말하는 우선순위 주제들.

큰 그림

주소 한 줄이 화면이 되기까지 무슨 일이 일어나는가.

  • Web 동작 큰 그림 — DNS·TCP·TLS·HTTP·HTML·CSS·JS URL 입력부터 첫 픽셀까지의 전체 흐름과 각 단계의 책임.

3대 언어

웹의 표현·스타일·동작을 책임지는 세 언어의 기본 모델.

  • HTML 핵심 — 시맨틱·form·접근성·SEO 의미 있는 마크업, 폼 처리, 접근성(a11y)의 기본기.
  • CSS 핵심 — 박스 모델·cascade·flex·grid 스타일이 어떻게 결정되고 어떻게 배치되는가.
  • JavaScript 핵심 — 실행 컨텍스트·closure·prototype 호이스팅, 스코프 체인, this, 모듈 시스템.

브라우저 런타임

JS·HTML·CSS가 만나 화면이 만들어지는 메커니즘.

  • DOM과 이벤트 — 트리·버블링·delegation DOM API의 비용, 이벤트 흐름과 위임 패턴.
  • 브라우저 렌더링 파이프라인 — Layout·Paint·Composite CSSOM/Render Tree 생성과 reflow·repaint 비용.

네트워크와 보안

서버와 통신하고, 사용자를 안전하게 지키기.

  • HTTP와 Web API — fetch·REST·CORS·스토리지 XHR에서 fetch까지, 쿠키 vs localStorage, JSON과 REST.
  • 웹 보안 기본 — XSS·CSRF·CSP·SameSite 가장 자주 나는 공격과 표준화된 방어 기제.

🔬 심화 / 상세

핵심을 한 번 본 뒤 더 깊이 파고들 만한 주제들. (작성 예정)

JS 런타임 심화

  • JavaScript 이벤트 루프 심층 작성 예정 call stack, task queue, microtask, rendering frame, async/await.
  • TypeScript 입문 작성 예정 structural typing, union/intersection, generic, type narrowing.

스토리지 / 통신

  • 브라우저 저장소 작성 예정 Cookie / localStorage / sessionStorage / IndexedDB / Cache Storage.
  • WebSocket / SSE / WebRTC 작성 예정 실시간 통신 선택 기준.
  • OAuth / OIDC 웹 로그인 흐름 작성 예정 Authorization Code + PKCE, token storage 위험.

성능과 아키텍처

  • 웹 성능 최적화 작성 예정 Core Web Vitals, LCP/INP/CLS, lazy loading, code splitting.
  • Service Worker / PWA 작성 예정 offline cache, push notification, background sync.
  • 프론트엔드 아키텍처 작성 예정 SPA / MPA / SSR / SSG / ISR, hydration, island architecture.
