🗄️ 데이터베이스

데이터베이스(Database)는 많은 데이터를 안전하게 저장하고, 빠르게 찾고, 동시에 여러 사용자가 다뤄도 깨지지 않게 만드는 시스템입니다. "어떻게 빨리 찾는지(인덱스)", "어떻게 일관성을 지키는지(트랜잭션·락·MVCC)", "어떻게 효율적으로 합치는지(조인·실행계획)" — 이 세 줄기를 따라가면 됩니다. 여기서는 면접·실무에서 가장 자주 등장하는 관계형 DB 핵심 개념을 시각화와 함께 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 예: 인덱스의 큰 그림 → B+Tree 내부 구조 / 격리 수준 → MVCC / WAL.

🧩 핵심 개념

DB를 처음 배우거나 면접 직전에 한 번 훑기 좋은 우선순위 개념들.

스키마 설계

테이블을 어떻게 나눠야 중복과 이상(anomaly)을 줄일 수 있는가.

  • 정규화 함수 종속과 1NF~BCNF, 비정규화의 정당화.

빠른 조회

"왜 인덱스만 있으면 빠른가"를 자료구조 관점으로 이해하기.

  • 인덱스 B+Tree 동작, 클러스터 vs 논클러스터, 커버링/복합 인덱스, 인덱스를 못 쓰는 패턴.

일관성과 동시성

여러 사용자가 동시에 같은 데이터를 다뤄도 깨지지 않게 만드는 도구들.

  • 트랜잭션과 ACID 원자성·일관성·격리성·지속성을 어떻게 보장하는지(WAL, 락).
  • 격리 수준 4단계 격리 수준과 발생 가능한 5가지 현상 매트릭스.
  • 락 공유/배타, 행/페이지/테이블, 인텐션·gap·next-key 락, 데드락.
  • MVCC 잠그지 않고 읽기 — PostgreSQL과 Oracle의 두 가지 구현 방식.

쿼리 실행

옵티마이저가 어떤 결정을 내리고, 그것을 어떻게 읽는가.

  • 조인 알고리즘 Nested Loop / Hash / Sort-Merge 단계별 시각화와 선택 기준.
  • 실행 계획(EXPLAIN) 읽기 Seq Scan / Index Scan / Bitmap, cost와 rows의 의미.

SQL과 데이터 모델링

다른 페이지가 모두 전제로 깔고 있는 — 스키마와 쿼리 자체.

  • ER 모델링과 DB 설계 엔티티·관계·카디널리티, 정규화 이전 단계의 스키마 설계 흐름.
  • SQL 기본 (DDL / DML / DCL / TCL) CREATE/ALTER/DROP, SELECT/INSERT/UPDATE/DELETE, GRANT/REVOKE, BEGIN/COMMIT 큰 그림.
  • 복제 (Replication) 동기/비동기 복제, 리더-팔로워, 읽기 분산, 페일오버.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 메커니즘을 별도 페이지로 정리합니다. (작성 예정)

스토리지 엔진 심화

  • WAL과 체크포인트 작성 예정 Write-Ahead Logging의 동작과 복구 절차.
  • B+Tree 내부 구조 심층 작성 예정 페이지 분할/병합, fill factor, 인덱스 단편화.
  • 인덱스 심화 작성 예정 함수 인덱스, 부분 인덱스, GiST/GIN, 멀티컬럼 정렬.

분산 / 확장성 심화

  • 샤딩과 파티셔닝 작성 예정 수평/수직 분할, 라우팅 키 선택, 리밸런싱.
  • 레플리케이션 작성 예정 동기/비동기, 마스터-슬레이브, 일관성 수준.
  • CAP / PACELC 작성 예정 분산 시스템의 트레이드오프와 사례 매핑.
  • 분산 트랜잭션 작성 예정 2PC, 3PC, Saga, Outbox 패턴.
  • NoSQL 데이터 모델 작성 예정 Document / Key-Value / Wide-Column / Graph.
