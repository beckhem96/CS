🛠️ 소프트웨어공학

소프트웨어공학(Software Engineering)은 "어떻게 코드를 만들 것인가"를 넘어 "어떤 절차로 만들고, 어떻게 확장 가능한 구조로 짜고, 어떻게 안전하게 배포할 것인가"를 다룹니다. 여기서는 면접·실무에 가장 자주 등장하는 핵심 — 개발 방법론(SDLC), OOP와 SOLID, 디자인 패턴, 테스트, CI/CD를 시각화와 함께 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 예: 디자인 패턴 입문 → 생성/구조/행위 패턴 단독 페이지 / 테스트 전략 → 테스트 더블·TDD 실전.

🧩 핵심 개념

소프트웨어공학을 처음 배우거나 면접 직전에 한 번 훑기 좋은 우선순위 개념들.

개발 프로세스

"어떤 순서·박자로 만들 것인가"를 정하는 방법론.

  • SDLC — 워터폴 / V-모델 / 애자일 / 스크럼·칸반 전통 모델부터 애자일까지, 각 모델이 풀려 한 문제와 한계.

코드 설계 패러다임

객체지향과 그 위에 얹히는 설계 원칙·재사용 가능한 해법.

  • OOP — 객체지향 프로그래밍 캡슐화·상속·다형성·추상화, 클래스/객체, 메시지 패싱.
  • SOLID 원칙 SRP·OCP·LSP·ISP·DIP — 위반 vs 준수 코드로 본 5원칙.
  • 디자인 패턴 입문 GoF 23개 한눈에, 생성·구조·행위 카테고리.

품질 보증과 배포

"안전하게 만들고, 반복 가능하게 배포한다"의 두 축.

  • 테스트 전략 테스트 피라미드, 단위·통합·E2E, Mock·Stub·Spy·Fake, TDD.
  • CI / CD 파이프라인 빌드·테스트·배포 자동화, Blue-Green / Canary / Rolling.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 주제를 별도 페이지로 정리합니다. (작성 예정)

디자인 패턴 단독

  • 생성 패턴 작성 예정 Singleton / Factory / Builder / Prototype.
  • 구조 패턴 작성 예정 Adapter / Decorator / Proxy / Facade / Composite / Bridge.
  • 행위 패턴 작성 예정 Strategy / Observer / Command / State / Template Method.

아키텍처 / 운영 심화

  • 클린 아키텍처 / 헥사고날 작성 예정 의존성 방향, 포트와 어댑터, 테스트 용이성.
  • DDD 핵심 개념 작성 예정 엔티티/값 객체/애그리거트/바운디드 컨텍스트.
  • 12 Factor App 작성 예정 구성, 백킹 서비스, 빌드/릴리스/실행 분리 등.
  • 마이크로서비스 분리 기준 작성 예정 서비스 경계, DB-per-service, 데이터 일관성.
