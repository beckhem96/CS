🧱 자료구조

자료구조(Data Structure)는 데이터를 효율적으로 저장하고 다루기 위한 형식과 규칙입니다. 어떤 자료구조를 쓰느냐에 따라 같은 작업이라도 속도와 메모리 사용량이 크게 달라지기 때문에, 문제 상황에 맞는 자료구조를 고르는 안목이 곧 개발자의 기본기입니다. 여기서는 가장 자주 쓰이는 자료구조들을 형태와 동작 방식을 중심으로 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 예: 해시 테이블 → 일관 해싱 / B+Tree → 페이지 분할 심층.

🧩 핵심 개념

자료구조를 처음 배우거나 면접 직전에 한 번 훑기 좋은 우선순위 개념들.

선형 구조

데이터가 일렬로 늘어선 형태. 순서가 있고, 한 방향(또는 양방향)으로 이동하면서 다룬다.

  • 배열(Array) 고정 크기 메모리에 같은 타입 데이터를 연속으로 저장. 인덱스 접근이 O(1).
  • 연결 리스트(Linked List) 각 노드가 다음 노드를 포인터로 가리키는 구조. 삽입/삭제가 빠른 대신 임의 접근이 느림.
  • 스택(Stack) LIFO(Last-In-First-Out). 함수 호출, 되돌리기, 괄호 검사 등에 쓰임.
  • 큐(Queue) FIFO(First-In-First-Out). 작업 대기열, BFS 탐색에 사용.
  • 덱(Deque) 양쪽 끝에서 모두 삽입/삭제가 가능한 큐. 슬라이딩 윈도우 기법에 활용.

해시 기반

키(key)를 해시 함수로 변환해 빠르게 위치를 찾는 구조. 평균 O(1) 조회가 강점.

  • 해시 테이블(Hash Table) 키-값 매핑. 충돌 처리(체이닝/오픈 어드레싱)와 부하율(load factor)이 핵심.

비선형 구조 — 트리

부모-자식 관계로 계층을 표현. 탐색·정렬·우선순위 처리 등 다양한 변형이 있다.

  • 이진 트리(Binary Tree) 자식이 최대 둘인 트리. 순회 방식(전위/중위/후위/레벨)이 기본기.
  • 이진 탐색 트리(BST) 왼쪽 서브트리 < 루트 < 오른쪽 서브트리. 평균 O(log n) 탐색.
  • AVL / 레드-블랙 트리 스스로 균형을 맞추는 BST. 최악의 경우에도 O(log n) 보장.
  • B-Tree / B+Tree 한 노드에 여러 키를 저장해 디스크 I/O를 줄임. DB 인덱스의 표준.
  • 힙(Heap) 완전 이진 트리 형태의 우선순위 큐. 최댓값/최솟값을 O(log n)에 꺼냄.
  • 트라이(Trie) 문자열을 트리로 저장해 자동완성·접두사 검색에 강함.

비선형 구조 — 그래프

노드(정점)와 엣지(간선)로 관계를 표현. 지도, SNS 친구 관계, 의존성 관리 등에 등장.

  • 그래프(Graph) 정점과 간선의 집합. 인접 행렬 vs 인접 리스트의 trade-off가 핵심.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 주제를 별도 페이지로 정리합니다. (작성 예정)

메모리·동시성 심화

  • 다차원 배열과 메모리 레이아웃 작성 예정 row-major vs column-major, cache locality 영향.
  • 이중 / 원형 연결 리스트 단독 작성 예정 단방향과의 차이, LRU 캐시 등 활용 사례.
  • 동시성 자료구조 작성 예정 ConcurrentHashMap 세그먼트, lock-free queue, CAS.

해시·확률·인덱스 심화

  • 일관 해싱(Consistent Hashing) 작성 예정 가상 노드, 분산 캐시·샤딩 적용.
  • 스킵 리스트(Skip List) 작성 예정 Redis sorted set 내부 구조.
  • 블룸 필터(Bloom Filter) 작성 예정 확률적 멤버십, Counting/Cuckoo 변형.
  • LSM-Tree 작성 예정 쓰기 최적화, SSTable, 컴팩션 (LevelDB/RocksDB).
