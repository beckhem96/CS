그래프(Graph)

그래프란, 데이터를 나타내는 정점(vertex/node)들과 그 사이의 관계를 나타내는 간선(edge)으로 이루어진 자료구조입니다. 트리는 그래프의 특수한 형태이고, "관계"를 표현해야 하는 거의 모든 문제는 그래프로 모델링됩니다.

구체적인 개념 설명

그래프의 가장 직관적인 비유는 지하철 노선도입니다. 역(정점)들이 점으로 표시되고, 두 역을 잇는 구간(간선)이 선으로 그어져 있죠. 어떤 역에서 어떤 역으로 갈 수 있는지, 가는 데 몇 정거장 걸리는지 같은 질문이 모두 그래프 문제입니다.

형식적으로 그래프 G는 정점 집합 V와 간선 집합 E의 쌍 G = (V, E)로 정의합니다. 그래프는 다음과 같은 여러 축을 따라 분류됩니다.

  • 방향성 — 간선에 방향이 있으면 방향 그래프(directed), 없으면 무방향 그래프(undirected). SNS 친구는 무방향, 트위터 팔로우는 방향.
  • 가중치 — 간선에 비용/거리가 붙어 있으면 가중 그래프. 도로의 거리, 네트워크 지연시간 등.
  • 사이클 — 같은 정점으로 돌아오는 경로가 있으면 사이클이 있고, 없으면 비순환(acyclic). 방향 비순환 그래프는 DAG.
  • 연결성 — 모든 두 정점 사이에 경로가 있으면 연결(connected) 그래프. 아니면 여러 연결 요소(component)로 나뉨.
  • 밀도 — 간선이 정점 수에 비해 많으면 밀집(dense), 적으면 희소(sparse). 표현 방식 선택의 기준이 됨.

트리 vs 그래프

트리는 "사이클이 없는 연결된 무방향 그래프"입니다. 정점이 N개일 때 간선이 정확히 N-1개이고, 어떤 두 정점 사이에도 경로가 정확히 하나뿐이라는 강한 제약이 추가됩니다. 그래서 트리는 그래프의 부분집합이고, 그래프 알고리즘이 트리에서도 그대로 통합니다(반대는 보통 안 됩니다).

왜 필요한가?

세상의 많은 문제가 본질적으로 "관계"입니다. 친구 관계, 도로망, 웹 페이지 사이의 링크, 회로의 연결, 작업 간 의존성, 추천 시스템의 사용자-상품 상호작용… 이런 데이터를 배열이나 트리로는 표현하기 어렵거나 부자연스럽습니다. 그래프는 이런 임의의 관계망을 자연스럽게 담아낼 수 있는 가장 일반적인 자료구조입니다.

게다가 그래프 위에는 최단 경로, 연결 요소 찾기, 위상 정렬, 사이클 탐지, 최소 신장 트리, 페이지랭크 같은 검증된 알고리즘이 풍부합니다. 한 번 그래프로 모델링하면 풀어낼 도구가 한꺼번에 따라옵니다.

어떻게 동작하는가?

1) 그래프의 모양 — 무방향 vs 방향 그래프

무방향은 양방향 통행, 방향 그래프는 정해진 방향으로만 이동 가능.

2) 표현 방법 1 — 인접 행렬(Adjacency Matrix)

정점이 N개일 때 N×N 2차원 배열을 만들어 adj[i][j] = 1이면 i에서 j로 가는 간선이 있다는 의미. 가중 그래프라면 1 대신 가중치 값을 저장합니다.

메모리는 O(V²). 정점이 많고 간선이 적으면 대부분 0으로 채워져 메모리 낭비가 심하다.

3) 표현 방법 2 — 인접 리스트(Adjacency List)

각 정점마다 자기와 연결된 정점들의 리스트를 들고 있는 방식. 실무에서 가장 많이 쓰는 표현입니다.

메모리 O(V + E). 희소 그래프(웹, SNS 등)에서 압도적으로 효율적이다.

4) 표현 방식 비교

연산인접 행렬인접 리스트
특정 간선 (u, v) 존재 확인O(1)O(deg(u))
한 정점의 모든 이웃 순회O(V)O(deg(u))
전체 간선 순회O(V²)O(V + E)
간선 추가/삭제O(1)O(deg(u))
메모리O(V²)O(V + E)

대부분의 실세계 그래프는 희소 그래프(간선이 정점 수에 가깝거나 그보다 많지 않음)이고, 알고리즘 대부분도 "이웃 순회"가 핵심이라 인접 리스트가 거의 항상 정답입니다. 행렬은 정점 수가 매우 작거나, 간선 존재 여부를 O(1)에 자주 확인해야 할 때만 고려합니다.

5) Java 코드로 보기

import java.util.*;

// 인접 리스트로 가중치 없는 무방향 그래프 표현
class Graph {
    int V;
    List<List<Integer>> adj;

    Graph(int V) {
        this.V = V;
        adj = new ArrayList<>();
        for (int i = 0; i < V; i++) adj.add(new ArrayList<>());
    }

    void addEdge(int u, int v) {
        adj.get(u).add(v);
        adj.get(v).add(u);   // 방향 그래프라면 이 줄 제거
    }
}

// 가중 그래프라면 이웃을 (정점, 가중치) 쌍으로 보관
List<List<int[]>> weighted;     // weighted.get(u) = [{v, w}, ...]

// BFS — 큐 사용
void bfs(Graph g, int start) {
    boolean[] visited = new boolean[g.V];
    Queue<Integer> q = new ArrayDeque<>();

    q.offer(start);
    visited[start] = true;

    while (!q.isEmpty()) {
        int u = q.poll();
        // u 처리
        for (int v : g.adj.get(u)) {
            if (!visited[v]) {
                visited[v] = true;
                q.offer(v);
            }
        }
    }
}

// DFS — 재귀
void dfs(Graph g, int u, boolean[] visited) {
    visited[u] = true;
    // u 처리
    for (int v : g.adj.get(u)) {
        if (!visited[v]) dfs(g, v, visited);
    }
}

6) BFS와 DFS의 차이

그래프 위에서 가장 자주 하는 작업이 탐색(traversal)입니다. 출발 정점에서 도달 가능한 정점들을 어떤 순서로 방문할 것인가의 문제이고, 두 가지가 표준입니다.

  • BFS(너비 우선 탐색) — 큐 사용. 가까운 정점부터 차례로 방문. 가중치 없는 그래프의 최단 경로를 보장.
  • DFS(깊이 우선 탐색) — 스택 또는 재귀 사용. 한 방향으로 끝까지 갔다가 막히면 되돌아옴. 사이클 탐지·위상 정렬·연결 요소 찾기 등에 활용.

시간/공간 복잡도

그래프 알고리즘의 비용은 표현 방식에 따라 크게 달라집니다 (V = 정점 수, E = 간선 수).

알고리즘시간비고
BFS / DFSO(V + E)인접 리스트 기준. 모든 정점/간선을 한 번씩
다익스트라 (가중 최단 경로)O((V+E) log V)우선순위 큐(힙) 기반
벨만-포드 (음수 가중치 허용)O(V·E)음의 사이클 검출 가능
플로이드-워셜 (모든 쌍 최단)O(V³)밀집 그래프나 V가 작을 때
위상 정렬O(V + E)DAG에서만 가능
최소 신장 트리 (크루스칼)O(E log E)유니온-파인드 사용

실무에서 어디에 쓰이는가?

  • 지도/내비게이션 — 도시 = 정점, 도로 = 간선, 거리 = 가중치. 다익스트라/A*로 최단 경로.
  • SNS / 추천 시스템 — 사람·상품 = 정점, 친구·구매 = 간선. "친구의 친구"는 BFS, 추천은 그래프 임베딩.
  • 웹 크롤링과 페이지랭크 — 웹 페이지 = 정점, 하이퍼링크 = 방향 간선. 구글이 만든 PageRank가 이 위에서 동작.
  • 의존성 관리 — 빌드 시스템, 패키지 매니저, Spark의 DAG 스케줄러 — 모두 위상 정렬 사용.
  • 네트워크 라우팅 — 라우터 = 정점, 링크 = 간선. OSPF는 다익스트라 기반.
  • 회로 설계, 컴파일러의 제어 흐름 그래프(CFG), 게임 AI의 길 찾기(A*) 등 광범위하게 활용.

면접에 나올 질문과 답변

인접 행렬과 인접 리스트 중 어느 걸 써야 하나요?
대부분 인접 리스트가 정답입니다. 실세계 그래프는 보통 희소(E ≪ V²)이고, 알고리즘 대부분이 "한 정점의 이웃을 순회"하는 패턴이라 인접 리스트의 O(deg)가 자연스럽기 때문입니다. 인접 행렬은 V가 매우 작거나(수백 이하), "두 정점이 직접 연결되어 있는지"를 O(1)에 자주 확인해야 할 때만 고려합니다.
BFS와 DFS는 각각 언제 쓰나요?
BFS는 가까운 정점부터 차례로 방문하기 때문에, 가중치가 없는 그래프에서 최단 거리를 구할 때 자연스럽게 답이 됩니다. DFS는 한 방향으로 끝까지 들어가는 성질을 활용해 사이클 탐지, 위상 정렬, 연결 요소 찾기, 강한 연결 요소(SCC)처럼 그래프의 구조를 분석할 때 잘 맞습니다. 둘 다 시간 복잡도는 O(V+E)로 같습니다.
트리와 그래프의 관계를 설명해 주세요.
트리는 그래프의 특수한 형태로, "사이클이 없는 연결된 무방향 그래프"입니다. 정점이 N개일 때 간선이 정확히 N-1개이고, 어떤 두 정점 사이의 경로가 정확히 하나뿐입니다. 이런 강한 제약 덕분에 부모-자식 관계가 자연스럽게 정의됩니다. 반대로 일반 그래프는 사이클이 있을 수 있어 부모가 여럿일 수 있고, 두 정점 사이 경로가 여러 개일 수 있습니다.
방향 그래프에서 사이클을 어떻게 검출하나요?
DFS 중에 정점의 상태를 세 가지로 나눠 추적하는 방법이 표준입니다. 아직 안 봄(WHITE), 현재 탐색 경로 위에 있음(GRAY), 탐색 끝남(BLACK). DFS 도중 GRAY인 정점으로 가는 간선을 만나면 "현재 경로로 다시 돌아왔다" → 사이클입니다. 무방향 그래프에서는 부모가 아닌 이미 방문한 정점으로 가는 간선이 사이클의 신호입니다. 혹은 유니온-파인드로 간선을 추가하면서 같은 집합 안에서 또 합치려 하는 순간을 잡아도 됩니다.
