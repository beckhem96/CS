이진 트리(Binary Tree)

이진 트리란, 각 노드가 최대 두 개의 자식 노드(왼쪽, 오른쪽)를 갖는 계층적 자료구조입니다. 트리의 가장 단순하고 가장 널리 쓰이는 형태로, 거의 모든 트리 변형의 기반입니다.

구체적인 개념 설명

트리(Tree)는 거꾸로 매달린 나무를 떠올리면 됩니다. 맨 위에 루트(root)가 있고, 거기서 가지가 뻗어 나가며 아래로 자식들이 매달립니다. 그 자식들이 또 자식을 갖고, 끝까지 가면 더 이상 자식이 없는 잎(leaf)이 나옵니다. 이 트리에서 "한 노드의 자식 수가 최대 둘"로 제한된 게 바로 이진 트리입니다.

각 노드의 구조는 보통 이렇게 생겼습니다.

class Node {
    int value;
    Node left;   // 왼쪽 자식
    Node right;  // 오른쪽 자식
}

이진 트리에서 자주 쓰이는 용어들도 한 번 정리해 둡시다.

  • 루트(root) — 가장 위의 노드. 트리의 시작점.
  • 잎(leaf) — 자식이 없는 노드.
  • 높이(height) — 루트에서 가장 깊은 잎까지의 간선 수.
  • 깊이(depth) — 루트에서 특정 노드까지의 간선 수.
  • 서브트리(subtree) — 어떤 노드와 그 후손 전체로 이루어진 트리.

특수한 이진 트리들

  • 완전 이진 트리(Complete) — 마지막 레벨을 제외하고 모두 차 있고, 마지막 레벨은 왼쪽부터 채워진 형태. 힙의 기반.
  • 포화 이진 트리(Full) — 모든 노드의 자식이 0개 또는 2개.
  • 완전 포화 이진 트리(Perfect) — 모든 잎의 깊이가 같고 내부 노드는 자식이 정확히 2개.
  • 편향 트리(Skewed) — 한쪽으로만 치우친 트리. 사실상 연결 리스트와 같아져 성능이 망가짐.

왜 필요한가?

배열이나 연결 리스트는 일렬이라 "위/아래"라는 계층 관계를 표현하기 어렵습니다. 파일 시스템의 폴더 구조, HTML DOM, 회사 조직도, 의사결정 과정처럼 부모-자식 관계가 본질인 데이터는 트리로 표현할 때 가장 자연스럽습니다.

그중에서도 이진 트리가 특별한 이유는 "이분(二分)"이라는 단순한 분기가 강력한 도구가 되기 때문입니다. 한 번의 비교로 두 갈래 중 하나를 고르는 구조는 탐색(BST), 우선순위 처리(힙), 표현식 해석, 결정 트리, 허프만 코딩 같은 곳에서 자연스럽게 등장합니다.

어떻게 동작하는가?

1) 기본 구조

다음과 같은 7개 노드짜리 이진 트리를 예시로 쓰겠습니다.

루트 1, 높이 2, 잎이 4개인 완전 포화 이진 트리.

2) 4가지 순회 방식

이진 트리를 다룰 때 가장 자주 쓰는 도구가 순회(traversal)입니다. "어떤 순서로 모든 노드를 한 번씩 방문할 것인가"의 문제죠. 크게 4가지가 있습니다.

  • 전위 순회(Preorder) — 루트 → 왼쪽 → 오른쪽 (위 트리: 1 2 4 5 3 6 7)
  • 중위 순회(Inorder) — 왼쪽 → 루트 → 오른쪽 (4 2 5 1 6 3 7)
  • 후위 순회(Postorder) — 왼쪽 → 오른쪽 → 루트 (4 5 2 6 7 3 1)
  • 레벨 순회(Level-order, BFS) — 위 레벨부터 한 줄씩 (1 2 3 4 5 6 7)
중위 순회는 BST에서 정렬된 순서로 모든 노드를 꺼낼 때 특히 유용하다.

3) Java 코드로 보기

class Node {
    int value;
    Node left, right;
    Node(int v) { this.value = v; }
}

// 전위 순회 (재귀)
void preorder(Node node) {
    if (node == null) return;
    System.out.println(node.value);    // 루트
    preorder(node.left);
    preorder(node.right);
}

// 중위 순회
void inorder(Node node) {
    if (node == null) return;
    inorder(node.left);
    System.out.println(node.value);
    inorder(node.right);
}

// 후위 순회
void postorder(Node node) {
    if (node == null) return;
    postorder(node.left);
    postorder(node.right);
    System.out.println(node.value);
}

// 레벨 순회 (BFS) — 큐 사용
void levelOrder(Node root) {
    if (root == null) return;
    Queue<Node> q = new ArrayDeque<>();
    q.offer(root);
    while (!q.isEmpty()) {
        Node n = q.poll();
        System.out.println(n.value);
        if (n.left  != null) q.offer(n.left);
        if (n.right != null) q.offer(n.right);
    }
}

시간/공간 복잡도

연산균형 트리편향 트리설명
탐색 (값으로 찾기)O(log n)O(n)일반 이진 트리는 정렬 규칙이 없어 어차피 모두 훑어야 함. BST는 O(log n) 가능
삽입 (잎에 추가)O(log n)O(n)위치를 찾는 비용이 곧 깊이
순회 (전부 방문)O(n)O(n)모든 노드를 한 번씩 방문

공간 복잡도는 노드 수 n에 비례한 O(n)이고, 재귀 순회 시 호출 스택이 트리 높이 h만큼 쌓이므로 O(h)가 추가됩니다. 균형 트리는 h ≈ log n, 편향 트리는 h ≈ n이라는 점이 핵심입니다.

실무에서 어디에 쓰이는가?

  • HTML DOM, XML 파싱 트리 — 문서 구조 자체가 트리.
  • 표현식 트리(Expression Tree) — 컴파일러/계산기에서 수식을 트리로 표현.
  • 결정 트리(Decision Tree) — 머신러닝의 분류/회귀 모델.
  • 허프만 코딩 — 데이터 압축에 쓰이는 이진 트리 기반 알고리즘.
  • BST/AVL/RB-Tree, B-Tree — 데이터베이스 인덱스나 정렬된 맵의 기반 구조.
  • 힙 (우선순위 큐) — 작업 스케줄러, 다익스트라, 이벤트 시뮬레이션의 핵심.

면접에 나올 질문과 답변

트리와 그래프의 차이를 설명해 주세요.
트리는 그래프의 특수한 형태입니다. 정점이 N개일 때 간선이 정확히 N-1개이고, 사이클이 없으며, 모든 정점이 연결되어 있습니다. 일반 그래프는 사이클이 있을 수도 있고, 간선 수에 제한이 없습니다. 그래서 트리는 부모-자식이라는 명확한 계층이 생기고, 어떤 두 노드 사이에도 경로가 정확히 하나뿐입니다.
전위/중위/후위 순회를 언제 쓰나요?
전위는 트리를 그대로 복제하거나 직렬화할 때(루트 정보가 먼저 필요해서) 적합합니다. 중위는 BST에서 정렬된 순서로 값을 꺼낼 때 자연스럽게 정렬 결과를 얻습니다. 후위는 자식의 결과를 모은 뒤 부모를 처리해야 하는 경우 — 예를 들어 트리 삭제, 표현식 트리 평가 — 에 알맞습니다. 레벨 순회는 가까운 곳부터 탐색해야 하는 BFS 계열 작업에 씁니다.
완전 이진 트리와 포화 이진 트리의 차이는요?
완전 이진 트리는 마지막 레벨을 제외하고는 모두 꽉 차 있고 마지막 레벨은 왼쪽부터 채워진 형태입니다 — 힙이 이 모양을 사용합니다. 포화 이진 트리는 모든 노드가 자식을 0개 또는 2개만 갖는 형태입니다. 둘 모두를 만족하면서 모든 잎의 깊이까지 같다면 완전 포화(perfect) 이진 트리가 됩니다.
편향 트리는 왜 문제가 되나요?
한쪽으로만 치우친 트리는 사실상 연결 리스트와 같아져 높이가 n에 가까워집니다. 그러면 탐색·삽입·삭제가 모두 O(n)이 되어 트리를 쓰는 의미가 사라집니다. 그래서 BST를 쓰면서도 균형을 보장하기 위해 AVL 트리, 레드-블랙 트리 같은 자가 균형 트리를 사용합니다.
