스택(Stack)

스택이란, 데이터를 한쪽 끝에서만 넣고 빼는 LIFO(Last-In-First-Out, 후입선출) 구조입니다. 가장 최근에 넣은 데이터가 가장 먼저 나옵니다.

구체적인 개념 설명

비유로 가장 잘 맞는 건 식당에 쌓아둔 접시 더미입니다. 새 접시는 위에 올리고, 꺼낼 때도 위에서 꺼냅니다. 아래에 있는 접시를 먼저 빼려고 하면 위에 쌓인 게 다 쏟아지죠. 그래서 늦게 올린 게 먼저 내려옵니다 — 이것이 LIFO입니다.

스택의 동작은 사실상 두 가지뿐입니다.

  • push(x) — 위에 새 데이터를 올린다.
  • pop() — 맨 위 데이터를 꺼내서 반환한다.

여기에 보조 연산으로 peek()(맨 위만 들여다봄), isEmpty(), size() 정도가 따라옵니다. "맨 위(top)"라는 위치 하나만 신경 쓰면 되기 때문에 구현이 매우 단순합니다.

스택은 추상 자료형(ADT)이다

스택은 "이런 동작이 보장되는 자료구조"라는 인터페이스이지, 그 자체가 메모리 구조는 아닙니다. 실제로는 배열로도 구현할 수 있고, 연결 리스트로도 구현할 수 있어요. Java의 ArrayDeque나 Deque를 스택처럼 쓰는 게 보통이고, 예전 java.util.Stack은 동기화 오버헤드 때문에 잘 쓰지 않습니다.

왜 필요한가?

"마지막에 한 일을 가장 먼저 되돌려야 한다"는 상황이 의외로 많기 때문입니다. 대표적인 예시들:

  • 함수 호출 — A가 B를 부르고, B가 C를 불렀다면 C가 끝나야 B가 끝나고, B가 끝나야 A가 끝남.
  • 웹 브라우저 뒤로 가기 — 가장 최근에 방문한 페이지로 먼저 돌아감.
  • 괄호/태그 매칭 — 가장 최근에 연 괄호가 가장 먼저 닫혀야 함.
  • 실행 취소(Undo) — 마지막 동작이 가장 먼저 취소됨.

이런 "가장 최근 것 먼저"라는 패턴이 보이면 자연스럽게 스택이 떠올라야 합니다.

어떻게 동작하는가?

1) push 와 pop 단계별 시각화

빈 스택에 10, 20, 30을 차례로 push한 뒤 pop을 한 번 하는 과정입니다.

top은 항상 가장 최근에 들어온 원소를 가리킨다. pop은 그 원소를 꺼내고 top을 한 칸 내린다.

2) 응용 — 괄호 짝 검사

"({[]})" 가 올바른 괄호 문자열인지 어떻게 알까요? 스택을 쓰면 한 번만 훑어도 됩니다. 여는 괄호가 나오면 push, 닫는 괄호가 나오면 top과 짝이 맞는지 확인하면서 pop. 끝났을 때 스택이 비어 있으면 OK.

스택 한 개로 짝 검사를 O(n)에 끝낼 수 있다.

3) Java 코드로 보기

import java.util.ArrayDeque;
import java.util.Deque;

// 일반적으로 권장되는 방식 (Stack 클래스보다 빠름)
Deque<Integer> stack = new ArrayDeque<>();

stack.push(10);
stack.push(20);
stack.push(30);

int top = stack.peek();   // 30 — 들여다보기만
int x   = stack.pop();    // 30 — 꺼냄
boolean empty = stack.isEmpty();

// 응용: 괄호 매칭
boolean isValid(String s) {
    Deque<Character> st = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(' || c == '{' || c == '[') {
            st.push(c);
        } else {
            if (st.isEmpty()) return false;
            char open = st.pop();
            if (c == ')' && open != '(') return false;
            if (c == '}' && open != '{') return false;
            if (c == ']' && open != '[') return false;
        }
    }
    return st.isEmpty();
}

시간/공간 복잡도

연산시간설명
pushO(1)top 위치에 쌓기
popO(1)top에서 꺼내기
peekO(1)top 들여다보기
탐색 (특정 값)O(n)스택의 본 목적이 아니어서 보통 권장되지 않음

공간 복잡도는 들어 있는 원소 수에 비례해 O(n).

실무에서 어디에 쓰이는가?

  • 함수 호출 스택(Call Stack) — JVM 등 모든 런타임이 함수 호출/반환을 관리하는 핵심 구조.
  • Undo/Redo — 에디터, 그래픽 툴의 실행 취소/재실행.
  • 브라우저 뒤로가기/앞으로가기 — 두 개의 스택을 두면 깔끔하게 구현 가능.
  • 수식 계산기 — 중위 표기식 → 후위 표기식 변환, 후위 표기식 평가에 사용.
  • DFS(깊이 우선 탐색) — 재귀를 명시적인 스택으로 풀어쓸 때.
  • JVM 스레드별 스택 메모리 — 지역 변수와 호출 정보를 담아 두는 영역.

면접에 나올 질문과 답변

스택과 큐의 차이를 설명해 주세요.
스택은 LIFO(후입선출), 큐는 FIFO(선입선출)입니다. 스택은 한쪽 끝(top)에서만 넣고 빼지만, 큐는 한쪽 끝에서 넣고(rear) 다른 쪽 끝에서 뺍니다(front). 그래서 "최근 것 먼저"가 필요하면 스택, "들어온 순서대로 처리"가 필요하면 큐를 씁니다.
재귀 함수와 스택은 어떤 관계인가요?
재귀 호출은 내부적으로 호출 스택(Call Stack)을 사용합니다. 함수가 자기 자신을 부를 때마다 새 프레임이 스택에 쌓이고, 함수가 끝나면 프레임이 pop되면서 결과가 돌아옵니다. 그래서 재귀 깊이가 너무 깊으면 StackOverflowError가 나죠. 모든 재귀는 명시적인 스택을 써서 반복문으로 바꿀 수 있다는 점도 기억해 두면 좋습니다.
Java에서 Stack 대신 ArrayDeque를 권장하는 이유는요?
java.util.Stack은 Vector를 상속해서 모든 메서드가 동기화되어 있어 단일 스레드에선 불필요한 오버헤드가 있고, 구조적으로도 낡은 설계라 권장되지 않습니다. ArrayDeque는 동기화가 없고 더 빠르며, 스택과 큐 모두로 쓸 수 있어 실무에서 표준처럼 쓰입니다.
스택을 큐로 만들 수 있나요?
가능합니다. 스택 두 개를 써서 입력용과 출력용으로 나눕니다. push는 입력 스택에 쌓고, pop은 출력 스택에서 꺼내는데, 출력 스택이 비어 있으면 입력 스택의 모든 원소를 옮겨 담습니다. 그러면 가장 오래된 원소가 출력 스택의 top이 되어 FIFO처럼 동작합니다. 분할상환으로 보면 push와 pop 모두 O(1)입니다.
