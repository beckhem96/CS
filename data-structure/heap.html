힙(Heap)

힙이란, 완전 이진 트리 모양을 유지하면서 "부모는 자식보다 항상 크다(또는 작다)"는 힙 속성(heap property)이 지켜지는 자료구조입니다. 최댓값(또는 최솟값)을 O(log n)에 꺼낼 수 있는 우선순위 큐의 표준 구현입니다.

구체적인 개념 설명

비유로는 토너먼트 대진표가 가깝습니다. 결승전 자리(루트)에는 최강자가 앉고, 그 아래로 갈수록 약한 선수들이 배치되죠. 한 라운드가 끝날 때마다 가장 강한 선수가 바로 등장(루트에서 추출)하고, 빈 자리를 다시 채워 넣으면 새 챔피언이 결정됩니다. 힙은 이걸 자료구조로 표현한 겁니다.

힙은 두 가지 변종이 있습니다.

  • 최대 힙(Max-Heap) — 부모 ≥ 모든 자식. 루트에 항상 최댓값.
  • 최소 힙(Min-Heap) — 부모 ≤ 모든 자식. 루트에 항상 최솟값.

중요한 점: 힙은 전체적으로 정렬된 게 아닙니다. 형제 노드 사이에는 어떤 순서도 없습니다. 오직 "부모-자식 관계"만 보장됩니다. 그래서 힙은 "전체 정렬"보다 비용이 훨씬 싸면서도 "최댓값/최솟값을 빠르게 꺼낸다"는 목적은 정확히 달성합니다.

힙은 보통 배열로 구현한다

완전 이진 트리는 모양이 정해져 있어 노드/포인터 없이 배열만으로 표현할 수 있습니다. 인덱스 i 노드의 부모와 자식은 다음과 같이 단순한 인덱스 계산으로 얻어집니다 (1-based 기준).

  • 부모: i / 2
  • 왼쪽 자식: 2i
  • 오른쪽 자식: 2i + 1

0-based로 하면 부모는 (i-1)/2, 자식은 2i+1, 2i+2가 됩니다.

왜 필요한가?

"데이터가 계속 들어오고 나가는 와중에 매번 가장 큰(또는 작은) 값이 필요한" 상황이 의외로 많습니다. 매번 정렬하는 건 너무 비싸고, 정렬된 배열에 삽입하면 O(n)이고, BST는 균형이 깨지면 O(n)으로 망가집니다.

힙은 이 모든 약점을 피합니다. 삽입과 최댓값 추출 모두 O(log n)이 보장되고, 구현은 배열 한 줄로 충분합니다. 메모리 효율도 좋고, 캐시 친화적이에요. 그래서 우선순위 큐의 사실상 표준 구현이 되었습니다.

어떻게 동작하는가?

1) 모양과 배열 표현

최대 힙의 예시입니다. 트리 모양으로 보든 배열로 보든 같은 데이터입니다.

완전 이진 트리는 배열 인덱스만으로 부모/자식이 정해져, 포인터 없이도 표현 가능하다.

2) 삽입 — 95를 넣으며 위로 올리기(sift-up)

새 원소는 일단 배열 맨 끝(트리에서는 마지막 자리)에 놓고, 부모와 비교하면서 더 크면 자리를 바꿉니다 (최대 힙 기준).

트리 높이만큼 비교/교환이 일어나 O(log n).

3) 추출 — 루트 꺼내고 아래로 내리기(sift-down)

루트(=최댓값)를 꺼내고, 마지막 원소를 루트로 옮긴 뒤 자식들과 비교하며 더 큰 자식과 교환을 반복합니다.

매 단계 더 큰 자식과 비교하며 내려간다 → 트리 높이만큼 O(log n).

4) Java 코드로 보기

import java.util.PriorityQueue;
import java.util.Collections;

// 최소 힙 (기본)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(50);
minHeap.offer(20);
minHeap.offer(90);

int min = minHeap.peek();   // 20
int x   = minHeap.poll();   // 20 꺼냄

// 최대 힙 — 비교자 뒤집기
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
maxHeap.offer(50);
maxHeap.offer(20);
maxHeap.offer(90);
int max = maxHeap.peek();   // 90

// 직접 구현해 본 최대 힙
class MaxHeap {
    int[] arr;
    int size;

    MaxHeap(int cap) { arr = new int[cap]; }

    void push(int v) {
        arr[size++] = v;
        int i = size - 1;
        while (i > 0 && arr[(i - 1) / 2] < arr[i]) {
            int p = (i - 1) / 2;
            int t = arr[i]; arr[i] = arr[p]; arr[p] = t;
            i = p;
        }
    }

    int pop() {
        int top = arr[0];
        arr[0] = arr[--size];
        int i = 0;
        while (true) {
            int l = 2 * i + 1, r = 2 * i + 2, big = i;
            if (l < size && arr[l] > arr[big]) big = l;
            if (r < size && arr[r] > arr[big]) big = r;
            if (big == i) break;
            int t = arr[i]; arr[i] = arr[big]; arr[big] = t;
            i = big;
        }
        return top;
    }
}

연산시간설명
최댓값/최솟값 조회 (peek)O(1)루트만 보면 됨
삽입 (push)O(log n)마지막에 놓고 sift-up
추출 (pop)O(log n)마지막을 루트로 옮기고 sift-down

n개로부터 힙 만들기 (heapify)O(n)한 번에 처리하면 O(n) — 자세한 분석은 따로

공간은 O(n). 배열 한 줄이라 메모리 오버헤드가 거의 없습니다.

실무에서 어디에 쓰이는가?

  • 우선순위 큐 — 작업 스케줄러, 이벤트 시뮬레이션, 게임의 AI 행동 평가.
  • 다익스트라(Dijkstra) 최단 경로 — "현재까지 거리가 가장 짧은 정점"을 매번 꺼낼 때.
  • 힙 정렬(Heap Sort) — O(n log n) 시간, O(1) 추가 메모리의 in-place 정렬.
  • Top-K 문제 — 스트림에서 상위 K개를 유지할 때 크기 K짜리 힙으로 O(n log K).
  • OS의 작업/타이머 관리 — 가장 가까운 만료 시각의 타이머 추출.
  • 중앙값 유지 — 최대 힙 + 최소 힙 두 개를 함께 굴려 스트리밍 중앙값을 O(log n)에 유지.

면접에 나올 질문과 답변

힙과 BST의 차이를 설명해 주세요.
둘 다 이진 트리이지만 보장하는 성질이 다릅니다. BST는 "왼쪽 < 자기 < 오른쪽"이라는 정렬 규칙을 모든 노드에서 만족해서 전체 데이터를 정렬된 순서로 다룰 수 있습니다. 힙은 부모-자식 사이의 대소 관계만 보장하고 형제 사이는 무관해서 전체 정렬은 못 하지만, 최댓값/최솟값을 O(log n)에 다루는 데 특화되어 있고 배열로 단순하게 구현됩니다.
힙은 왜 보통 배열로 구현하나요?
힙은 항상 완전 이진 트리 모양을 유지하기 때문에 트리 모양이 사실상 정해져 있고, 부모/자식 관계를 인덱스 계산 — (i-1)/2, 2i+1, 2i+2 — 로 얻을 수 있습니다. 그래서 노드/포인터 없이 배열만으로 표현 가능하고, 메모리 연속성 덕분에 캐시 효율도 좋습니다.
n개의 원소로 힙을 만들 때 O(n)이 가능하다는데 어떻게요?
한 원소씩 push하면 O(n log n)이지만, 모든 원소를 배열에 그냥 넣은 뒤 마지막 비-잎 노드부터 루트까지 sift-down을 적용하면 O(n)이 됩니다. 잎에 가까운 노드일수록 sift-down 비용이 작고, 그 노드의 수가 더 많기 때문에 전체 비용을 합하면 합이 O(n)으로 수렴합니다(엄밀한 분석은 합 ∑ h(2^h) ≤ 2n).
스트리밍 데이터에서 상위 K개만 유지하려면 어떻게 하나요?
크기 K짜리 최소 힙을 유지하면 됩니다. 원소를 받을 때마다 힙이 K보다 작으면 그냥 push, 가득 찼다면 새 값이 힙의 최솟값(루트)보다 클 때만 루트를 pop하고 새 값을 push합니다. 이렇게 하면 힙에는 항상 지금까지의 상위 K개만 남고, 전체 비용은 O(n log K)입니다. (전체 정렬은 O(n log n)인데 이게 더 가볍습니다.)
