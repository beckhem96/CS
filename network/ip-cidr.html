IP 주소 / 서브넷 / CIDR

IP 주소는 네트워크에서 장치를 식별하고 패킷을 목적지까지 보내기 위한 주소입니다. 서브넷은 큰 네트워크를 작은 네트워크로 나눈 단위이고, CIDR(Classless Inter-Domain Routing)은 192.168.1.0/24처럼 네트워크 비트 길이로 주소 범위를 표현하는 방식입니다.

구체적인 개념 설명

IP 주소는 집 주소와 비슷하지만, "어느 동네인지"와 "그 동네 안의 몇 번 집인지"가 함께 들어 있습니다. 앞쪽 일부 비트는 네트워크 부분, 뒤쪽 일부 비트는 호스트 부분입니다. 라우터는 네트워크 부분을 보고 다음 경로를 결정하고, 같은 네트워크 안에서는 호스트 부분으로 장치를 구분합니다.

IPv4는 32비트라 약 43억 개 주소만 표현합니다. 그래서 사설 IP와 NAT가 등장했고, 더 큰 주소 공간을 위해 IPv6(128비트)가 만들어졌습니다. CIDR은 예전의 A/B/C 클래스 고정 구분을 버리고, 필요한 만큼만 네트워크 크기를 정하는 유연한 표기법입니다.

/24는 "앞 24비트가 네트워크"라는 뜻

192.168.1.0/24는 앞 24비트가 네트워크 ID이고 뒤 8비트가 호스트 ID입니다. 호스트 비트가 8개면 2^8 = 256개 주소가 있고, 일반적으로 네트워크 주소와 브로드캐스트 주소를 빼 254개 호스트를 사용할 수 있습니다.

왜 필요한가?

인터넷은 거대한 하나의 평면 주소록이 아닙니다. 라우터가 수십억 개 개별 IP를 모두 외우는 건 불가능합니다. 대신 주소를 네트워크 단위로 묶으면 라우팅 테이블을 작게 유지할 수 있습니다. 192.168.1.0/24 하나만 알아도 그 안의 256개 주소를 한 번에 대표할 수 있습니다.

서브넷은 보안과 운영에도 중요합니다. 서버망, DB망, 관리망을 나누면 방화벽 정책을 다르게 적용할 수 있고, 장애 범위도 줄어듭니다. 클라우드 VPC에서 서브넷을 나누는 이유가 바로 이것입니다.

어떻게 동작하는가?

1) IPv4 32비트 구조

IPv4는 8비트씩 4개(octet)로 나누어 10진수로 표기한다.

2) 네트워크 주소, 브로드캐스트 주소, 호스트 범위

192.168.1.42/24에서 호스트 비트는 마지막 8비트입니다.

  • 네트워크 주소: 호스트 비트를 모두 0 → 192.168.1.0
  • 브로드캐스트 주소: 호스트 비트를 모두 1 → 192.168.1.255
  • 사용 가능한 호스트: 192.168.1.1 ~ 192.168.1.254

예외적으로 /31은 point-to-point 링크에서 두 주소 모두 사용 가능하고, /32는 단일 호스트 하나를 의미합니다.

3) 서브넷 분할 — /24를 /26 네 개로 나누기

/26은 32비트 중 앞 26비트가 네트워크. 남은 6비트로 64개 주소 블록이 된다.

4) 자주 쓰는 CIDR 크기

CIDR총 주소 수일반 호스트 수용도 예

5) 사설 IP 대역

인터넷에 직접 라우팅되지 않고 내부망에서만 쓰는 주소입니다. NAT와 함께 거의 모든 가정·회사·클라우드 VPC에서 사용합니다.

대역CIDR규모
10.0.0.0 ~ 10.255.255.25510.0.0.0/8대규모 사내망·클라우드
172.16.0.0 ~ 172.31.255.255172.16.0.0/12중간 규모

127.0.0.0/8Loopback자기 자신(localhost)

6) IPv6

IPv6는 128비트 주소입니다. 16비트씩 8그룹을 16진수로 표기하고, 연속된 0은 한 번만 ::로 압축할 수 있습니다.

전체 표기: 2001:0db8:0000:0000:0000:ff00:0042:8329
압축 표기: 2001:db8::ff00:42:8329

IPv6에는 브로드캐스트가 없고 멀티캐스트/애니캐스트를 사용합니다. NAT 없이도 충분한 주소 공간을 제공하는 것이 목표입니다.

7) 계산 방법

// /26 블록 크기 계산
hostBits = 32 - 26 = 6
blockSize = 2^6 = 64
subnets in last octet = 0, 64, 128, 192

// 192.168.1.130/26
130은 128~191 블록에 포함
network   = 192.168.1.128
broadcast = 192.168.1.191
hosts     = 192.168.1.129 ~ 192.168.1.190

시간/공간 복잡도 또는 trade-off

  • 작은 서브넷은 주소 낭비가 적지만 라우팅/운영 단위가 많아집니다.
  • 큰 서브넷은 운영은 단순하지만 브로드캐스트 도메인이 커지고 장애·보안 범위도 커집니다.
  • IPv4 + NAT는 주소 부족을 해결했지만 P2P·서버 공개·연결 추적을 복잡하게 만들었습니다.
  • IPv6는 주소 문제를 풀지만 운영자가 익숙하지 않고, 이중 스택 전환 비용이 있습니다.

실무에서 어디에 쓰이는가?

  • 클라우드 VPC 설계 — VPC CIDR을 넉넉히 잡고 public/private subnet을 AZ별로 나눔.
  • 방화벽 규칙 — 10.0.0.0/8 허용, 203.0.113.42/32만 접근 허용 같은 방식.
  • 쿠버네티스 — Pod CIDR, Service CIDR, Node CIDR이 겹치면 네트워크 장애 발생.
  • VPN — 회사망과 집/클라우드 사설 대역이 겹치면 라우팅 충돌.

면접에 나올 질문과 답변

192.168.1.10/24의 네트워크 주소와 사용 가능한 범위는?
/24는 앞 24비트가 네트워크라 네트워크 주소는 192.168.1.0, 브로드캐스트는 192.168.1.255입니다. 일반 호스트 범위는 192.168.1.1~192.168.1.254입니다.
CIDR은 왜 등장했나요?
기존 클래스 A/B/C 방식은 필요한 주소 수와 맞지 않아 낭비가 컸습니다. CIDR은 네트워크 비트 길이를 자유롭게 정해 주소 공간을 효율적으로 쓰고, 라우팅 테이블을 집약할 수 있게 합니다.
사설 IP는 인터넷에서 직접 접근 가능한가요?
아니요. 10/8, 172.16/12, 192.168/16은 공용 인터넷에서 라우팅되지 않습니다. 인터넷에 나가려면 NAT를 통해 공인 IP로 변환해야 합니다.
/24와 /26의 차이는?
/24는 호스트 비트 8개라 256개 주소, /26은 호스트 비트 6개라 64개 주소입니다. /26은 /24를 4개의 작은 서브넷으로 나눕니다.
IPv6에서 ::는 무엇인가요?
연속된 0 그룹을 생략한 표기입니다. 한 주소에서 한 번만 사용할 수 있습니다. 예를 들어 2001:db8::1은 중간의 여러 0 그룹이 생략된 주소입니다.
브로드캐스트 주소는 왜 호스트에 못 쓰나요?
해당 서브넷 전체 호스트에게 보내는 특수 목적 주소이기 때문입니다. 호스트 비트가 모두 1인 주소는 특정 장치가 아니라 전체 네트워크를 의미합니다.
