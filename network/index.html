🌐 네트워크

네트워크(Network)는 떨어져 있는 컴퓨터들이 약속된 규칙에 따라 데이터를 주고받는 기술입니다. 웹 페이지를 여는 짧은 순간에도 DNS·TCP·TLS·HTTP가 차곡차곡 동작합니다. 여기서는 면접·실무에서 자주 등장하는 개념을 계층(Layer) 관점에서 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 핵심 페이지마다 관련된 심화 페이지로 가는 링크가 있습니다.

🧩 핵심 개념

네트워크를 처음 배우거나 면접 직전에 한 번 훑기 좋은 우선순위 개념들.

계층 모델

네트워크는 책임을 나눠 쌓은 층 구조다. 각 계층이 어떤 일을 하는지 알면 나머지가 쉬워진다.

  • OSI 7계층 / TCP/IP 4계층 개념 모델(OSI)과 실제 모델(TCP/IP)의 매핑과 각 계층의 역할.

전송 계층(Transport)

애플리케이션과 네트워크 사이에서 신뢰성·속도·연결 관리를 담당하는 핵심 계층.

  • TCP vs UDP 신뢰성 있는 연결형(TCP)과 빠른 비연결형(UDP)의 차이와 선택 기준.
  • 3-way / 4-way handshake TCP 연결을 여는 3단계, 닫는 4단계. SYN/ACK/FIN의 의미와 TIME_WAIT.

응용 계층(Application)

사용자에게 직접 보이는 프로토콜과 보안.

  • HTTP / HTTPS 메시지 구조, 메서드, 상태 코드, 캐시, 무상태성, TLS로 만드는 HTTPS.
  • HTTP/1.1 vs 2 vs 3 keep-alive, 멀티플렉싱, HOL blocking, QUIC — 더 빠른 웹의 진화.
  • DNS 도메인 계층, 재귀/반복 질의, 레코드 종류, TTL과 캐시.
  • CORS 동일 출처 정책, preflight, 자주 헷갈리는 헤더와 credentials 동작.
  • 쿠키 / 세션 / JWT 상태 유지 방식 비교와 보안 이슈.

네트워크 계층(Network)

경로를 결정하고 사설망과 공용망을 잇는 도구.

  • IP 주소 / 서브넷 / CIDR IPv4/IPv6, 서브넷 마스크 계산, 사설 IP 대역.
  • 라우팅 / NAT 라우팅 테이블, NAT의 종류, 포트포워딩, NAT traversal.

HTTP 디테일과 데이터 링크

HTTP 페이지에서 짧게만 다룬 디테일과, 거의 다루지 않은 L2 계층을 단독으로.

  • HTTP 메서드와 상태 코드 작성 예정 GET/POST/PUT/PATCH/DELETE 의미와 멱등성, 1xx~5xx 분류와 자주 헷갈리는 코드.
  • 이더넷 / MAC / ARP 작성 예정 L2 프레임 구조, MAC 주소, ARP 동작 — IP 패킷이 실제로 같은 LAN에서 전달되는 방식.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 메커니즘을 별도 페이지로 정리합니다.

TCP 심화

TCP의 핵심 메커니즘 — 신뢰성·속도·공정성을 만들어내는 알고리즘들.

  • TCP 흐름 제어 슬라이딩 윈도우, rwnd, Zero Window, Window Scale, Nagle, Delayed ACK, Silly Window.
  • TCP 혼잡 제어 Slow Start, AIMD, Fast Retransmit/Recovery, Tahoe·Reno·Cubic·BBR, ECN.
  • TCP 재전송과 RTT 작성 예정 RTT 측정(Jacobson/Karn), RTO, SACK, F-RTO.
  • TCP 상태 머신 작성 예정 11개 상태 전이도, 동시 열기/닫기, half-close, RST.

TLS / 보안 심화

HTTPS 안쪽에서 일어나는 일과 인증서·암호화 협상의 디테일.

  • TLS 핸드셰이크 상세 작성 예정 TLS 1.2 vs 1.3 단계 비교, 인증서 체인, ECDHE, 0-RTT 보안 trade-off.

응용 계층 심화

특정 응용 프로토콜·메커니즘의 깊이 있는 동작.

  • QUIC 심화 작성 예정 UDP 위 멀티스트림, 0-RTT 재연결, 연결 마이그레이션.
  • WebSocket 작성 예정 HTTP 업그레이드, 프레이밍, ping/pong, SSE와의 차이.
  • DNS 보안과 캐시 포이즈닝 작성 예정 DNSSEC, DoH/DoT, 캐시 포이즈닝 공격과 방어.
