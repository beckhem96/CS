프로세스(Process)와 스레드(Thread)

프로세스는 실행 중인 프로그램의 독립된 자원 단위이고, 스레드는 한 프로세스 안에서 CPU를 배정받아 실행되는 흐름입니다. 프로세스는 메모리를 분리하고, 스레드는 같은 메모리를 공유합니다.

구체적인 개념 설명

프로그램은 디스크에 저장된 정적인 파일입니다. 이 프로그램을 실행하면 운영체제가 메모리를 할당하고, 실행 상태를 만들고, CPU에서 돌릴 수 있는 단위로 올립니다. 이 실행 중인 프로그램이 프로세스(Process)입니다.

스레드(Thread)는 프로세스 안의 실제 실행 흐름입니다. 한 프로세스 안에 스레드가 여러 개 있으면 같은 코드, 힙, 파일 핸들을 공유하면서 각자 스택과 레지스터 상태를 가지고 실행됩니다. 비유하면 프로세스는 식당 하나, 스레드는 그 안에서 일하는 직원들입니다.

핵심 차이는 격리와 공유입니다

프로세스끼리는 기본적으로 메모리가 분리되어 안전하지만 통신 비용이 큽니다. 스레드끼리는 메모리를 공유해 빠르게 협업할 수 있지만, 동시에 같은 데이터를 고치면 경쟁 상태(race condition)가 생길 수 있습니다.

왜 필요한가?

운영체제는 여러 프로그램을 동시에 실행해야 합니다. 브라우저, IDE, 터미널, 음악 플레이어가 함께 돌아가려면 각 프로그램을 독립적으로 관리할 단위가 필요합니다. 이 단위가 프로세스입니다. 프로세스 덕분에 한 프로그램이 망가져도 다른 프로그램의 메모리를 함부로 건드리기 어렵습니다.

반면 서버처럼 한 프로그램 안에서 많은 요청을 동시에 처리해야 할 때는 스레드가 유용합니다. 요청마다 프로세스를 새로 만들면 비용이 크지만, 스레드는 같은 프로세스의 메모리를 공유하므로 더 가볍게 동시 작업을 처리할 수 있습니다.

어떻게 동작하는가?

1) 프로세스의 메모리 구조

프로세스마다 독립된 주소 공간이 있고, 한 프로세스 안의 스레드들은 힙과 데이터를 공유한다.

2) 프로세스와 스레드 비교

구분프로세스스레드
메모리기본적으로 독립 주소 공간같은 프로세스의 힙/데이터 공유
생성/전환 비용상대적으로 큼상대적으로 작음
안정성한 프로세스 오류가 다른 프로세스에 덜 영향한 스레드 오류가 전체 프로세스에 영향 가능
통신IPC 필요공유 메모리로 빠름

3) Java 코드로 보는 스레드

// 하나의 JVM 프로세스 안에서 여러 스레드가 실행된다.
Thread worker = new Thread(() -> {
    System.out.println("작업 스레드 실행: " + Thread.currentThread().getName());
});

worker.start();
System.out.println("메인 스레드 실행: " + Thread.currentThread().getName());

// 두 스레드는 같은 프로세스의 heap 객체를 공유할 수 있다.
// 공유 객체를 동시에 수정한다면 synchronized, Lock, AtomicInteger 같은 동기화가 필요하다.

시간/공간 복잡도 또는 trade-off

선택장점trade-off
프로세스 기반격리 강함, 장애 전파 낮음생성/통신/전환 비용 큼
스레드 기반가볍고 공유 데이터 접근 빠름동기화 실수 시 race condition 발생
스레드 풀스레드 생성 비용 재사용풀 크기 설정과 큐 대기 관리 필요

실무에서 어디에 쓰이는가?

  • 웹 서버 — 요청 처리 스레드, 이벤트 루프, 워커 풀 설계에 필요합니다.
  • JVM 이해 — 하나의 Java 애플리케이션은 프로세스이고, main/gc/worker thread가 함께 실행됩니다.
  • 장애 격리 — 브라우저가 탭을 프로세스로 분리해 한 탭 장애가 전체 브라우저로 번지는 것을 줄입니다.
  • 성능 튜닝 — CPU-bound 작업과 I/O-bound 작업에 맞게 스레드 수를 조절합니다.

면접에 나올 질문과 답변

프로세스와 스레드의 가장 큰 차이는 무엇인가요?
프로세스는 독립된 주소 공간과 자원을 가진 실행 단위이고, 스레드는 프로세스 안에서 실행되는 흐름입니다. 스레드는 같은 프로세스의 힙과 데이터를 공유하지만 각자 스택과 레지스터 상태를 가집니다.
스레드가 프로세스보다 가볍다고 하는 이유는요?
스레드는 이미 존재하는 프로세스의 주소 공간과 자원을 공유하므로 새 주소 공간을 만들거나 무거운 IPC를 준비할 필요가 적습니다. 그래서 생성과 컨텍스트 스위칭 비용이 일반적으로 프로세스보다 낮습니다.
스레드 공유 메모리의 위험은 무엇인가요?
여러 스레드가 같은 객체를 동시에 수정하면 실행 순서에 따라 결과가 달라지는 경쟁 상태가 생길 수 있습니다. 임계 영역은 동기화 도구로 보호해야 합니다.
