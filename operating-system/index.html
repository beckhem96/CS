⚙️ 운영체제

운영체제(Operating System)는 프로그램과 하드웨어 사이에서 자원을 나눠주고, 보호하고, 실행 흐름을 관리하는 기본 소프트웨어입니다. 프로세스, 스레드, 스케줄링, 동기화 같은 개념을 알면 서버가 동시에 많은 요청을 처리하는 방식과 장애 원인을 더 잘 이해할 수 있습니다. 여기서는 OS를 "자원 관리자"와 "실행 관리자" 관점으로 나눠 그림과 함께 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 예: 페이징 → 페이지 교체 알고리즘 / 동기화 → IPC.

🧩 핵심 개념

OS를 처음 배우거나 면접 직전에 한 번 훑기 좋은 우선순위 개념들.

실행 단위

프로그램이 실행될 때 어떤 단위로 메모리와 CPU를 나눠 쓰는지 다룹니다.

  • 프로세스와 스레드 프로세스는 독립된 실행 공간, 스레드는 같은 프로세스 안에서 실행 흐름을 공유하는 단위.
  • 컨텍스트 스위칭(Context Switching) CPU가 실행 대상을 바꿀 때 레지스터와 실행 상태를 저장·복원하는 과정과 비용.

CPU 시간 나누기

여러 프로세스와 스레드가 CPU를 공정하고 효율적으로 쓰게 만드는 정책입니다.

  • CPU 스케줄링 FCFS, SJF, 라운드 로빈, 우선순위 스케줄링의 기준과 trade-off.

동시성 제어

여러 실행 흐름이 같은 데이터를 안전하게 다루기 위한 규칙입니다.

  • 동기화 — 뮤텍스 / 세마포어 / 모니터 임계 영역을 보호하고 경쟁 상태를 막는 기본 도구.
  • 데드락(Deadlock) 서로 자원을 기다리며 영원히 멈추는 상황과 4가지 발생 조건.

프로세스 내부 구조

OS가 프로세스 하나를 어떻게 관리하고, 그 안에 메모리를 어떻게 배치하는지 다룹니다.

  • PCB / TCB 프로세스/스레드 제어 블록 — OS가 실행 단위마다 들고 있는 자료구조의 정체.
  • 프로세스 메모리 레이아웃 code / data / heap / stack / 공유 라이브러리 영역과 각각의 성장 방향.

커널 인터페이스

응용 프로그램이 커널의 힘을 빌리는 통로 — 모든 시스템 동작의 출발점입니다.

  • 시스템 콜 / 유저 vs 커널 모드 진입 메커니즘(syscall, sysenter), 모드 전환 비용, 권한 분리.
  • IPC (프로세스 간 통신) 파이프, 메시지 큐, 공유 메모리, 시그널, 도메인 소켓.

메모리와 파일

프로그램에 메모리와 저장 공간을 어떻게 보여주고 관리하는지 다룹니다.

  • 페이징 / 세그먼테이션 가상 메모리를 일정 크기 또는 논리 단위로 나눠 관리하는 방식.
  • 파일 시스템 파일 이름, 디렉터리, 아이노드, 저널링으로 충돌에 견디는 구조.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 주제를 별도 페이지로 정리합니다.

메모리 관리 심화

  • 페이지 교체 알고리즘 FIFO, LRU, LFU, Clock — 빈 프레임이 없을 때 누구를 내보낼지.

커널·I/O 심화

  • I/O 모델 작성 예정 블로킹/논블로킹/비동기, select·poll·epoll·io_uring.

부트·격리 심화

  • 부트 프로세스 작성 예정 BIOS/UEFI → 부트로더 → 커널 → init.
  • 컨테이너 격리 메커니즘 작성 예정 네임스페이스(PID/NET/MNT), cgroup, capabilities.
