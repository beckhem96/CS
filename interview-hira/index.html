🏥 심평원(HIRA) 면접 대비

자기소개서 4개 문항을 기반으로 면접관이 던질 가능성이 높은 경험 deep dive 질문과 그 안에서 자연스럽게 묻기 좋은 연관 CS 개념 질문, 그리고 답변 가이드를 정리합니다. 각 페이지는 자소서 원문 → 예상 질문 → 답변 → 꼬리 질문 순으로 구성되어 있어, 말로 설명하는 연습용으로 그대로 읽으면서 사용할 수 있습니다.

🎯 활용 방법

  • 각 자소서 항목 페이지를 열고, 답변을 보지 않은 상태에서 질문만 보고 30초 안에 답변 흐름을 만들어 보세요.
  • 답변을 본 뒤에는 자기 답변과 비교해 비어 있는 키워드를 메모해 둡니다.
  • 연관 CS 개념 질문이 막히면 같은 페이지의 "관련 CS 노트" 링크로 이동해 다시 학습.

📝 자소서 항목별 예상 질문

1. 직무 지식·기술 습득 경험

Java/SQL/Python 역량을 어떻게 쌓았고, 입사 후 어떻게 활용할 것인가.

  • PreparedStatement Full Scan · ETL · FastAPI 실행 계획 해석, 바인드 변수 picking, ETL 단계, FastAPI 선택 이유, 대용량 처리 전략.

2. 관행을 깨고 원칙을 지킨 경험

STT 협력사 지연 → 외부 API PoC → gRPC 스트리밍 + Race Condition 해결.

  • STT API · gRPC 스트리밍 · 임계영역 제어 PoC 절차, gRPC vs REST, 4가지 스트리밍 모드, Race Condition과 동기화 도구.

3. 단점으로 인한 소통 어려움 극복 경험

Spring Batch vs Crontab 사전 협의 누락 → 의사결정 공유 습관화.

  • Spring Batch · Crontab · 의사결정 공유 두 도구의 차이, Job/Step 구조, ADR/RFC, 기술 결정 사전 합의의 중요성.

4. 효율적 방식 도입 경험

설정값 복호화 유틸리티 자체 개발 → 팀 생산성 향상.

  • 설정값 암호화 · 키 관리 · 사내 도구 배포 대칭키 vs 비대칭키, AES, Jasypt/Vault/KMS, 키 보관 표준, 보안 위험 분석.

💡 면접 전날 빠르게 다시 보기

  • 각 페이지 맨 위 "한 줄 핵심 메시지"를 외워, 답변의 첫 문장으로 활용.
  • 각 페이지 맨 아래 "실패 가능성 있는 꼬리 질문"을 한 번 더 읽고 답변 키워드 점검.
  • 본 사이트의 다음 CS 페이지를 함께 복습: 실행 계획, 인덱스, 동기화, HTTP/HTTPS, CI/CD.
