이터레이터 / 제너레이터

이터레이터는 값을 하나씩 꺼내는 표준 프로토콜이고, 제너레이터는 yield로 이터레이터를 쉽게 만드는 함수입니다. 핵심 가치는 lazy evaluation — 필요한 순간에 하나씩 계산해 메모리를 아끼는 것입니다.

구체적인 개념 설명

Python의 for문은 리스트 전용 문법이 아닙니다. 내부적으로 iter(obj)를 호출해 이터레이터를 얻고, 반복마다 next()를 호출합니다. 값이 더 없으면 StopIteration이 발생하고 for문이 종료됩니다.

제너레이터는 이터레이터를 직접 클래스와 __next__로 구현하지 않아도 되게 해줍니다. 함수 안에 yield가 있으면 일반 함수가 아니라 제너레이터 함수가 되고, 호출 시 즉시 실행되지 않고 제너레이터 객체를 반환합니다.

비유 — 도시락 한 번에 다 만들기 vs 주문마다 만들기

리스트는 모든 도시락을 한 번에 만들어 쌓아두는 방식이고, 제너레이터는 손님이 올 때마다 하나씩 만드는 방식입니다. 손님이 3명만 오면 3개만 만들면 됩니다.

왜 필요한가?

  • 메모리 절약 — 큰 파일이나 무한 시퀀스를 한 번에 만들지 않는다.
  • 파이프라인 구성 — 필터링/변환을 단계별로 연결해도 중간 리스트가 생기지 않는다.
  • 스트리밍 처리 — 로그, 네트워크, DB 커서처럼 순차 데이터를 자연스럽게 처리.
  • 추상화 — 내부 자료구조를 숨기고 순회 방식만 노출.

어떻게 동작하는가?

1) Iterator protocol

for문은 iterable이면 무엇이든 순회할 수 있다. list, dict, file, generator 모두 같은 프로토콜을 따른다.
xs = [1, 2, 3]
it = iter(xs)
print(next(it))  # 1
print(next(it))  # 2
print(next(it))  # 3
# next(it) -> StopIteration

2) 직접 만드는 이터레이터

class CountDown:
    def __init__(self, start):
        self.current = start
    def __iter__(self):
        return self
    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value

for n in CountDown(3):
    print(n)  # 3, 2, 1

3) 제너레이터 함수

yield를 만나면 함수 실행이 잠시 멈추고 값을 반환합니다. 다음 next() 호출 때 멈췄던 지점부터 다시 실행됩니다.

yield는 값을 내보내면서 실행 위치와 지역 변수를 보존한다.
def countdown(start):
    while start > 0:
        yield start
        start -= 1

for n in countdown(3):
    print(n)

4) generator expression

# list comprehension: 전체 리스트를 즉시 생성
squares_list = [x * x for x in range(1_000_000)]

# generator expression: 필요할 때 하나씩 생성
squares_gen = (x * x for x in range(1_000_000))

print(sum(x for x in range(10) if x % 2 == 0))

5) 파일 객체도 이터레이터

파일을 한 줄씩 읽을 때 readlines()로 전체를 메모리에 올리지 말고 파일 객체를 직접 순회하면 lazy하게 처리됩니다.

with open("access.log", encoding="utf-8") as f:
    for line in f:          # 한 줄씩 lazy read
        if "ERROR" in line:
            print(line)

6) itertools로 파이프라인 만들기

from itertools import islice

def numbers():
    n = 0
    while True:
        yield n
        n += 1

first_five_even = islice((n for n in numbers() if n % 2 == 0), 5)
print(list(first_five_even))  # [0, 2, 4, 6, 8]

실무에서 어디에 쓰이는가?

  • 대용량 로그 처리 — 파일을 한 줄씩 읽어 메모리 폭발 방지.
  • ETL 파이프라인 — extract → transform → filter를 lazy하게 연결.
  • API pagination — 페이지 단위로 받아 하나씩 yield.
  • 테스트 데이터 생성 — 무한/큰 시퀀스를 필요한 만큼만 생성.

면접에 나올 질문과 답변

Iterable과 Iterator의 차이는?
Iterable은 iter(obj)로 Iterator를 만들 수 있는 객체이고, Iterator는 __next__로 값을 하나씩 반환하며 StopIteration으로 종료를 알리는 객체입니다. list는 iterable이지만 iterator 자체는 아니고, iter(list)가 iterator를 만듭니다.
제너레이터는 일반 함수와 무엇이 다른가요?
제너레이터 함수는 yield를 포함하며 호출 시 즉시 실행되지 않고 generator 객체를 반환합니다. next()가 호출될 때 실행되고 yield에서 멈췄다가 다음 호출 때 이어서 실행됩니다.
generator expression과 list comprehension의 차이는?
list comprehension은 결과 리스트를 즉시 모두 만들고, generator expression은 값을 필요할 때 하나씩 만듭니다. 큰 데이터나 한 번만 순회할 데이터는 generator가 메모리 효율적입니다.
제너레이터는 재사용 가능한가요?
보통 아닙니다. generator는 한 번 소진되면 다시 처음으로 돌아가지 않습니다. 여러 번 순회해야 한다면 새 generator를 만들거나 list로 materialize해야 합니다.
StopIteration은 언제 쓰이나요?
이터레이터가 더 이상 반환할 값이 없음을 알릴 때 사용합니다. 직접 for문에서 보통 볼 일은 없지만, next()를 직접 호출하거나 iterator class를 구현할 때 중요합니다.
큰 파일을 읽을 때 왜 readlines()보다 for line이 좋나요?
readlines()는 전체 파일을 리스트로 메모리에 올립니다. 파일 객체를 직접 순회하면 한 줄씩 읽으므로 메모리 사용량이 작고, 큰 로그 파일도 안정적으로 처리할 수 있습니다.
