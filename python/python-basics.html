Python 언어 기본

Python은 인터프리터 기반의 고수준 동적 언어입니다. 소스 코드는 CPython 기준으로 bytecode로 컴파일된 뒤 Python Virtual Machine에서 실행되고, 타입은 변수명이 아니라 객체에 붙습니다. 기준 버전은 Python 3.12입니다.

구체적인 개념 설명

Python은 "읽기 쉬운 코드"와 "빠른 개발"을 목표로 설계된 언어입니다. 들여쓰기로 블록을 표현하고, 변수 선언 없이 이름에 객체를 바인딩하며, 대부분의 값이 객체입니다. 그래서 문법은 단순하지만, 이름 바인딩·객체 참조·모듈 로딩 같은 실행 모델을 이해하지 못하면 예상치 못한 버그를 만나기 쉽습니다.

가장 널리 쓰이는 구현은 CPython입니다. 우리가 터미널에서 python을 실행할 때 보통 CPython 인터프리터가 소스 코드를 파싱하고, bytecode로 컴파일하고, 가상 머신 루프에서 명령을 실행합니다.

비유 — Python 파일은 레시피, 인터프리터는 요리사

.py 파일은 요리 순서가 적힌 레시피입니다. CPython은 그 레시피를 바로 읽는 게 아니라 먼저 요리사가 보기 쉬운 중간 지시서(bytecode)로 바꾼 뒤 한 줄씩 실행합니다.

왜 필요한가?

  • 동작 예측 — 변수와 객체의 관계를 알아야 얕은 복사·mutable 기본값 같은 함정을 피한다.
  • 모듈 구조 설계 — import가 실행이라는 사실을 알아야 순환 import와 초기화 부작용을 피한다.
  • 성능 감각 — bytecode와 인터프리터 실행 모델을 알아야 왜 순수 Python 루프가 느린지 이해한다.
  • 환경 문제 해결 — 어떤 Python 실행 파일과 어떤 패키지 경로를 쓰는지 알아야 배포 문제를 줄인다.

어떻게 동작하는가?

1) 실행 흐름 — source → AST → bytecode → PVM

Python은 소스를 매번 문자 그대로 실행하지 않고, bytecode라는 중간 표현으로 컴파일한 뒤 실행한다.

2) 동적 타이핑 — 타입은 이름이 아니라 객체에 붙는다

Python의 변수는 상자가 아니라 이름표에 가깝습니다. 이름은 객체를 가리키고, 객체가 타입과 값을 가집니다.

동적 타이핑은 이름이 여러 타입의 객체를 가리킬 수 있다는 뜻이다.
x = 10          # x → int 객체
x = "hello"     # x → str 객체로 재바인딩
print(type(x))  # <class 'str'>

3) 들여쓰기와 블록

Python은 중괄호 대신 들여쓰기로 블록을 구분합니다. 같은 블록의 들여쓰기 폭은 일관되어야 합니다. PEP 8은 공백 4칸을 권장합니다.

if score >= 80:
    print("pass")
    print("good")
else:
    print("retry")

4) 모듈과 패키지

모듈은 하나의 .py 파일이고, 패키지는 모듈을 담는 디렉터리입니다. import는 단순 선언이 아니라 해당 모듈의 최상위 코드를 실행합니다. 그래서 최상위에 무거운 작업이나 외부 호출을 두면 import만 해도 부작용이 생깁니다.

# good.py
def main():
    print("run as script")

if __name__ == "__main__":
    main()  # import될 때는 실행되지 않음

5) Python답게 쓰는 기본 관용구

  • Truthiness — 빈 리스트/문자열/딕셔너리, 0, None은 False처럼 평가됩니다.
  • EAFP — "허락보다 용서가 쉽다". 검사보다 시도 후 예외 처리(try)를 선호할 때가 많습니다.
  • Duck typing — 타입 이름보다 필요한 메서드를 갖췄는지가 중요합니다.
  • PEP 8 — 이름·공백·import 순서를 맞춰 협업 비용을 줄입니다.

실무에서 어디에 쓰이는가?

면접에 나올 질문과 답변

Python은 인터프리터 언어인데 컴파일을 하나요?
합니다. CPython은 소스 코드를 AST와 bytecode로 컴파일한 뒤 Python Virtual Machine이 bytecode를 실행합니다. 다만 C/Java처럼 별도 바이너리를 만들어 배포하는 정적 컴파일 방식이 아니라 실행 시점에 내부 중간 표현으로 컴파일하는 방식입니다.
동적 타이핑이란 무엇인가요?
변수명에 고정 타입이 붙는 것이 아니라, 이름이 가리키는 객체가 타입을 갖는 방식입니다. 같은 이름이 실행 중 int 객체를 가리켰다가 str 객체를 가리킬 수 있습니다. 유연하지만 타입 오류가 런타임에 드러날 수 있습니다.
import할 때 무슨 일이 일어나나요?
모듈을 찾고, 필요하면 bytecode로 컴파일하고, 모듈 객체를 만든 뒤 최상위 코드를 실행합니다. 이후 sys.modules에 캐시되어 같은 모듈을 다시 import하면 보통 재실행하지 않습니다.
if __name__ == "__main__"은 왜 쓰나요?
파일이 직접 실행될 때만 특정 코드를 실행하고, 다른 모듈에서 import될 때는 실행되지 않게 하기 위해 씁니다. import 부작용을 줄이고 테스트 가능한 구조를 만드는 기본 관용구입니다.
Python이 느린 이유는 무엇인가요?
CPython은 bytecode를 인터프리터 루프에서 실행하고, 대부분 값이 객체라 동적 타입 확인과 참조 관리 비용이 있습니다. 순수 Python 루프는 느리지만, numpy처럼 C로 구현된 벡터화 연산이나 I/O-bound 작업에서는 충분히 빠릅니다.
Duck typing이란?
"오리처럼 걷고 꽥꽥거리면 오리로 본다"는 말처럼, 객체의 실제 클래스보다 필요한 메서드나 프로토콜을 만족하는지가 중요하다는 사고방식입니다. 예를 들어 파일처럼 read()가 있으면 실제 파일 객체가 아니어도 파일처럼 다룰 수 있습니다.
