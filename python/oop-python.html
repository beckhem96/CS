객체지향 Python

객체지향 Python은 클래스와 인스턴스를 사용해 상태와 동작을 묶는 방식입니다. Python에서는 모든 값이 객체이며, 메서드 호출은 실제로 함수에 self 인스턴스를 첫 인자로 넘기는 동작입니다. 기준 버전은 Python 3.12입니다.

구체적인 개념 설명

Python의 객체지향은 Java처럼 엄격한 접근 제어와 타입 선언을 강제하지 않습니다. 대신 duck typing, dunder method, descriptor, dataclass 같은 도구로 유연하게 객체를 모델링합니다. "private"도 문법으로 강제하기보다 언더스코어 관례로 표현합니다.

Python 클래스를 잘 이해하려면 인스턴스 속성과 클래스 속성, 그리고 함수가 메서드로 바인딩되는 과정을 알아야 합니다. 이 세 가지를 모르면 공유 mutable 클래스 속성 같은 흔한 버그를 만나게 됩니다.

비유 — 클래스는 설계도, 인스턴스는 실제 물건

클래스에 적힌 함수는 설계도 위의 작업 지침이고, 인스턴스는 실제 물건입니다. 물건이 작업 지침을 호출하면 Python이 자동으로 "이 물건(self)"을 첫 인자로 붙여 줍니다.

왜 필요한가?

  • 도메인 모델링 — 관련 상태와 동작을 한 객체에 모아 불변식을 지킴.
  • 프로토콜 활용 — __len__, __iter__ 등으로 Python 내장 문법과 통합.
  • 데이터 구조 표현 — dataclass로 값 객체를 간결하게 정의.
  • 확장성 — 상속보다 합성, duck typing으로 느슨한 결합을 유지.

어떻게 동작하는가?

1) class, instance, self

class Account:
    def __init__(self, owner: str, balance: int = 0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount: int) -> None:
        if amount <= 0:
            raise ValueError("amount must be positive")
        self.balance += amount

acc = Account("Kim")
acc.deposit(10_000)  # Account.deposit(acc, 10_000)와 비슷하게 동작

2) 메서드 바인딩

self는 키워드가 아니라 관례적 이름이다. 첫 번째 인자로 인스턴스가 들어간다는 점이 핵심이다.

3) 인스턴스 속성 vs 클래스 속성

class Team:
    members = []  # 클래스 속성: 모든 인스턴스가 공유하므로 위험

    def __init__(self, name):
        self.name = name  # 인스턴스 속성

# 좋은 패턴
class SafeTeam:
    def __init__(self, name):
        self.name = name
        self.members = []  # 인스턴스마다 새 리스트

4) dunder method — Python 문법과 객체 연결

__len__, __iter__, __eq__, __repr__ 같은 dunder method를 정의하면 내 객체가 Python 내장 문법과 자연스럽게 통합됩니다.

class Bag:
    def __init__(self, items):
        self.items = list(items)
    def __len__(self):
        return len(self.items)
    def __iter__(self):
        return iter(self.items)
    def __repr__(self):
        return f"Bag({self.items!r})"

bag = Bag([1, 2, 3])
print(len(bag))
for x in bag: print(x)

5) dataclass

값을 담는 클래스를 간단히 만들 때 dataclass가 유용합니다. __init__, __repr__, __eq__ 등을 자동으로 생성합니다.

from dataclasses import dataclass

@dataclass(frozen=True)
class Point:
    x: int
    y: int

p = Point(1, 2)
# p.x = 3  # frozen=True라 변경 불가

6) 상속보다 합성, 그리고 duck typing

Python에서는 추상 부모보다 프로토콜과 행동이 더 중요한 경우가 많다.

시간/공간 복잡도 또는 trade-off

기능장점주의점실무 감각
클래스상태+동작 캡슐화과한 추상화도메인 규칙이 있으면 적합
상속공통 구현 재사용강한 결합합성을 먼저 검토
dunder내장 문법과 통합마법처럼 보일 수 있음명확한 프로토콜에만 사용
dataclass보일러플레이트 제거행동 없는 데이터 그릇 남발값 객체에 적합

실무에서 어디에 쓰이는가?

  • 도메인 객체 — 주문, 계좌, 정책처럼 불변식이 있는 모델.
  • 값 객체 — 좌표, 기간, 금액 등 dataclass(frozen=True).
  • 어댑터 — 외부 API 응답을 내부 인터페이스에 맞춤.
  • 테스트 더블 — 필요한 메서드만 가진 fake 객체로 duck typing 활용.

면접에 나올 질문과 답변

Python의 self는 무엇인가요?
인스턴스 메서드의 첫 번째 인자로 전달되는 현재 객체입니다. 키워드는 아니고 관례적으로 self라고 부릅니다. acc.deposit(100)은 내부적으로 Account.deposit(acc, 100)처럼 self가 붙은 호출로 볼 수 있습니다.
클래스 속성과 인스턴스 속성의 차이는?
클래스 속성은 클래스 객체에 저장되어 모든 인스턴스가 공유합니다. 인스턴스 속성은 각 인스턴스의 __dict__에 저장됩니다. mutable 클래스 속성을 두면 모든 인스턴스가 같은 리스트/딕셔너리를 공유하는 버그가 생기기 쉽습니다.
dunder method는 왜 쓰나요?
내 객체가 Python의 내장 문법과 프로토콜에 참여하도록 하기 위해 씁니다. 예를 들어 __len__은 len(obj), __iter__는 for문, __enter__/__exit__는 with문과 연결됩니다.
dataclass의 장점은?
데이터를 담는 클래스에서 __init__, __repr__, __eq__ 같은 보일러플레이트를 자동 생성합니다. frozen=True를 쓰면 불변 값 객체처럼 사용할 수 있어 안전합니다.
Python에서 private 접근 제어가 있나요?
Java 같은 강제 private은 없습니다. 단일 언더스코어(_name)는 내부용이라는 관례이고, 이중 언더스코어(__name)는 name mangling으로 서브클래스 충돌을 줄입니다. 보안 장벽이라기보다 협업 신호에 가깝습니다.
duck typing이란 무엇인가요?
객체의 클래스 이름보다 필요한 행동을 제공하는지가 중요하다는 방식입니다. 파일 처리 함수가 read()만 필요하다면 실제 파일이든 StringIO든 같은 방식으로 처리할 수 있습니다.
