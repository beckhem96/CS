함수와 스코프

Python 함수는 값처럼 변수에 담고 인자로 넘기고 반환할 수 있는 first-class object입니다. 이름 탐색은 LEGB(Local → Enclosing → Global → Built-in) 규칙을 따르고, 함수는 자신이 만들어진 환경을 기억하는 클로저(closure)를 만들 수 있습니다.

구체적인 개념 설명

Python에서 함수는 단순한 코드 블록이 아니라 객체입니다. 그래서 함수 이름은 함수 객체를 가리키는 이름일 뿐이고, 다른 변수에 대입하거나 리스트에 담거나 다른 함수의 결과로 반환할 수 있습니다. 이 특징은 데코레이터, 콜백, 고차 함수의 기반입니다.

스코프는 이름을 어디서 찾을지의 규칙입니다. Python은 중괄호 블록이 아니라 함수·클래스·모듈 단위로 스코프를 만듭니다. 특히 for/if 블록은 새 스코프를 만들지 않는다는 점이 JavaScript의 let/const와 다릅니다.

비유 — 이름 찾기 순서

함수 안에서 x를 찾을 때는 먼저 내 책상(Local)을 보고, 없으면 바깥 방(Enclosing), 없으면 사무실 게시판(Global), 그래도 없으면 공용 사전(Built-in)을 봅니다. 이 순서가 LEGB입니다.

왜 필요한가?

  • 버그 예방 — LEGB와 mutable default argument를 모르면 값이 예상과 다르게 공유된다.
  • 추상화 — 함수를 값으로 다루면 전략·콜백·데코레이터를 깔끔하게 표현할 수 있다.
  • 상태 캡슐화 — 클로저로 작은 상태를 숨기고 함수 인터페이스만 노출할 수 있다.
  • 테스트 용이성 — 의존 함수를 인자로 주입하면 mock 없이도 대체 가능하다.

어떻게 동작하는가?

1) 함수는 first-class object

def greet(name):
    return f"hello, {name}"

f = greet              # 함수 객체를 다른 이름에 바인딩
print(f("Kim"))

def apply(fn, value):  # 함수를 인자로 받기
    return fn(value)

print(apply(str.upper, "python"))

2) LEGB 이름 탐색

이름 탐색과 이름 대입의 규칙은 다르다. 대입은 기본적으로 현재 Local에 새 이름을 만든다.
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"
        return x
    return inner()

print(outer())  # local

3) global과 nonlocal

함수 안에서 바깥 이름에 대입하려면 global 또는 nonlocal을 명시해야 합니다. 그렇지 않으면 Python은 그 이름을 Local로 간주합니다.

def counter():
    count = 0
    def inc():
        nonlocal count
        count += 1
        return count
    return inc

c = counter()
print(c(), c())  # 1 2

4) 클로저(Closure)

클로저는 함수가 만들어질 때의 바깥 스코프 변수를 기억하는 함수입니다. 위의 inc는 counter 호출이 끝난 뒤에도 count를 기억합니다.

클로저는 작은 상태를 함수 안에 숨기기에 좋지만, 과하면 클래스보다 흐름을 추적하기 어렵다.

5) 기본 인자는 함수 정의 시점에 한 번 평가된다

Python의 대표 함정입니다. 기본 인자로 mutable 객체를 쓰면 호출마다 새 리스트가 생기는 것이 아니라 같은 리스트를 공유합니다.

# 나쁜 예
def add_item(x, items=[]):
    items.append(x)
    return items

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2]  ← 공유됨

# 좋은 예
def add_item(x, items=None):
    if items is None:
        items = []
    items.append(x)
    return items

6) 인자 전달 — call by object sharing

Python은 흔히 "객체 참조 전달" 또는 call by object sharing이라고 설명합니다. 함수는 객체에 대한 참조를 받습니다. mutable 객체를 제자리 변경하면 호출자에게도 보이고, 이름을 재바인딩하면 호출자 이름은 바뀌지 않습니다.

def mutate(xs):
    xs.append(3)     # 객체 제자리 변경

def rebind(xs):
    xs = [9, 9]      # 로컬 이름 재바인딩

a = [1, 2]
mutate(a)
print(a)  # [1, 2, 3]
rebind(a)
print(a)  # [1, 2, 3]

시간/공간 복잡도 또는 trade-off

기능장점주의점왜 그런가?
first-class function콜백/전략 표현 쉬움흐름 추적 어려움함수가 값으로 이동
클로저상태 은닉과하면 디버깅 어려움환경 cell을 붙잡음
LEGB단순한 이름 탐색대입 시 UnboundLocalError대입은 local 선언으로 해석

실무에서 어디에 쓰이는가?

  • 콜백 — 정렬 key, 이벤트 핸들러, retry 정책.
  • 데코레이터 — 로깅, 인증, 캐싱, 트랜잭션 같은 횡단 관심사.
  • 팩토리 함수 — 설정을 캡처한 함수를 반환.
  • 테스트 — 의존 함수를 인자로 주입해 쉽게 대체.

면접에 나올 질문과 답변

LEGB 규칙이란?
Python이 이름을 찾는 순서입니다. Local(현재 함수), Enclosing(바깥 함수), Global(모듈), Built-in(내장 이름) 순서로 탐색합니다. 읽을 때는 이 순서로 찾지만, 대입은 기본적으로 Local 이름을 새로 만듭니다.
클로저란 무엇인가요?
함수가 자신이 정의된 바깥 스코프의 변수를 기억하는 구조입니다. 바깥 함수 실행이 끝나도 내부 함수가 그 변수를 계속 사용할 수 있습니다. 데코레이터와 함수 팩토리의 기반입니다.
mutable default argument 함정은 왜 생기나요?
기본 인자는 함수가 호출될 때마다 평가되는 것이 아니라 함수 정의 시점에 한 번 평가됩니다. 그래서 [] 같은 mutable 객체를 기본값으로 두면 모든 호출이 같은 리스트를 공유합니다. 보통 None을 기본값으로 두고 함수 안에서 새로 만듭니다.
global과 nonlocal의 차이는?
global은 모듈 전역 이름에 대입하겠다는 선언이고, nonlocal은 가장 가까운 바깥 함수 스코프의 이름에 대입하겠다는 선언입니다. 클래스/함수 밖의 모듈 변수에는 global, 클로저 상태 변경에는 nonlocal을 씁니다.
Python의 인자 전달 방식은?
객체 공유(call by object sharing)로 설명하는 것이 정확합니다. 함수는 객체 참조를 받고, mutable 객체를 제자리 변경하면 호출자에게 보입니다. 하지만 파라미터 이름을 다른 객체로 재바인딩해도 호출자 이름은 바뀌지 않습니다.
함수가 first-class object라는 말은?
함수를 값처럼 변수에 담고, 인자로 넘기고, 반환값으로 돌려줄 수 있다는 뜻입니다. 이 덕분에 고차 함수, 데코레이터, 콜백, 전략 주입 같은 패턴을 간결하게 표현할 수 있습니다.
