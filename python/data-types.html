자료형과 컬렉션

Python 컬렉션은 여러 값을 담는 내장 자료형입니다. 핵심은 순서 있는 list/tuple, 키-값 기반 dict, 중복 없는 set이며, 실무에서 가장 중요한 구분은 mutable(변경 가능)과 immutable(변경 불가)입니다.

구체적인 개념 설명

Python의 자료형은 모두 객체입니다. 숫자, 문자열, 리스트, 함수까지 객체이고 각 객체는 타입과 값을 가집니다. 컬렉션은 이 객체들을 참조로 담습니다. 그래서 리스트에 큰 객체를 넣는다고 객체 자체가 복사되는 것이 아니라 그 객체를 가리키는 참조가 들어갑니다.

실무에서 가장 흔한 버그는 mutable 객체를 공유한다는 사실을 잊는 데서 나옵니다. 리스트를 다른 이름에 대입하면 새 리스트가 생기는 게 아니라 같은 리스트를 두 이름이 같이 가리킵니다. 복사와 변경 가능성을 명확히 이해해야 합니다.

비유 — 리스트는 물건 상자가 아니라 주소록

리스트는 값 자체를 통째로 들고 있다기보다 객체 주소를 줄줄이 들고 있는 주소록에 가깝습니다. 주소록을 복사하지 않고 이름만 하나 더 붙이면 같은 주소록을 같이 쓰게 됩니다.

왜 필요한가?

  • 문제에 맞는 구조 선택 — 순서 보존은 list, 빠른 membership은 set/dict.
  • 성능 예측 — list 중간 삽입은 O(n), dict 조회는 평균 O(1).
  • 버그 예방 — mutable 공유, 얕은 복사, 기본 인자 함정 이해.
  • Python다운 코드 — slicing, comprehension, unpacking으로 간결한 표현.

2) mutable vs immutable

mutable 객체는 제자리 변경이 가능해서 여러 이름이 같은 객체를 공유하면 모두 영향받는다.
a = [1, 2, 3]
b = a
b.append(4)
print(a)  # [1, 2, 3, 4]

x = 10
y = x
y += 1
print(x)  # 10, int는 immutable이라 새 객체로 재바인딩

3) list — 동적 배열

Python list는 연결 리스트가 아니라 동적 배열입니다. 인덱스 접근은 O(1), 끝에 append는 amortized O(1), 중간 삽입/삭제는 요소를 밀어야 하므로 O(n)입니다.

nums = [1, 2, 3]
nums.append(4)      # 끝 추가: amortized O(1)
nums.insert(0, 0)   # 앞 삽입: O(n)
last = nums[-1]     # 음수 인덱스
part = nums[1:4]    # slicing은 새 list 생성

4) dict / set — 해시 테이블

dict와 set은 해시 테이블 기반이라 평균 조회/삽입/삭제가 O(1)입니다. key는 hash 가능해야 하므로 immutable이어야 합니다. list는 변경 가능해 hash가 없고 dict key로 쓸 수 없습니다.

dict/set의 빠른 membership은 해시 테이블 덕분이다.

5) comprehension — 컬렉션을 만드는 Python다운 표현

squares = [x * x for x in range(10) if x % 2 == 0]
index = {name: i for i, name in enumerate(["kim", "lee", "park"])}
unique_lengths = {len(word) for word in words}

comprehension은 간결하지만 너무 복잡해지면 읽기 어려워집니다. 조건과 중첩이 2개 이상이면 일반 for문이 더 낫습니다.

6) 얕은 복사와 깊은 복사

import copy

a = [[1], [2]]
b = a.copy()       # 얕은 복사: 바깥 list만 새로 만듦
b[0].append(99)
print(a)           # [[1, 99], [2]]

c = copy.deepcopy(a)  # 내부 객체까지 복사

실무에서 어디에 쓰이는가?

  • API 응답 가공 — list/dict comprehension으로 변환.
  • 캐시와 인덱스 — dict로 id → object 매핑.
  • 중복 제거 — set 사용.
  • 불변 레코드 — tuple, namedtuple, dataclass(frozen=True).

면접에 나올 질문과 답변

list와 tuple의 차이는?
list는 mutable 동적 배열이고 tuple은 immutable 순서형 컬렉션입니다. tuple은 변경할 수 없어 hash 가능한 원소만 담으면 dict key로 쓸 수 있고, 고정된 레코드나 함수 반환값 표현에 적합합니다.
dict 조회가 평균 O(1)인 이유는?
key의 hash 값을 계산해 해시 테이블의 슬롯을 바로 찾기 때문입니다. 충돌이 있으면 추가 탐색이 필요하지만, 테이블 크기를 조절해 평균적으로 O(1)을 유지합니다.
왜 list는 dict key가 될 수 없나요?
list는 mutable이라 내용이 바뀔 수 있습니다. dict key는 hash 값이 안정적이어야 하는데, list가 변경되면 hash가 바뀌어 테이블에서 찾을 수 없게 됩니다. 그래서 list는 hashable하지 않습니다.
얕은 복사와 깊은 복사의 차이는?
얕은 복사는 바깥 컨테이너만 새로 만들고 내부 객체 참조는 공유합니다. 깊은 복사는 내부 객체까지 재귀적으로 복사합니다. 중첩 리스트나 dict를 다룰 때 차이가 중요합니다.
comprehension의 장점과 단점은?
짧고 선언적으로 컬렉션을 만들 수 있어 가독성이 좋고 대체로 빠릅니다. 하지만 조건과 중첩이 많아지면 읽기 어려워져 일반 for문이 더 적합합니다.
Python dict는 순서를 보장하나요?
Python 3.7부터 삽입 순서 보장이 언어 스펙입니다. 3.6 CPython에서도 구현상 보장됐지만 공식 스펙은 3.7부터입니다. 다만 set은 논리적 순서를 기대하면 안 됩니다.
