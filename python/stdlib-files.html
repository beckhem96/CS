파일 / 표준 라이브러리

Python 표준 라이브러리는 별도 설치 없이 바로 쓰는 기본 도구 모음입니다. 실무에서 자주 쓰는 핵심은 pathlib, json, datetime, collections, itertools이며, 파일 처리는 with와 encoding을 명시하는 습관이 중요합니다.

구체적인 개념 설명

Python의 강점 중 하나는 "batteries included" — 배터리가 포함되어 있다는 철학입니다. 파일 경로, JSON, 날짜/시간, 고급 컬렉션, 반복 도구 대부분을 표준 라이브러리만으로 해결할 수 있습니다. 작은 자동화 스크립트나 백엔드 유틸리티는 외부 패키지 없이도 충분히 깔끔하게 작성할 수 있습니다.

파일 I/O에서 가장 흔한 실수는 경로를 문자열로 이어 붙이고, 인코딩을 생략하고, 파일을 닫지 않는 것입니다. pathlib.Path, with open(..., encoding="utf-8")를 기본으로 사용하면 이런 문제를 크게 줄일 수 있습니다.

비유 — 기본 공구함

표준 라이브러리는 Python이 기본으로 주는 공구함입니다. 나사를 조이려고 매번 전동 공구를 새로 살 필요가 없듯, 간단한 파일·JSON·시간 처리는 외부 패키지 없이 표준 도구부터 쓰는 게 좋습니다.

왜 필요한가?

  • 의존성 감소 — 표준 라이브러리를 쓰면 설치·보안·호환성 부담이 줄어든다.
  • 이식성 — pathlib은 OS별 경로 구분자 차이를 흡수한다.
  • 안전한 I/O — with로 파일 핸들을 자동 해제한다.
  • 생산성 — collections/itertools로 흔한 패턴을 짧고 빠르게 표현.

어떻게 동작하는가?

1) pathlib — 경로를 객체로 다루기

from pathlib import Path

root = Path("logs")
for file in root.glob("*.log"):
    text = file.read_text(encoding="utf-8")
    print(file.name, len(text))

out = Path("out") / "result.json"
out.parent.mkdir(parents=True, exist_ok=True)
out.write_text('{"ok": true}', encoding="utf-8")
Path 객체는 읽기/쓰기/검색/부모 디렉터리 생성까지 경로 관련 작업을 한곳에 모은다.

2) 안전한 파일 처리

with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:
        process(line.rstrip("\n"))
# 예외가 나도 자동 close

텍스트 파일은 encoding을 명시하세요. 운영체제 기본 인코딩에 의존하면 개발 환경에서는 되지만 서버에서 깨지는 일이 생깁니다.

3) json — 직렬화와 역직렬화

import json

payload = {"name": "Kim", "age": 20}
text = json.dumps(payload, ensure_ascii=False, indent=2)
obj = json.loads(text)

with open("user.json", "w", encoding="utf-8") as f:
    json.dump(payload, f, ensure_ascii=False)
  • dumps/loads — 문자열 ↔ Python 객체.
  • dump/load — 파일 객체 ↔ Python 객체.
  • 주의 — datetime, Decimal 같은 객체는 기본 직렬화가 안 되므로 custom encoder가 필요합니다.

4) datetime — naive vs aware

날짜/시간에서 가장 중요한 구분은 timezone이 없는 naive datetime과 timezone이 있는 aware datetime입니다. 서버·DB·API 경계에서는 timezone을 명확히 해야 합니다.

from datetime import datetime, timezone

now = datetime.now(timezone.utc)          # aware datetime
print(now.isoformat())                    # 2026-05-08T...
parsed = datetime.fromisoformat(now.isoformat())

5) collections

from collections import Counter, defaultdict, deque

cnt = Counter("banana")                 # {'a': 3, 'n': 2, 'b': 1}
groups = defaultdict(list)
for user in users:
    groups[user.team].append(user)

q = deque([1, 2, 3])
q.appendleft(0)
q.pop()
Counter/defaultdict/deque는 실무 코드의 의도를 짧게 드러낸다.

6) itertools — lazy 반복 도구

from itertools import chain, islice, groupby

for x in chain([1, 2], [3, 4]):
    print(x)

first_10 = list(islice((x*x for x in range(1_000_000)), 10))

# groupby는 인접한 같은 키끼리 묶으므로 보통 정렬 후 사용
rows.sort(key=lambda r: r.category)
for category, group in groupby(rows, key=lambda r: r.category):
    print(category, list(group))

실무에서 어디에 쓰이는가?

  • 배치 스크립트 — pathlib + json + datetime으로 파일 처리 자동화.
  • 로그 처리 — 파일 iterator, Counter, itertools 조합.
  • API 응답 — JSON 직렬화/역직렬화.
  • 큐/슬라이딩 윈도우 — deque로 양끝 삽입/삭제.

면접에 나올 질문과 답변

파일을 열 때 왜 with를 쓰나요?
예외가 발생해도 파일이 자동으로 닫히도록 보장하기 위해서입니다. with는 컨텍스트 매니저의 __exit__를 호출해 자원을 정리합니다.
pathlib의 장점은?
경로를 문자열이 아니라 객체로 다뤄 OS별 separator 차이를 줄이고, 읽기/쓰기/glob/부모 디렉터리 생성 같은 작업을 일관된 API로 제공합니다. 코드 의도도 문자열 조작보다 명확합니다.
json.dumps와 json.dump의 차이는?
dumps는 Python 객체를 JSON 문자열로 만들고, dump는 파일 객체에 바로 씁니다. 반대로 loads는 문자열에서 읽고, load는 파일 객체에서 읽습니다.
datetime의 naive와 aware 차이는?
naive datetime은 timezone 정보가 없고, aware datetime은 timezone 정보를 포함합니다. 서버와 API 경계에서는 UTC aware datetime을 사용하는 것이 안전합니다. 둘을 섞어 비교하면 오류가 나거나 의미가 불명확해집니다.
defaultdict는 언제 쓰나요?
키가 없을 때 기본값을 자동으로 만들고 싶을 때 씁니다. 예를 들어 그룹핑할 때 defaultdict(list)를 쓰면 매번 if key not in dict를 쓰지 않아도 됩니다.
itertools.groupby의 함정은?
전체에서 같은 키를 모으는 게 아니라 인접한 같은 키를 묶습니다. 그래서 일반적으로 groupby 전에 같은 key로 정렬해야 기대한 결과가 나옵니다.
