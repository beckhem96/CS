정렬 알고리즘 입문

정렬(sort)이란 데이터를 어떤 기준에 따라 오름차순/내림차순으로 재배치하는 작업입니다. 알고리즘은 십수 가지가 있지만, 핵심은 시간 복잡도·공간 복잡도·안정성·동작 방식(in-place 등)이라는 네 축을 이해하는 것입니다.

구체적인 개념 설명

정렬은 가장 단순해 보이지만 실무에서 끊임없이 등장합니다. 이진 탐색·중위 순회·구간 합 같은 알고리즘은 모두 정렬을 전제로 하고, "비슷한 데이터를 모은다"는 것만으로 풀리는 문제도 많아요. 그래서 정렬 자체보다 "이 상황에 어떤 정렬이 어울리는가"를 고를 줄 아는 게 중요합니다.

판단 기준 4가지

  • 시간 복잡도 — n이 클 때 O(n²)와 O(n log n)는 차원이 다른 차이를 만듦.
  • 공간 복잡도 — 추가 메모리가 필요한가? 제자리(in-place) 정렬은 추가 메모리가 O(1).
  • 안정성(stability) — 같은 값을 가진 원소들의 원래 순서가 정렬 후에도 유지되는가? 예: "이름순 정렬 후 다시 점수순 정렬"이 의미를 가지려면 안정 정렬이어야 함.
  • 동작 특성 — 비교 기반인가, 키 자체를 인덱스로 쓰는 비비교 정렬인가? 데이터 분포에 따라 유리한 게 달라짐.

비교 정렬은 O(n log n)가 한계다

원소를 두 개씩 비교하기만 해서는 어떤 알고리즘도 O(n log n)보다 빨라질 수 없습니다(정보 이론적 하한). 그래서 그보다 빠른 정렬은 비교 외의 정보(예: 키 값의 범위, 자릿수)를 활용하는 비비교 정렬(계수·기수·버킷 정렬)뿐입니다.

왜 필요한가?

이진 탐색은 정렬 없이는 동작하지 않고, "중복 제거"·"가장 가까운 두 원소 찾기"·"중앙값"·"구간 카운팅" 같은 문제도 정렬해두면 한 번 훑기만으로 끝납니다. 또 정렬은 자주 일어나는 작업이라 라이브러리·DB가 내장 정렬을 제공하고, 각자 다른 알고리즘(자바는 Timsort, C++ std::sort는 introsort)을 씁니다. 어떤 게 어떤 이유로 선택됐는지 아는 것 자체가 알고리즘 안목입니다.

어떻게 동작하는가?

1) 한눈에 보는 6가지 정렬

2) 삽입 정렬 — 단계별 시각화

카드 정렬과 똑같습니다. 왼쪽이 정렬된 영역, 오른쪽이 미정렬 영역. 미정렬에서 한 장씩 뽑아 정렬된 영역의 적절한 자리에 끼워넣어요. 거의 정렬된 데이터에선 매우 빠르고, 작은 배열에선 단순함 덕분에 실제로 빠릅니다.

왼쪽 구간(|의 왼쪽)은 항상 정렬된 상태로 유지된다.

3) 병합 정렬 — 분할정복

배열을 절반으로 계속 쪼개다 길이 1이 되면, 두 개씩 합치면서 정렬된 배열을 만들어 갑니다. 합치는 작업이 O(n), 단계 수가 log n이라서 전체 O(n log n). 안정 정렬이고 최악도 보장됩니다.

4) 퀵 정렬 — 피벗과 분할

피벗(pivot)을 하나 골라 그보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 모으고(파티션), 양쪽을 재귀적으로 같은 방식으로 정렬합니다. 병합 정렬과 달리 추가 메모리를 거의 안 쓰고, 평균은 더 빠르지만 피벗 선택이 나쁘면(이미 정렬된 입력에서 첫 원소를 피벗으로 고르면) 최악 O(n²)이 될 수 있습니다. 그래서 보통 임의 피벗 또는 중앙값-of-3를 사용합니다.

5) Java 코드로 보기

// 삽입 정렬
void insertionSort(int[] a) {
    for (int i = 1; i < a.length; i++) {
        int key = a[i], j = i - 1;
        while (j >= 0 && a[j] > key) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = key;
    }
}

// 병합 정렬
void mergeSort(int[] a, int lo, int hi) {
    if (lo >= hi) return;
    int mid = (lo + hi) / 2;
    mergeSort(a, lo, mid);
    mergeSort(a, mid + 1, hi);
    merge(a, lo, mid, hi);
}

void merge(int[] a, int lo, int mid, int hi) {
    int[] tmp = new int[hi - lo + 1];
    int i = lo, j = mid + 1, k = 0;
    while (i <= mid && j <= hi) {
        tmp[k++] = (a[i] <= a[j]) ? a[i++] : a[j++];
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= hi)  tmp[k++] = a[j++];
    System.arraycopy(tmp, 0, a, lo, tmp.length);
}

// 퀵 정렬 (Lomuto 파티션)
void quickSort(int[] a, int lo, int hi) {
    if (lo >= hi) return;
    int p = partition(a, lo, hi);
    quickSort(a, lo, p - 1);
    quickSort(a, p + 1, hi);
}

int partition(int[] a, int lo, int hi) {
    int pivot = a[hi];     // 마지막 원소를 피벗으로
    int i = lo - 1;
    for (int j = lo; j < hi; j++) {
        if (a[j] <= pivot) {
            i++;
            int t = a[i]; a[i] = a[j]; a[j] = t;
        }
    }
    int t = a[i + 1]; a[i + 1] = a[hi]; a[hi] = t;
    return i + 1;
}

// 표준 라이브러리
import java.util.Arrays;
Arrays.sort(intArr);                   // 원시 타입: dual-pivot quicksort
Arrays.sort(objArr);                   // 객체: Timsort (병합+삽입 하이브리드)

시간/공간 복잡도

위 표 참고. 한 번 더 짚자면:

  • O(n²) 그룹 (버블/선택/삽입) — 작은 배열, 거의 정렬된 입력, 교육용에서만 의미가 있음.
  • O(n log n) 그룹 (병합/퀵/힙) — 실무에서 거의 모든 경우 이 셋 중 하나 또는 그 변형이 쓰임.
  • 비비교 정렬 (계수/기수/버킷) — 키의 범위가 작거나 정수일 때 O(n)에 가능. 키 분포가 핵심 가정.

실무에서 어디에 쓰이는가?

  • 표준 라이브러리 — Java Arrays.sort(Object[]) = Timsort, C++ std::sort = introsort, Python sorted = Timsort.
  • 데이터베이스 — ORDER BY, GROUP BY, DISTINCT 처리. 큰 데이터엔 디스크 기반 외부 병합 정렬.
  • 이진 탐색의 전제조건 — 정렬 없이는 이진 탐색을 적용할 수 없음.
  • 중복 제거 / 그룹핑 / 통계 — 정렬 후 한 번 훑으면 끝.
  • 지도/내비게이션의 우선순위 처리 — 힙 정렬과 같은 원리(우선순위 큐)로 자주 등장.

면접에 나올 질문과 답변

"안정 정렬"이란 무엇이고 왜 중요한가요?
안정 정렬은 같은 키를 가진 원소들이 정렬 전후로 같은 상대 순서를 유지하는 것입니다. 실무에선 다중 키 정렬에 결정적이에요. 예를 들어 "이름순 정렬 후 점수순 정렬"을 했을 때, 점수가 같은 학생들이 이름 알파벳 순으로 남아 있으려면 두 번째 정렬이 안정 정렬이어야 합니다. 병합·삽입·버블은 안정, 퀵·힙·선택은 일반적으로 불안정입니다.
퀵 정렬의 평균은 빠른데 최악이 O(n²)인 이유는요?
퀵 정렬은 피벗 기준으로 양쪽으로 나누는데, 매번 한쪽에 0개·반대쪽에 n-1개가 몰리는 극단적인 분할이 일어나면 재귀 깊이가 n이 되어 전체가 O(n²)이 됩니다. 정렬된 입력에서 첫/마지막 원소를 피벗으로 고를 때 발생합니다. 해결책은 임의 피벗이나 중앙값-of-3를 쓰는 것이고, 더 견고하게는 일정 깊이를 넘으면 힙 정렬로 전환하는 introsort가 있습니다(C++ std::sort가 그 방식).
병합 정렬과 퀵 정렬 중 어느 걸 써야 하나요?
퀵 정렬은 추가 메모리가 거의 필요 없고 평균이 빨라 메모리에서 정렬할 때 유리합니다. 병합 정렬은 추가 O(n) 메모리가 필요하지만 안정 정렬이고 최악도 O(n log n)이라 신뢰성이 좋습니다. 그래서 자바는 객체 정렬에 안정성이 필요해 Timsort(병합 기반)를, 원시 타입엔 빠른 dual-pivot 퀵소트를 씁니다. 디스크처럼 데이터가 크고 순차 접근이 유리한 환경에선 외부 병합 정렬이 표준이에요.
계수 정렬(Counting Sort)이 O(n)에 가능한 이유는요?
비교를 안 하기 때문입니다. 키의 범위가 0~k 사이의 정수라고 가정하고, "각 값이 몇 번 나오는지"를 세어 그걸 누적해서 직접 위치를 계산합니다. 비교 정렬의 O(n log n) 하한은 비교만으로 정렬할 때 얘기이고, 키의 범위(k)가 작다고 알고 있으면 O(n + k)로 가능해집니다. 다만 k가 n보다 훨씬 크면 의미가 없고, 부동소수점이나 임의 객체에는 그대로 쓸 수 없는 한계도 있습니다.
