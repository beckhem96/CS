비트 조작 / 비트마스킹 (Bit Manipulation / Bitmasking)

비트 조작은 정수를 0과 1의 시퀀스로 보고 AND·OR·XOR·Shift 같은 연산으로 직접 다루는 기법입니다. 비트마스킹은 이 기법을 활용해 N개 원소의 부분집합이나 여러 boolean 플래그를 정수 하나로 표현하는 패턴입니다. 탐색·DP·자료구조에서 시간과 공간을 동시에 줄이는 가장 강력한 도구 중 하나입니다.

구체적인 개념 설명

컴퓨터에서 정수는 결국 비트 배열입니다. 64비트 정수 하나는 0/1 64개를 묶어둔 것이고, "x & (1 << k)가 0이 아니다"는 "x의 k번 비트가 켜져 있다"와 정확히 같은 의미입니다. 이렇게 정수와 boolean 배열을 자유롭게 오가는 시각이 비트마스킹의 출발입니다.

"N≤20인 부분집합 모두 시도하기"는 2^20 ≈ 100만개를 도는 일이고, 각 부분집합을 정수 하나로 표현하면 메모리·캐시·해시가 매우 가벼워집니다. 비트 DP, 외판원 문제(TSP), 권한 플래그, 색상 비트필드 같은 영역이 모두 이 시각 위에서 동작합니다.

핵심 매핑 — 비트 ↔ 부분집합 ↔ 플래그

  • k번 비트 = "원소 k가 들어 있는가"
  • 비트마스크 정수 m = "어떤 부분집합인가"
  • OR/AND/XOR = 합집합 / 교집합 / 대칭차
  • popcount(m) = 부분집합의 크기

왜 필요한가?

  • 속도 — 비트 연산은 CPU 1~2 사이클이면 끝납니다. 같은 일을 boolean 배열로 하면 캐시·메모리 비용이 더 큽니다.
  • 공간 — 64개 원소의 boolean 집합이 단 8바이트(64비트 정수 하나)에 들어갑니다.
  • 비트 DP의 상태 — "지금까지 방문한 도시 집합" 같은 상태를 그대로 정수로 만들어 DP 테이블의 인덱스로 쓸 수 있습니다.
  • 저수준 시스템 — 권한 플래그(O_RDONLY|O_CREAT), 색상 비트필드, 프로토콜 헤더가 모두 비트 단위로 정의됩니다.

어떻게 동작하는가?

1) 비트 연산자 기본

AND·OR·XOR·NOT 진리표와 시프트의 의미. XOR의 두 가지 항등식이 비트 트릭의 핵심.

2) 자주 쓰는 비트 트릭 6가지

목적식설명
k번 비트 켜기x | (1 << k)해당 자리만 1, 나머지 0인 마스크와 OR
k번 비트 끄기x & ~(1 << k)해당 자리만 0인 마스크와 AND
k번 비트 토글x ^ (1 << k)XOR로 0↔1 뒤집기
k번 비트 검사(x >> k) & 1k번을 LSB로 옮긴 뒤 AND 1
가장 낮은 1비트만 추출x & -x"isolate lowest set bit" — Fenwick tree의 핵심
가장 낮은 1비트 끄기x & (x - 1)popcount 루프, "x가 2의 거듭제곱인가" 검사에 활용
 
# Python — 자주 쓰는 트릭
def set_bit(x, k):     return x | (1 << k)
def clear_bit(x, k):   return x & ~(1 << k)
def toggle_bit(x, k):  return x ^ (1 << k)
def get_bit(x, k):     return (x >> k) & 1

# x가 2의 거듭제곱인가? (단, x > 0)
def is_power_of_two(x):
    return x > 0 and (x & (x - 1)) == 0

3) 비트마스크로 부분집합 표현

원소가 N개인 집합의 부분집합은 총 2^N개입니다. 부분집합 하나를 N비트 정수 하나로 표현하면, 모든 부분집합을 for m in range(1 << N)으로 순회할 수 있습니다.

k번 비트가 1이면 "원소 k가 부분집합에 포함" — 정수와 집합이 1:1로 대응된다. 
# 모든 부분집합 순회와 합집합/교집합
N = 3
items = ['a', 'b', 'c']

for m in range(1 << N):                # 0 .. 2^N - 1
    subset = [items[k] for k in range(N) if (m >> k) & 1]
    print(bin(m), subset)

A = 0b1010   # {b, d}
B = 0b1100   # {c, d}
print(bin(A | B))   # 합집합  → 0b1110
print(bin(A & B))   # 교집합  → 0b1000
print(bin(A ^ B))   # 대칭차  → 0b0110

4) popcount — 켜진 비트 수 세기

"이 부분집합의 크기는?" "두 사용자가 공통으로 가진 권한이 몇 개?" 같은 질문이 모두 popcount로 풀립니다. Python은 bin(x).count('1') 또는 x.bit_count()(3.10+)를, C/C++은 __builtin_popcount를 제공합니다. 직접 구현은 Brian Kernighan의 알고리즘이 우아합니다 — 가장 낮은 1비트를 끄는 횟수가 곧 켜진 비트 수입니다. 

def popcount(x):
    cnt = 0
    while x:
        x &= x - 1     # 가장 낮은 1비트 하나 제거
        cnt += 1
    return cnt

# Python 3.10+
print((0b10110101).bit_count())   # 5

5) 비트 DP — TSP를 예시로

외판원 문제(TSP)는 N개 도시를 모두 한 번씩 방문하는 최단 경로를 찾는 문제입니다. "지금까지 방문한 도시 집합"이라는 상태를 표현해야 하는데, 이때 비트마스크를 쓰면 상태가 정수 하나로 표현되어 DP 테이블이 dp[mask][last] 형태가 됩니다 — 시간 O(2^N · N²), 공간 O(2^N · N)으로 N≈20까지 가능합니다. 

INF = float('inf')

def tsp(dist):
    N = len(dist)
    dp = [[INF] * N for _ in range(1 << N)]
    dp[1][0] = 0       # 0번 도시에서 시작, 방문집합 = {0}

    for mask in range(1 << N):
        for last in range(N):
            if dp[mask][last] == INF:
                continue
            for nxt in range(N):
                if mask & (1 << nxt):     # 이미 방문
                    continue
                nmask = mask | (1 << nxt)
                cand = dp[mask][last] + dist[last][nxt]
                if cand < dp[nmask][nxt]:
                    dp[nmask][nxt] = cand

    full = (1 << N) - 1
    return min(dp[full][last] + dist[last][0] for last in range(N))

6) XOR 트릭 — 한 번 등장한 수 찾기 / 변수 swap

XOR의 두 가지 성질, a ^ a = 0과 a ^ 0 = a는 강력한 트릭의 원천입니다. "정수 배열에서 모든 원소가 두 번씩 나오는데 단 하나만 한 번 등장한다"는 문제는 전체를 XOR하면 답이 나옵니다 — 두 번 나오는 값들은 자기 자신과 XOR되어 0으로 사라지고 하나만 살아남습니다. 

# 한 번만 등장하는 수
def find_unique(arr):
    r = 0
    for x in arr:
        r ^= x
    return r

print(find_unique([2, 3, 4, 3, 2]))   # 4

# 임시 변수 없이 swap (학술적 — 실무에선 그냥 a, b = b, a)
a, b = 7, 11
a = a ^ b
b = a ^ b   # = a
a = a ^ b   # = b
print(a, b)   # 11 7

trade-off

설계 결정장점비용
비트마스크로 집합 표현매우 작은 메모리, 빠른 합집합/교집합N이 비트 폭(보통 64) 이상이면 적용 불가
비트 DP로 부분집합 순회O(2^N · 폴리(N)) 풀이가 가능N≈20을 넘기면 사실상 불가
비트 트릭 적극 사용코드가 짧고 캐시 친화적가독성 ↓, 잘못된 마스크/시프트는 디버깅 어려움
플래그를 비트필드로한 정수에 여러 boolean 압축의미 잃기 쉬움 — 상수·enum으로 명명 필수
signed shift 사용나눗셈을 빠르게 대체음수 시프트 동작은 언어/컴파일러마다 다름 — 주의

실무에서 어디에 쓰이는가?

  • 권한 / 옵션 플래그 — open(path, O_RDWR | O_CREAT | O_TRUNC)처럼 OR로 옵션을 합치는 패턴이 어디에나 있습니다.
  • 색상 / 픽셀 — RGBA 32비트, 마스크로 채널 추출 ((rgba >> 16) & 0xFF로 R 추출).
  • Bloom filter / Bitset — 큰 boolean 배열을 비트로 압축해 캐시 효율을 극대화.
  • Fenwick Tree (BIT) — 인덱스 갱신이 i &= i - 1, i += i & -i 같은 비트 트릭으로 동작.
  • 외판원·집합 분할 같은 NP 문제 (소규모) — N≤20 환경에서 비트 DP가 표준 도구.
  • 네트워크 / 프로토콜 헤더 — TCP 플래그(SYN/ACK/FIN/RST), IP 헤더의 옵션, MAC의 유니/멀티캐스트 비트.
  • 해시 / 체크섬 / 암호 — XOR 기반 mixing, MurmurHash, SipHash, CRC가 모두 비트 연산 위에서 동작.

면접에 나올 질문과 답변

x가 2의 거듭제곱인지 어떻게 한 줄로 판단하나요?
x > 0이면서 x & (x - 1) == 0이면 2의 거듭제곱입니다. 2의 거듭제곱은 비트 표현이 정확히 한 자리만 1이고, x - 1은 그 자리부터 아래가 모두 1로 바뀌므로 AND 결과가 0이 됩니다. 예를 들어 8(1000)과 7(0111)을 AND하면 0입니다.
N개 원소의 부분집합을 비트마스크로 표현하면 어떤 점이 좋은가요?
부분집합 하나가 N비트 정수 하나로 표현되어 메모리·해시·캐시가 가벼워지고, 합집합·교집합·대칭차가 OR/AND/XOR 한 번이면 끝납니다. 모든 부분집합을 순회하는 것도 for m in range(1 << N)으로 단순합니다. 다만 N이 비트 폭(보통 64)을 넘으면 정수 하나로는 불가능해 일반 배열로 돌아가야 합니다.
XOR의 어떤 성질이 알고리즘에서 자주 쓰이나요?
두 가지가 핵심입니다 — a ^ a = 0(자기 자신과 XOR하면 0)과 a ^ 0 = a(0과 XOR하면 자기 자신). 이 두 성질 덕분에 "두 번 등장하는 값은 사라지고 한 번 등장하는 값만 남는" 트릭이 가능하고, 한 번에 모든 값을 XOR하면 짝이 맞지 않는 유일한 값을 O(N) 시간 O(1) 공간으로 찾을 수 있습니다.
비트 DP로 풀 수 있는 문제와 한계는 무엇인가요?
"지금까지 방문한 집합"이나 "지금까지 사용한 자원" 같은 부분집합 상태를 정수로 표현해야 하는 문제에 적합합니다. TSP, 비트마스크 할당, 메뉴/조합 최적화 같은 문제들이 대표적입니다. 한계는 N이 작아야 한다는 점으로, 시간 O(2^N · N) 또는 O(2^N · N²) 풀이가 일반적이라 N≈20이 사실상의 상한입니다.
popcount(켜진 비트 수)를 효율적으로 구하는 방법은 무엇인가요?
많은 언어가 빌트인을 제공하고(C/C++의 __builtin_popcount, Python 3.10+의 int.bit_count()), 직접 구현 시 Brian Kernighan의 알고리즘이 자연스럽습니다 — x &= x - 1로 가장 낮은 1비트를 한 번에 하나씩 끄면서 횟수를 세면, 켜진 비트 수만큼만 루프를 돕니다. 시간 복잡도는 켜진 비트 수에 비례해 O(popcount(x))입니다.
"가장 낮은 1비트를 추출"하는 식 x & -x는 어떻게 동작하나요?
2의 보수에서 -x는 ~x + 1과 같습니다. 그래서 -x는 가장 낮은 1비트는 그대로 유지되고, 그보다 높은 비트들은 모두 반전됩니다. AND를 하면 가장 낮은 1비트만 살아남습니다. 예: x = 0b10110100이면 x & -x = 0b00000100. 이 트릭은 Fenwick Tree(BIT)의 인덱스 갱신·구간 합 계산의 핵심입니다.
