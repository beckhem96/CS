백트래킹(Backtracking)

백트래킹이란, 가능한 선택을 하나씩 시도하다가 더 이상 답이 될 수 없다고 판단되면 즉시 되돌아가는 완전 탐색 기법입니다. 핵심은 모든 경우를 보되, 불가능한 가지는 일찍 잘라내는 가지치기(pruning)입니다.

구체적인 개념 설명

백트래킹은 미로 탐험과 같습니다. 갈림길에서 한 방향을 골라 들어가고, 막다른 길이면 마지막 갈림길로 돌아와 다른 방향을 시도합니다. 모든 길을 다 가보긴 하지만, 이미 막힌 길은 더 깊게 들어가지 않습니다.

일반적인 완전 탐색은 가능한 모든 경우를 끝까지 만들어 본 뒤 검사합니다. 백트래킹은 다릅니다. 중간 상태에서 이미 조건을 위반했다면, 그 아래로 이어지는 수많은 경우를 만들 필요 없이 바로 중단합니다. 그래서 최악 시간은 여전히 지수적일 수 있지만, 실제로는 가지치기 덕분에 훨씬 빨라지는 경우가 많습니다.

백트래킹의 기본 틀

  1. 현재 선택 상태를 만든다.
  2. 현재 상태가 유효하지 않으면 즉시 돌아간다.
  3. 정답 조건을 만족하면 기록한다.
  4. 다음 후보를 하나 선택하고 재귀 호출한다.
  5. 재귀가 끝나면 선택을 취소하고 다음 후보를 시도한다.

왜 필요한가?

순열, 조합, 부분집합, N-Queens, 스도쿠, 그래프 색칠, 단어 퍼즐처럼 "가능한 배치"를 찾아야 하는 문제는 경우의 수가 폭발합니다. 하지만 대부분 제약 조건이 있습니다. 예를 들어 N-Queens는 같은 열이나 대각선에 퀸이 있으면 그 순간 실패가 확정됩니다. 백트래킹은 이런 제약을 최대한 빨리 적용해 탐색 공간을 줄입니다.

DP가 "중복 부분 문제를 저장"하는 방식이라면, 백트래킹은 "불가능한 미래를 미리 버리는" 방식입니다. 둘 다 지수적 탐색을 줄이는 도구지만 접근 방식이 다릅니다.

어떻게 동작하는가?

1) 선택 트리와 가지치기

집합 {1, 2, 3}에서 길이 2짜리 순열을 만들되, 같은 숫자는 두 번 쓰지 않는다고 해봅시다.

백트래킹은 선택 트리를 DFS로 탐색하면서 불가능한 가지를 잘라낸다.

2) N-Queens 예시

N×N 체스판에 퀸 N개를 서로 공격하지 않도록 배치하는 문제입니다. 한 행에 퀸을 하나씩 놓는다고 정하면, 각 행에서 가능한 열을 시도하고, 같은 열/대각선에 이미 퀸이 있으면 즉시 가지치기합니다.

각 행에 하나씩 놓되, 열과 두 대각선 충돌을 검사하며 진행한다.

3) Java 코드로 보기

import java.util.*;

// 순열 생성: nums 중 r개 선택
void permutations(int[] nums, int r) {
    boolean[] used = new boolean[nums.length];
    List<Integer> path = new ArrayList<>();
    backtrackPerm(nums, r, used, path);
}

void backtrackPerm(int[] nums, int r, boolean[] used, List<Integer> path) {
    if (path.size() == r) {
        System.out.println(path);
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;       // 가지치기: 이미 사용
        used[i] = true;
        path.add(nums[i]);

        backtrackPerm(nums, r, used, path);

        path.remove(path.size() - 1); // 선택 취소
        used[i] = false;
    }
}

// N-Queens: 해의 개수 세기
int solveNQueens(int n) {
    boolean[] col = new boolean[n];
    boolean[] diag1 = new boolean[2 * n];     // row - col + n
    boolean[] diag2 = new boolean[2 * n];     // row + col
    return place(0, n, col, diag1, diag2);
}

int place(int row, int n, boolean[] col, boolean[] diag1, boolean[] diag2) {
    if (row == n) return 1;
    int count = 0;
    for (int c = 0; c < n; c++) {
        int d1 = row - c + n;
        int d2 = row + c;
        if (col[c] || diag1[d1] || diag2[d2]) continue;

        col[c] = diag1[d1] = diag2[d2] = true;
        count += place(row + 1, n, col, diag1, diag2);
        col[c] = diag1[d1] = diag2[d2] = false;
    }
    return count;
}

백트래킹은 최악 시간만 보면 지수/팩토리얼이라 무섭지만, 가지치기 조건이 강할수록 실제 탐색량이 급격히 줄어듭니다. 그래서 문제의 핵심은 "어떤 조건으로 얼마나 빨리 잘라낼 수 있는가"입니다.

실무에서 어디에 쓰이는가?

  • 제약 만족 문제(CSP) — 스도쿠, 스케줄링, 좌석 배치, 배차 문제.
  • 퍼즐/게임 AI — 체스 엔진의 탐색, 단어 퍼즐, 미로 생성·해결.
  • 컴파일러/형식 검증 — 가능한 상태 공간을 탐색하며 조건 위반 검출.
  • 설정 조합 탐색 — 제한 조건이 있는 옵션 조합 생성.
  • 테스트 케이스 생성 — 모든 가능한 입력 조합 중 유효한 케이스만 생성.

면접에 나올 질문과 답변

백트래킹과 DFS의 관계는 무엇인가요?
백트래킹은 DFS의 응용입니다. 선택 트리를 DFS로 내려가며 탐색하되, 현재 상태가 조건을 만족하지 못하면 그 아래 서브트리를 탐색하지 않고 돌아옵니다. 즉 DFS + 가지치기 + 선택 취소 패턴이 백트래킹입니다.
백트래킹과 완전 탐색의 차이는요?
완전 탐색은 가능한 모든 경우를 끝까지 생성한 뒤 검사하는 경향이 있습니다. 백트래킹은 중간 상태에서 이미 불가능하다고 판단되면 그 아래의 모든 경우를 만들지 않고 중단합니다. 그래서 최악 시간은 같아도 실제 수행 시간은 가지치기 정도에 따라 크게 줄어듭니다.
백트래킹 코드에서 선택 취소가 중요한 이유는요?
같은 path, used 같은 상태를 재귀 호출 사이에서 공유하기 때문입니다. 한 후보를 선택해 재귀를 내려갔다가 돌아오면, 그 선택을 지워야 다음 후보가 깨끗한 상태에서 시작할 수 있습니다. 선택 취소를 빼먹으면 이전 가지의 상태가 다음 가지에 섞여 잘못된 결과가 나옵니다.
좋은 가지치기 조건은 어떻게 찾나요?
"현재 상태에서 앞으로 아무리 잘해도 답이 될 수 없는가?"를 묻는 게 출발점입니다. N-Queens라면 같은 열/대각선 충돌이 즉시 실패 조건이고, 조합 합 문제라면 현재 합이 목표를 넘으면 중단할 수 있습니다. 정렬을 먼저 해두면 더 강한 가지치기(남은 값이 모두 커서 불가능 등)를 만들 수 있는 경우도 많습니다.
