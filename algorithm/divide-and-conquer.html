분할 정복(Divide & Conquer)

분할 정복이란, 큰 문제를 서로 비슷한 작은 문제들로 나누고(Divide), 작은 문제를 각각 해결한 뒤(Solve), 그 결과를 합쳐서(Combine) 전체 문제를 푸는 알고리즘 설계 기법입니다. 병합 정렬, 퀵 정렬, 이진 탐색, 큰 수 곱셈, FFT 같은 알고리즘의 핵심 사고방식입니다.

구체적인 개념 설명

분할 정복은 이름 그대로 쪼개서 이긴다는 전략입니다. 한 번에 풀기 어려운 큰 문제를 절반 또는 여러 조각으로 나누고, 각 조각을 같은 방식으로 재귀적으로 풉니다. 조각이 충분히 작아지면 바로 답을 구하고, 그 답들을 합쳐 전체 답을 만듭니다.

항상 세 단계로 생각하면 됩니다.

  1. Divide — 문제를 작은 하위 문제로 나눈다.
  2. Conquer — 하위 문제를 재귀적으로 해결한다.
  3. Combine — 하위 문제의 답을 합쳐 원래 문제의 답을 만든다.

DP와 뭐가 다를까?

분할 정복의 하위 문제들은 보통 서로 독립입니다. 병합 정렬의 왼쪽 절반과 오른쪽 절반은 겹치지 않죠. 반면 DP는 같은 하위 문제가 반복해서 등장합니다. 중복이 크면 DP, 독립적인 분할이면 분할 정복을 먼저 떠올리면 됩니다.

왜 필요한가?

큰 문제를 직접 풀면 O(n²)처럼 느릴 수 있지만, 적절히 나누면 O(n log n) 또는 그보다 더 빠르게 만들 수 있습니다. 정렬의 대표 예가 병합 정렬과 퀵 정렬입니다. 단순 정렬은 O(n²)이지만, 분할 정복 정렬은 평균 또는 최악 O(n log n)을 달성합니다.

또 분할 정복은 병렬 처리와도 잘 맞습니다. 하위 문제가 독립적이면 서로 다른 스레드나 머신에서 동시에 계산하고 마지막에 합치면 됩니다. MapReduce, 병렬 정렬, 이미지 처리의 타일 분할 같은 시스템도 이런 생각을 기반으로 합니다.

어떻게 동작하는가?

1) 병합 정렬의 분할 정복 구조

배열을 계속 절반으로 나누고, 길이 1짜리 배열들을 정렬된 상태로 합쳐 올라갑니다.

분할 자체는 싸고, 병합 단계에서 각 레벨마다 전체 n개를 한 번씩 처리한다.

2) 재귀식으로 성능 보기

분할 정복 알고리즘은 보통 다음 형태의 재귀식으로 시간 복잡도를 표현합니다.

T(n) = a · T(n / b) + f(n)
  • a — 하위 문제 개수
  • b — 문제 크기가 얼마나 줄어드는지
  • f(n) — 나누고 합치는 데 드는 비용
알고리즘재귀식복잡도설명
이진 탐색T(n)=T(n/2)+O(1)O(log n)절반 하나만 봄
병합 정렬T(n)=2T(n/2)+O(n)O(n log n)양쪽 해결 + 병합 O(n)
퀵 정렬 평균T(n)=2T(n/2)+O(n)O(n log n)균형 분할 가정
카라츠바 곱셈T(n)=3T(n/2)+O(n)O(n^1.585)곱셈 4번을 3번으로 줄임

3) Java 코드로 보기 — 병합 정렬

void mergeSort(int[] a) {
    int[] tmp = new int[a.length];
    mergeSort(a, tmp, 0, a.length - 1);
}

void mergeSort(int[] a, int[] tmp, int lo, int hi) {
    if (lo >= hi) return;                 // 충분히 작은 문제

    int mid = lo + (hi - lo) / 2;
    mergeSort(a, tmp, lo, mid);           // Divide + Conquer
    mergeSort(a, tmp, mid + 1, hi);
    merge(a, tmp, lo, mid, hi);           // Combine
}

void merge(int[] a, int[] tmp, int lo, int mid, int hi) {
    int i = lo, j = mid + 1, k = lo;
    while (i <= mid && j <= hi) {
        tmp[k++] = (a[i] <= a[j]) ? a[i++] : a[j++];
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= hi) tmp[k++] = a[j++];

    for (int p = lo; p <= hi; p++) a[p] = tmp[p];
}

// 이진 탐색도 분할 정복의 한 예
int binarySearch(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] == target) return mid;
        if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

시간/공간 복잡도

분할 정복의 비용은 재귀 깊이와 각 레벨에서 처리하는 총량으로 보면 직관적입니다. 병합 정렬은 깊이가 log n이고, 각 레벨마다 병합으로 전체 n개를 처리하므로 O(n log n)입니다.

알고리즘시간공간비고
이진 탐색O(log n)O(1)반복문 기준
병합 정렬O(n log n)O(n)안정 정렬
퀵 정렬평균 O(n log n), 최악 O(n²)O(log n)피벗 선택 중요
분할 정복 최대 부분 배열O(n log n)O(log n)DP/Kadane은 O(n)

실무에서 어디에 쓰이는가?

  • 정렬 라이브러리 — 병합 정렬, 퀵 정렬, introsort, Timsort의 기반.
  • 검색 — 이진 탐색, B+Tree 노드 내부 탐색.
  • 병렬 처리 — 큰 작업을 독립 조각으로 나눠 여러 코어에서 처리.
  • 대규모 데이터 처리 — MapReduce의 map/reduce 구조도 넓은 의미의 분할-결합.
  • 계산 기하 — 최근접 점 쌍, 볼록 껍질 일부 알고리즘.
  • 신호 처리 — FFT는 대표적인 고급 분할 정복 알고리즘.

면접에 나올 질문과 답변

분할 정복과 DP의 차이를 설명해 주세요.
분할 정복은 문제를 독립적인 하위 문제로 나눠 각각 풀고 합칩니다. 병합 정렬의 왼쪽/오른쪽 절반은 겹치지 않습니다. DP는 하위 문제가 서로 겹쳐 같은 계산이 반복되므로 결과를 저장해 재사용합니다. 중복 부분 문제가 보이면 DP, 독립적인 분할이면 분할 정복을 먼저 떠올리면 됩니다.
병합 정렬이 O(n log n)인 이유는요?
배열을 절반씩 나누면 재귀 깊이가 log n입니다. 각 깊이에서는 여러 조각이 있지만, 조각들을 모두 합치면 전체 원소 n개를 한 번씩 처리합니다. 그래서 레벨당 O(n), 레벨 수 O(log n), 전체 O(n log n)입니다.
퀵 정렬도 분할 정복인데 왜 최악이 O(n²)인가요?
퀵 정렬의 분할이 균형 잡히면 병합 정렬처럼 O(n log n)이지만, 피벗이 계속 최솟값/최댓값이 되면 한쪽 크기가 0, 다른 쪽이 n-1이 됩니다. 그러면 재귀 깊이가 n이 되고 각 레벨에서 파티션 O(n)이 누적되어 O(n²)이 됩니다. 그래서 랜덤 피벗이나 중앙값-of-3, introsort 같은 방어 전략을 씁니다.
마스터 정리는 언제 쓰나요?
T(n)=aT(n/b)+f(n) 형태의 균일한 분할 정복 재귀식을 빠르게 해석할 때 씁니다. 병합 정렬처럼 하위 문제가 같은 크기로 나뉘고, 결합 비용이 명확할 때 유용합니다. 다만 퀵 정렬 최악처럼 분할 크기가 매번 달라지는 경우에는 그대로 적용하기 어렵습니다.
