🧮 알고리즘

알고리즘(Algorithm)은 어떤 문제를 풀기 위한 명확한 절차입니다. 같은 문제도 어떤 알고리즘을 쓰느냐에 따라 1초에 끝날 수도, 100년이 걸릴 수도 있죠. 여기서는 면접·코딩테스트·실무에서 가장 자주 쓰이는 알고리즘들을 왜 그렇게 동작하는지와 언제 어떤 걸 골라야 하는지를 중심으로 정리합니다.

📚 학습 가이드 — 핵심 먼저, 그다음 심화

먼저 핵심 개념을 한 번 훑어 큰 그림을 만든 뒤, 깊이 알고 싶은 주제는 같은 카테고리의 심화 / 상세로 들어가세요. 예: 정렬 입문 → 퀵/병합/힙 정렬 단독 / DP → 세그먼트 트리.

🧩 핵심 개념

알고리즘을 처음 배우거나 코딩 테스트·면접 직전에 한 번 훑기 좋은 우선순위 개념들.

탐색(Search)

"찾는다"는 가장 기본적인 작업. 데이터의 형태(정렬됨/그래프 등)에 따라 적절한 알고리즘이 달라진다.

  • 이진 탐색(Binary Search) 정렬된 배열을 절반씩 줄여 O(log n)에 찾기. lower_bound / upper_bound 변형이 필수.
  • BFS / DFS (그래프 탐색) 너비 우선(큐)과 깊이 우선(스택/재귀). 가중치 없는 최단 경로, 사이클 탐지 등.

정렬(Sorting)

가장 자주 등장하는 도구. 안정성·in-place 여부·평균/최악 복잡도가 선택 기준이 된다.

  • 정렬 알고리즘 입문 버블/선택/삽입/병합/퀵/힙 정렬 비교. 안정 정렬, in-place, 비교 vs 비비교.

그래프 알고리즘

최단 경로, 신장 트리, 위상 정렬 등 그래프의 구조를 활용하는 문제들.

  • 다익스트라(Dijkstra) 가중치 그래프의 최단 경로. 우선순위 큐로 O((V+E) log V).
  • 벨만-포드 / 플로이드-워셜 음수 가중치 허용, 모든 쌍 최단 경로.
  • 유니온-파인드(Union-Find) 경로 압축 + 랭크 합치기로 거의 O(1)에 동치 클래스 관리.
  • 위상 정렬(Topological Sort) DAG에서 의존 순서로 정점 나열. 빌드 시스템·작업 스케줄러의 기반.
  • 최소 신장 트리 — 크루스칼 / 프림 모든 정점을 잇는 최소 비용 트리. 네트워크 설계에서 자주 등장.

설계 기법(Design Paradigm)

특정 알고리즘이 아니라, 문제를 푸는 사고의 큰 틀.

  • 동적 계획법(DP) 중복 부분 문제를 메모이제이션 또는 타뷸레이션으로 해결.
  • 그리디(Greedy) 매 순간 가장 좋아 보이는 선택. 최적 부분 구조 + 그리디 선택 속성이 필요.
  • 분할 정복(Divide & Conquer) 문제를 작은 부분으로 나눠 풀고 합치기. 병합 정렬, 퀵 정렬, 마스터 정리.
  • 백트래킹(Backtracking) 결정을 시도하다 막히면 되돌아가기. N-Queens, 부분집합, 순열 생성.

비트와 수학적 기법

면접·코딩테스트에서 의외로 자주 등장하는 영역.

  • 비트 조작 / 비트마스킹 AND/OR/XOR/Shift, 부분집합 표현, 켜진 비트 수, 비트 DP, XOR 트릭.

🔬 심화 / 상세

핵심 개념을 한 번 본 뒤, 면접에서 깊이 물어볼 만한 주제를 별도 페이지로 정리합니다. (작성 예정)

정렬 심화

  • 퀵 정렬 단독 작성 예정 피벗 선택, Lomuto/Hoare 파티션, 3-way partition, introsort.
  • 병합 정렬 단독 작성 예정 외부 병합(디스크 정렬), TimSort 변형.
  • 힙 정렬 단독 작성 예정 힙 구성 단계별 시각화.
  • 비비교 정렬 작성 예정 계수 / 기수 / 버킷 정렬.

그래프·자료구조 심화

  • 세그먼트 트리 / 펜윅 트리(BIT) 작성 예정 구간 합·최솟값 쿼리, lazy propagation.
  • 네트워크 플로우 작성 예정 포드-풀커슨, 에드몬드-카프, 디닉.
  • LCA / 희소 테이블 작성 예정 트리에서 두 노드의 최소 공통 조상.

문자열 심화

  • 문자열 알고리즘 작성 예정 KMP, Rabin-Karp, Z 알고리즘, 접미사 배열.
