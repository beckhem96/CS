최소 신장 트리(Minimum Spanning Tree)

최소 신장 트리(MST)란, 가중치가 있는 무방향 연결 그래프에서 모든 정점을 연결하되 사이클 없이, 간선 가중치 합이 최소가 되는 트리입니다. 대표 알고리즘은 크루스칼(Kruskal)과 프림(Prim)입니다.

구체적인 개념 설명

비유하면 여러 도시를 광케이블로 연결하려고 할 때, 모든 도시가 서로 통신 가능해야 하지만 케이블 비용은 최소로 하고 싶습니다. 모든 도시를 직접 다 연결하면 비싸고, 중복 경로(사이클)는 비용만 늘립니다. 그래서 필요한 간선만 골라 트리를 만들고, 그중 비용이 가장 작은 것이 MST입니다.

MST의 조건은 세 가지입니다.

  • 모든 정점을 포함한다.
  • 사이클이 없다 — 정점이 V개라면 간선은 정확히 V-1개.
  • 가중치 합이 최소다.

최단 경로와 다르다

MST는 전체 네트워크를 싸게 연결하는 문제이고, 최단 경로는 특정 출발점에서 특정 도착점까지 가장 짧은 길을 찾는 문제입니다. MST 안에서 두 정점 사이의 경로가 원래 그래프의 최단 경로라는 보장은 없습니다.

왜 필요한가?

네트워크 설계, 전기 배선, 도로/철도 최소 연결, 클러스터링 등에서 "모두 연결하되 비용은 최소"라는 요구가 자주 등장합니다. MST는 이 요구를 정확히 수학적으로 모델링합니다.

또한 MST 알고리즘은 그리디가 왜 맞는지 보여주는 대표 사례입니다. 크루스칼은 가장 싼 간선부터 고르고, 프림은 현재 연결된 집합에서 바깥으로 나가는 가장 싼 간선을 고릅니다. 둘 다 컷 속성(cut property) 덕분에 최적해를 보장합니다.

어떻게 동작하는가?

1) 예제 그래프와 MST

모든 정점이 연결되고, 간선은 V-1개이며, 사이클이 없다.

2) 크루스칼 알고리즘

모든 간선을 가중치 오름차순으로 정렬하고, 싼 간선부터 보면서 사이클을 만들지 않으면 선택합니다. 사이클 여부는 유니온-파인드로 빠르게 확인합니다.

  1. 간선을 가중치 오름차순으로 정렬.
  2. 각 정점을 독립 집합으로 초기화.
  3. 간선을 하나씩 보며 양 끝 정점이 다른 집합이면 선택하고 union.
  4. 간선이 V-1개 선택되면 종료.

3) 프림 알고리즘

하나의 정점에서 시작해, 현재 MST에 포함된 정점 집합에서 바깥으로 나가는 간선 중 가장 싼 간선을 계속 고릅니다. 다익스트라처럼 우선순위 큐를 쓰지만, 거리 합이 아니라 집합에 연결되는 간선 비용을 기준으로 고릅니다.

크루스칼은 간선 중심, 프림은 정점 집합 확장 중심.

4) Java 코드로 보기

import java.util.*;

class Edge {
    int u, v, w;
    Edge(int u, int v, int w) { this.u = u; this.v = v; this.w = w; }
}

class UnionFind {
    int[] parent, rank;
    UnionFind(int n) {
        parent = new int[n]; rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }
    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }
    boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;
        if (rank[ra] < rank[rb]) parent[ra] = rb;
        else if (rank[ra] > rank[rb]) parent[rb] = ra;
        else { parent[rb] = ra; rank[ra]++; }
        return true;
    }
}

// 크루스칼
int kruskal(int n, List<Edge> edges) {
    edges.sort(Comparator.comparingInt(e -> e.w));
    UnionFind uf = new UnionFind(n);
    int cost = 0, count = 0;

    for (Edge e : edges) {
        if (uf.union(e.u, e.v)) {
            cost += e.w;
            if (++count == n - 1) break;
        }
    }
    if (count != n - 1) throw new IllegalStateException("그래프가 연결되어 있지 않음");
    return cost;
}

// 프림: adj[u] = {to, weight}
int prim(List<List<int[]>> adj, int n) {
    boolean[] visited = new boolean[n];
    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[1]));
    pq.offer(new int[]{0, 0}); // {정점, 연결 비용}

    int cost = 0, count = 0;
    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int u = cur[0], w = cur[1];
        if (visited[u]) continue;

        visited[u] = true;
        cost += w;
        count++;

        for (int[] edge : adj.get(u)) {
            if (!visited[edge[0]]) pq.offer(edge);
        }
    }
    if (count != n) throw new IllegalStateException("그래프가 연결되어 있지 않음");
    return cost;
}

시간/공간 복잡도

알고리즘시간공간좋은 상황
크루스칼O(E log E)O(V)간선 리스트가 있고, 희소 그래프
프림 + 이진 힙O(E log V)O(V + E)인접 리스트, 연결 그래프
프림 + 인접 행렬O(V²)O(V²)밀집 그래프, V가 작음

실무에서 어디에 쓰이는가?

  • 네트워크 설계 — 최소 비용으로 모든 지점 연결(통신망, 전력망, 배관망).
  • 클러스터링 — MST를 만든 뒤 큰 간선을 끊어 군집 분리.
  • 이미지 분할 — 픽셀/영역을 그래프로 보고 유사도 기반 연결.
  • 근사 알고리즘 — TSP 근사해 구성의 빌딩 블록.
  • 게임/지도 — 절차적 맵 생성에서 방들을 최소 연결 후 추가 간선으로 순환 경로 생성.

면접에 나올 질문과 답변

MST와 최단 경로 트리는 같은 건가요?
아닙니다. MST는 전체 간선 비용 합을 최소화하는 트리이고, 최단 경로 트리는 특정 출발점에서 각 정점까지의 거리를 최소화하는 트리입니다. MST 안의 두 정점 사이 경로가 원래 그래프의 최단 경로일 필요는 없습니다.
크루스칼과 프림 중 어느 걸 써야 하나요?
간선 리스트가 주어지고 그래프가 희소하면 크루스칼이 단순합니다. 간선을 정렬하고 유니온-파인드로 사이클만 막으면 됩니다. 인접 리스트로 연결 그래프를 탐색하듯 확장하고 싶으면 프림이 자연스럽습니다. 밀집 그래프라면 인접 행렬 기반 프림 O(V²)이 힙보다 단순하고 빠를 수 있습니다.
왜 가장 싼 간선을 계속 골라도 최적해가 되나요?
MST의 핵심 정리는 컷 속성입니다. 그래프의 정점을 두 그룹으로 나누는 임의의 컷에서, 그 컷을 가로지르는 가장 싼 간선은 어떤 MST에 반드시 포함될 수 있습니다. 크루스칼과 프림은 이 안전한 간선을 반복해서 고르는 알고리즘이라 최적해를 보장합니다.
그래프가 연결되어 있지 않으면 어떻게 되나요?
모든 정점을 하나의 트리로 연결할 수 없으므로 MST가 존재하지 않습니다. 대신 각 연결 요소마다 MST를 만든 최소 신장 숲(Minimum Spanning Forest)을 만들 수 있습니다. 코드에서는 선택된 간선 수가 V-1보다 적거나, 방문한 정점 수가 V보다 작으면 연결 그래프가 아니라고 판단합니다.
