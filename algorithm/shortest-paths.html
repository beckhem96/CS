벨만-포드 / 플로이드-워셜

벨만-포드는 음수 가중치가 있는 그래프에서도 한 출발점의 최단 경로를 구하고, 음수 사이클까지 감지할 수 있는 알고리즘입니다. 플로이드-워셜은 모든 정점 쌍 사이의 최단 거리를 한 번에 구하는 O(V³) 알고리즘입니다.

구체적인 개념 설명

다익스트라는 빠르지만 음수 간선이 있으면 깨집니다. 반면 벨만-포드는 느린 대신 더 튼튼합니다. 모든 간선을 여러 번 훑으면서 "이 간선을 지나가면 더 짧아질까?"를 반복 확인합니다. 정점이 V개라면, 사이클 없는 최단 경로는 간선을 최대 V-1개만 사용하므로 V-1번 전체 간선 완화를 반복하면 충분합니다.

플로이드-워셜은 관점이 다릅니다. "i에서 j로 갈 때, 중간에 k번 정점을 거쳐도 되는가?"를 하나씩 늘려갑니다. 출발점 하나가 아니라 모든 출발점과 모든 도착점의 최단 거리를 동시에 갱신합니다.

언제 무엇을 쓰나?

  • 다익스트라 — 음수 없음, 한 출발점, 빠른 성능 필요.
  • 벨만-포드 — 음수 간선 있음, 음수 사이클 감지 필요.
  • 플로이드-워셜 — 정점 수가 작고, 모든 쌍 최단 거리가 필요.

왜 필요한가?

현실의 비용은 항상 양수만 있는 게 아닙니다. 환율 차익, 할인/보상, 에너지 회수처럼 어떤 간선은 비용을 줄이는 효과를 가질 수 있습니다. 이런 그래프에서는 다익스트라의 "한 번 확정하면 끝"이라는 가정이 맞지 않습니다.

또 어떤 문제는 한 출발점이 아니라 모든 지점 사이의 거리가 필요합니다. 도시 간 최단 거리 표, 작은 규모 네트워크의 모든 지연시간 계산, 그래프의 전이 폐쇄 같은 문제는 플로이드-워셜이 단순하고 강력합니다.

어떻게 동작하는가?

1) 벨만-포드 — 간선 완화 반복

출발점 A에서 시작해 모든 간선을 반복해서 완화합니다. 음수 간선 C→B(-2) 때문에 B의 거리가 나중에 더 줄어들 수 있습니다.

음수 간선이 있으면 나중에 더 짧은 경로가 발견될 수 있으므로 반복 완화가 필요하다.
반복ABCD설명
초기0∞∞∞출발점만 0
1회0357A→B, A→C, C→B, B→D 갱신
2회0356B가 3으로 줄어든 뒤 B→D로 D=6

2) 음수 사이클 감지

V-1번 완화가 끝났는데도 한 번 더 완화했을 때 거리가 줄어든다면, 간선을 V개 이상 쓰는 더 짧은 경로가 있다는 뜻입니다. 이는 어떤 사이클을 돌수록 비용이 계속 줄어드는 음수 사이클이 존재한다는 뜻입니다.

3) 플로이드-워셜 — 중간 정점을 하나씩 허용

dist[i][j]를 i에서 j까지의 현재 최단 거리라고 하고, k를 0부터 V-1까지 늘립니다. 각 단계에서 "i에서 j로 바로 가는 것"과 "i → k → j로 가는 것" 중 더 짧은 값을 고릅니다.

k를 경유지로 허용했을 때 더 짧아지는 모든 쌍을 동시에 갱신한다.

4) Java 코드로 보기

import java.util.*;

class Edge {
    int from, to, weight;
    Edge(int f, int t, int w) { from = f; to = t; weight = w; }
}

// 벨만-포드: 음수 사이클이 있으면 null 반환
long[] bellmanFord(int n, List<Edge> edges, int start) {
    long INF = Long.MAX_VALUE / 4;
    long[] dist = new long[n];
    Arrays.fill(dist, INF);
    dist[start] = 0;

    for (int i = 0; i < n - 1; i++) {
        boolean updated = false;
        for (Edge e : edges) {
            if (dist[e.from] == INF) continue;
            if (dist[e.to] > dist[e.from] + e.weight) {
                dist[e.to] = dist[e.from] + e.weight;
                updated = true;
            }
        }
        if (!updated) break;
    }

    for (Edge e : edges) {
        if (dist[e.from] != INF && dist[e.to] > dist[e.from] + e.weight) {
            return null; // 음수 사이클
        }
    }
    return dist;
}

// 플로이드-워셜: 모든 쌍 최단 거리
long[][] floydWarshall(long[][] dist) {
    int n = dist.length;
    long INF = Long.MAX_VALUE / 4;

    for (int k = 0; k < n; k++) {
        for (int i = 0; i < n; i++) {
            if (dist[i][k] == INF) continue;
            for (int j = 0; j < n; j++) {
                if (dist[k][j] == INF) continue;
                dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
            }
        }
    }
    return dist;
}

시간/공간 복잡도

알고리즘대상시간공간특징
벨만-포드한 출발점O(VE)O(V)음수 간선, 음수 사이클 감지
플로이드-워셜모든 쌍O(V³)O(V²)구현 단순, V가 작을 때 좋음
다익스트라한 출발점O((V+E)logV)O(V+E)음수 간선 불가, 빠름

실무에서 어디에 쓰이는가?

  • 네트워크 라우팅 — RIP 같은 거리 벡터 라우팅은 벨만-포드 계열 아이디어를 사용.
  • 환율 차익 탐지 — 로그 변환 후 음수 사이클 탐지로 무위험 차익 가능 여부 확인.
  • 작은 그래프의 모든 쌍 거리 — 도시 수가 적은 문제, 게임 맵의 전처리 거리표.
  • 전이 폐쇄 — 플로이드-워셜 변형으로 도달 가능성 행렬 계산.
  • 음수 보상/할인 모델 — 비용이 줄어드는 간선이 있는 최적화 문제.

면접에 나올 질문과 답변

벨만-포드는 왜 V-1번만 반복하면 되나요?
사이클이 없는 최단 경로는 같은 정점을 두 번 방문할 필요가 없으므로 간선을 최대 V-1개만 사용합니다. 한 번의 반복은 최단 경로에서 사용할 수 있는 간선 수를 하나씩 늘리는 효과가 있습니다. 따라서 V-1번이면 모든 단순 경로의 최단 거리가 반영됩니다.
음수 사이클은 어떻게 감지하나요?
V-1번 완화 후 한 번 더 모든 간선을 확인합니다. 이때도 어떤 거리든 줄어든다면, 간선을 V개 이상 사용해서 더 짧아진다는 뜻이고 이는 사이클을 포함한다는 뜻입니다. 그 사이클의 총 비용이 음수이기 때문에 돌수록 거리가 계속 줄어듭니다.
플로이드-워셜은 언제 다익스트라 여러 번보다 낫나요?
정점 수가 작고 그래프가 밀집되어 있거나, 구현 단순성이 중요하고 모든 쌍 거리가 필요할 때 좋습니다. V가 400 이하 정도인 알고리즘 문제에서는 O(V³)이 충분히 통과하는 경우가 많습니다. 반대로 V가 크고 간선이 희소하면 다익스트라를 여러 번 돌리는 편이 보통 더 효율적입니다.
플로이드-워셜에서도 음수 사이클을 알 수 있나요?
가능합니다. 알고리즘이 끝난 뒤 dist[i][i] < 0인 정점이 있으면 i에서 출발해 다시 i로 돌아오는 음수 사이클이 있다는 뜻입니다. 자기 자신까지의 최단 거리는 원래 0이어야 하는데, 음수 사이클을 돌면 더 작아질 수 있기 때문입니다.
